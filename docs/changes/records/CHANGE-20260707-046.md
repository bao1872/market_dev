# CHANGE-20260707-046: 修复 pytdx_adapter 对 aware 1m start/end 的比较异常

## 变更编号
CHANGE-20260707-046

## 任务名称
修复 `pytdx_adapter.get_minute_bars` 接收 `Asia/Shanghai` aware start/end 时与 naive `datetime` 列比较的 `Invalid comparison between dtype=datetime64[us] and Timestamp` 异常。

## 需求出处
生产验证：PR #35（CHANGE-20260707-045）部署后，下午盘 `worker-monitor` 仍无法获得 1m 数据，日志出现新的异常：
- `Pytdx 拉取 1m 失败 ... Invalid comparison between dtype=datetime64[us] and Timestamp`

## 修改前行为
- PR #35 已保证 `MarketDataAggregationService` 传入 `pytdx_adapter.get_minute_bars` 的 `start`/`end` 同为 `Asia/Shanghai` aware datetime；
- 但 `pytdx_adapter._fetch_with_retry` 内部将 pytdx 返回的 1m 数据 `datetime` 列显式 `tz_localize(None)` 为 naive；
- `get_minute_bars` 第 619 行 `mask = (df["datetime"] >= start_ts) & (df["datetime"] <= end_ts)` 中，`start_ts`/`end_ts` 为 aware Timestamp，`df["datetime"]` 为 `datetime64[us]` naive，触发：
  - `Invalid comparison between dtype=datetime64[us] and Timestamp`
- 导致：
  - `worker-monitor` 每个周期对 19 个自选股的 1m 拉取全部失败；
  - `monitor_evaluations` / `monitor_states` / `strategy_events` 仍为 0；
  - 个股详情 `/bars?timeframe=1d&include_realtime=true` 交易时段仍无法合成 partial daily bar。

## 修改后行为
- `pytdx_adapter.get_minute_bars` 在按时间范围过滤前，若 `start_ts`/`end_ts` 含时区，则先 `tz_convert("Asia/Shanghai")` 再 `tz_localize(None)`，与 pytdx 返回的 naive `datetime` 列保持一致；
- naive 输入保持兼容；
- MDAS aware start/end 与 pytdx naive 数据列的比较链路打通。
