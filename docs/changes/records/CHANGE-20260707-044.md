# CHANGE-20260707-044: DSA visual_segments 时间格式按 timeframe 序列化

## 变更编号
CHANGE-20260707-044

## 任务名称
PR #34 — 修复 15m/1h DSA 开关可打开但 canvas 看不到线的问题

## 需求出处
PR #33 已清理前端 1d-only / 1w-1mo skip 硬编码，开关问题基本解决。但 15m/1h DSA 仍看不到线。

明确根因：
- `dsa_polyline` renderer 不直接画 `dsa_vwap` 数组，而是画 `visual_segments.points`
- 后端 `dynamic_swing_anchored_vwap.py::_make_segment` 把 segment point time 写死为 `strftime("%Y-%m-%d")`
- `DSASelector.compute_indicators` 也把 `time` 写死为 `idx.strftime("%Y-%m-%d")`
- `compute_dsa_bundle` `anchor["time"]` 同样写死 YYYY-MM-DD
- 15m/1h K 线时间是 `YYYY-MM-DDTHH:MM:SS` 或 canonical `"YYYY-MM-DD HH:MM"`
- `visual_segments.points.time` 只有日期，`normalizeChartTime(pt.time, '15m')` 返回 `null`
- renderer matched=0，开关打开也画不出来

明确约束：
- 不改 DSA/BB 数学公式（`dsa_vwap` / `dsa_dir` / `regime_id` / `visual_segments.direction` / `points.value` 不变）
- 不做事件/相似场景/板块聚合
- 不新增表/worker/全市场循环
- 不生成截图/coverage/大日志

## 修改前行为
- `_make_segment` segment point time：`pd.Timestamp(x).strftime("%Y-%m-%d")`
- `_show_segments`（`__main__` plotly figure 路径）segment time：`seg_times[i].strftime("%Y-%m-%d")`
- `compute_dsa_bundle` `anchor["time"]`：`[pd.Timestamp(lab["x"]).strftime("%Y-%m-%d") for lab in pivot_labels]`
- `DSASelector.compute_indicators` 返回 `time`：`[idx.strftime("%Y-%m-%d") for idx in bundle["factor_time"]]`
- 1d 行为正确（无时间部分，YYYY-MM-DD 即 canonical key）
- 15m/1h segment time 丢失时间信息，`normalizeChartTime('15m'/'1h')` 返回 `null`，renderer matched=0

## 修改后行为
- 新增 `format_dsa_time(x)` 函数（`dynamic_swing_anchored_vwap.py`）：
  - 若 `pd.Timestamp(x)` 的 `hour/minute/second/microsecond` 全为 0 → 返回 `strftime("%Y-%m-%d")`
  - 否则 → 返回 `isoformat()`（含 `T`）
- `_make_segment` segment point time：`format_dsa_time(x)`
- `_show_segments` segment time：`format_dsa_time(seg_times[i])`
- `compute_dsa_bundle` `anchor["time"]`：`[format_dsa_time(lab["x"]) for lab in pivot_labels]`
- `DSASelector.compute_indicators` 返回 `time`：`[format_dsa_time(idx) for idx in bundle["factor_time"]]`
- 1d 仍返回 `YYYY-MM-DD`（向后兼容）
- 15m/1h 返回 `YYYY-MM-DDTHH:MM:SS...`（与 K线 `displayTimes` canonical 对齐）
- 前端 `renderDsaPolyline` 在 `?debugIndicatorAlignment=1` 时调用 `computeDsaSegmentMatchStats` 输出 segment matched 诊断：
  - `total` / `matched` / `ratio` / `degradedReason` / `firstSegTime` / `lastSegTime` / `firstDisplayTime` / `lastDisplayTime`
  - `degradedReason`：`null` / `no_segments` / `no_points` / `no_display_times` / `segment_time_no_match`
- 不改变 `dsa_vwap` / `dsa_dir` / `regime_id` / `visual_segments.direction` / `points.value` 的数学含义，只改时间序列化格式

## 风险
- 后端 `format_dsa_time` 是新序列化函数，若 `pd.Timestamp(x)` 解析失败可能抛异常；
  但原 `strftime("%Y-%m-%d")` 同样要求 `pd.Timestamp(x)` 可解析，未引入新解析路径。
- 前端 `computeDsaSegmentMatchStats` 仅在 `?debugIndicatorAlignment=1` 时调用，不影响生产渲染性能。
- 1d/1w/1mo 行为完全不变（`hour/minute/second/microsecond` 全 0 仍走 `strftime("%Y-%m-%d")` 分支）。
