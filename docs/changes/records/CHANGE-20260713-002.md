## 2. 修改前理解

- 当前产品行为：
  - StockContext API 在无快照或快照归属缺失时统一返回空 state，前端 EventStatePanel 统一显示"暂无可用状态数据"，用户无法判断具体原因；
  - 生产库 `stock_feature_snapshots` 约 21272 行，其中 `source_run_id` 为 NULL 的约 100%（legacy 历史 backfill 未关联 run），精确关联数为 0；
  - EventStatePanel 的 reasonCode 文案逻辑内联在 .tsx 组件中，无法被纯 TypeScript 测试覆盖。
- 当前前端流程：EventStatePanel 调用 `useStockContext` 获取 state，state=null 时显示统一文案。
- 当前后端流程：`_get_snapshot_for_instrument` 返回 snapshot 或 None，不返回原因；`_build_data_quality` 不含 reasonCode 字段。
- 已阅读的当前设计文档：AGENTS.md、docs/current/MANIFEST.md、docs/current/02-data-api-contracts.md、docs/current/05-testing-acceptance.md、docs/maps/test-coverage-map.md。
- 发现的代码—文档—生产冲突：生产库 source_run_id NULL 率 ~100% 导致 StockContext 大量空态，但无明确原因供前端展示。

## 3. 修改目标

- 当前问题：StockContext 空态原因不明确；legacy 快照未关联 run；EventStatePanel 文案逻辑不可测试。
- 修改后行为：
  1. **reasonCode 机制**：StockContext API 返回 `dataQuality.reasonCode` 字段解释空态原因（`no_published_full_run`/`snapshot_missing`/`snapshot_run_not_linked`/`legacy_snapshot_ambiguous`/`null`）；`dataQuality` 含 `runTradeDate`/`runPublishedAt`/`hasSucceededRun`/`hasSnapshot`/`degradedReasons`。
  2. **快照归属修复工具**：新增 `tools/repair_snapshot_run_ownership.py`，默认 dry-run，按 (trade_date, schema_version, primary/secondary timeframe, adj) 匹配 canonical succeeded+published+full run，唯一候选→repairable，0 候选→orphan，>1 候选→ambiguous；写模式单事务、失败 rollback、幂等。
  3. **EventStatePanel 纯函数抽取**：将 reasonCode 文案逻辑抽取到 `frontend/src/features/research-context/reasonCodeMessages.ts`（纯 TypeScript，无 JSX），使文案逻辑可被 `node --experimental-strip-types --test` 测试覆盖。
  4. **GET context 只读**：GET `/api/v1/stocks/{symbol}/context` 不得产生任何写副作用（不创建 snapshot/run）。
- 验收标准：9 项 required 测试全通过；生产 dry-run + 实际修复统计；抽查 10 自选股 + 10 市场股 + 1 异常股；最终 E2E 12 项检查。
- 明确不修改：Capture 路由、飞书投递、Migration 历史、DSA 计算逻辑、MACD/SQZMOM 算法、阶段一已完成的 ConsensusZone 移除/图层单一真源/K线 viewport/published snapshot 保护。

## 5. 实际修改

### 代码文件

**后端**：
- `backend/app/api/stock_context.py`：`_get_snapshot_for_instrument` 改为返回 `(snapshot, reasonCode)` 元组；`_build_data_quality` 新增 `reasonCode`/`runTradeDate`/`runPublishedAt`/`hasSucceededRun`/`hasSnapshot`/`degradedReasons` 字段；`_build_stock_context` 在 run=None 时返回 `no_published_full_run`，snapshot=None 时返回 `snapshot_missing`；`effective_reason = None if snapshot is not None else reason_code`（reasonCode 只在 state=null 时显示）。
- `backend/tests/test_stock_state_and_events.py`：新增 `_create_db_run` 和 `_create_db_snapshot` 辅助函数；新增 6 个 reasonCode API 测试（test_p02_context_*）；新增 1 项无写副作用测试（test_p02_context_get_no_write_side_effect）；修复 pre-existing ruff F841。
- `backend/tests/test_repair_snapshot_run_ownership.py`：**新建**，2 项测试（dry-run 不写库 + apply 正确且幂等），使用 sys.path 操纵导入 tools 模块。

**工具**：
- `tools/repair_snapshot_run_ownership.py`：**新建**（从阶段一已创建，本轮验证完整）。默认 dry-run；按 (trade_date, schema_version, primary/secondary timeframe, adj) 匹配 canonical succeeded+published+full run；输出 total/repairable/repaired/ambiguous/unmatched；写模式单事务、失败 rollback、批量 UPDATE、幂等（WHERE source_run_id IS NULL）。

**前端**：
- `frontend/src/features/research-context/reasonCodeMessages.ts`：**新建**，纯 TypeScript 函数 `getReasonCodeMessage(reasonCode, runTradeDate)` 返回 `{title, meta?}`；覆盖 5 种 reasonCode + null + 未知 code。
- `frontend/src/features/research-context/EventStatePanel.tsx`：移除内联的 `getReasonCodeMessage` 函数；改为从 `./reasonCodeMessages` 导入；JSX 中使用 IIFE 调用纯函数替代多个条件渲染。
- `frontend/src/features/research-context/__tests__/reasonCodeMessages.test.ts`：**新建**，8 个子测试覆盖所有 reasonCode 场景（no_published_full_run / snapshot_missing 含/不含 runTradeDate / snapshot_run_not_linked 含"待修复归属" / legacy_snapshot_ambiguous / null / 未知 code / 所有已知 code 非默认文案）。
