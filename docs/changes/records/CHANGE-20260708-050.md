# CHANGE-20260708-050: 修复 Monitor 与 Notification latest-event 图片 Capture Token Claims

## 变更编号
CHANGE-20260708-050

## 任务名称
修复 monitor 图片链路与 notification latest-event 图片 Capture Token claims 缺失

## 需求出处
用户发现 monitor 文字通知已收到但图片缺失。静态代码审查定位根因：
1. `backend/app/services/monitor_batch_service.py::_send_chart_images_via_outbox()` 生成 capture token 时只传 `subject/event_id/expires_delta`；
2. `backend/app/services/notification_service.py::test_channel_latest_event()` 也只传 `subject/event_id/expires_delta`；
3. 但 `backend/app/core/deps.py::get_capture_token_payload` 要求 `scope=stock_detail_capture` 且必须含 `user_id/instrument_id/event_id`；
4. `backend/app/api/capture.py` 还校验 `token.instrument_id == path instrument_id`。

字段缺失会导致 worker-capture 返回 401/403，`image_url` 为空，后续 image Outbox / MessageDelivery 不生成，但文字通知不受影响。

## 修改前行为
- `monitor_batch_service._send_chart_images_via_outbox()` 生成的 capture token 只有 `sub/exp/type/event_id`，缺少 `scope/user_id/instrument_id`；
- `notification_service.test_channel_latest_event()` 生成的 capture token 同样缺少上述字段；
- `stock_detail_feishu_service.py` 虽然已正确传递，但硬编码了 `"stock_detail_capture"` 字符串；
- `create_capture_token` 文档注释仍保留“旧调用方不传新参数时仍可工作”的过时描述。

## 修改后行为
- 新增纯常量模块 `backend/app/constants/capture.py`，定义 `CAPTURE_SCOPE_STOCK_DETAIL = "stock_detail_capture"`，避免服务层直接 import `app.core.deps` 带来的循环依赖风险；
- `backend/app/core/deps.py` 改为从 `app.constants.capture` 导入 `CAPTURE_SCOPE_STOCK_DETAIL`；
- `monitor_batch_service._send_chart_images_via_outbox()` 生成 token 时补全 `scope=CAPTURE_SCOPE_STOCK_DETAIL`、`instrument_id=str(inst_id)`、`user_id=str(user_ids[0])`；
- `notification_service.test_channel_latest_event()` 生成 token 时补全 `scope=CAPTURE_SCOPE_STOCK_DETAIL`、`instrument_id=str(event.instrument_id)`、`user_id=str(channel.user_id)`；
- `stock_detail_feishu_service.py` 硬编码字符串改为常量（业务逻辑不变）；
- `create_capture_token` 文档明确：所有调用 capture worker 的代码（monitor_batch_service / notification_service / stock_detail_feishu_service）都必须传递 `scope/user_id/instrument_id/event_id`，否则 `get_capture_token_payload` 会返回 401/403。
