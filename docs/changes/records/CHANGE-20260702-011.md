## 当前问题

1. `CaptureStockPage.tsx` 仍分别调用 `/instruments/by-symbol/{symbol}`、`/api/v1/instruments/{id}/bars`、`/api/v1/instruments/{id}/indicators`、`/api/v1/instruments/{id}/quote` 四个普通端点，未真正使用 Snapshot API。
2. `stock_capture_service.py` 截图 URL 未携带 `instrument_id`，前端无法从 URL 读取并调用 Snapshot API。
3. `get_share_status` 返回字段缺少 `capture_status` / `image_upload_status`，`ShareStatusResponse.overall_status` 枚举未包含 `partial_failed`。
4. ALIGN-018 被错误标为 CLOSED，实际前端未完全切换到 Snapshot 链路。

## 当前设计理解

- 专用 Capture 链路：`Capture Worker → /capture/stock/:symbol → CaptureStockPage → captureClient → GET /api/v1/capture/stocks/{instrument_id}/snapshot`。
- Capture Token 只能访问 `/api/v1/capture/*`，普通 access token 不能访问 Capture API。
- Snapshot API 一次返回 instrument/bars/indicators/events/quote 等截图所需完整数据。
- 图文状态机终态：`success`、`partial_failed`、`failed`；`partial_failed` 下支持仅重试图片。
