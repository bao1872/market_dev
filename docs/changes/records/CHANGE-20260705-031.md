# CHANGE-20260705-031

## 变更编号

CHANGE-20260705-031

## 任务名称

个股详情页新增结构状态因子面板（V1.7）

## 需求出处

- 用户要求实现"结构状态因子层 V1"：在个股详情页 `/stock/:symbol` 右侧新增结构状态因子面板，展示双周期（1d+15m）5 组结构因子。
- 关键约束：前端只渲染后端 DTO，严禁重新计算；250-500 bar lookback；15m 仅已完成 bar；Swing 仅已确认 pivot（无未来函数）；Node/POC 不可用时返回 null + degraded_reasons；不引入新依赖、不新增数据库表、不接入事件/通知/飞书。
- 设计文档：`.trae/documents/structural-state-factors-v1.md`（已用户确认）。

## 修改前行为

- 个股详情页 `/stock/:symbol` 工作区为单列布局（`grid-template-columns: minmax(0, 1fr)`）。
- 无结构状态因子 API、无相关后端服务、无前端面板组件。
- ATR 在多处重复实现（10+ 处），无 SSOT。

## 修改后行为

### 后端

- 新增 ATR SSOT `app.strategy_assets.algorithms.features.atr_utils.compute_atr`（Pine RMA 等价），消除 10+ 重复实现。
- 新增 `app.services.structural_factor_service.compute_structural_factors` 异步服务：
  - 5 组因子：DSA 段质量 / Swing 结构位置 / 成本节点 / 动量波动 / 成交参与；
  - 双周期（1d + 15m）独立计算；
  - 主周期 lookback=250，副周期 lookback=500；
  - 副周期 `include_realtime=False`（仅已完成 bar）；
  - Swing 使用 `_tv_pivots_confirmed`（已确认 pivot，无未来函数）；
  - 每组因子独立 try/except，单组失败返回 null + degraded_reasons，不阻塞其他组；
  - 返回 `primary`/`secondary`/`relation`/`meta` 结构。
- 新增 API 路由 `app/api/structural_factors.py`：
  - `GET /api/v1/instruments/{instrument_id}/structural-factors`；
  - 查询参数：`primary_timeframe`（默认 1d）、`secondary_timeframe`（默认 15m）、`adj`（默认 qfq）、`as_of`（默认 latest）；
  - 无认证要求（与 indicators API 一致）。
- `app/main.py` 注册 `structural_factors_router`。

## 风险

- 结构状态因子面板高度依赖后端 5 组因子计算，若任一 SSOT（ATR/Node/BB/Swing/SQZMOM/DSA）行为变化，面板显示会同步变化。
- 副周期 15m 仅使用已完成 bar，盘中实时性较弱（最迟 15 分钟延迟）；交易时段 60s 轮询仅刷新主周期（1d 实时合并由前端 `mergeRealtimeQuoteIntoBars` 单独处理，结构因子面板不复用该合并）。
- 双列布局在 1250-1400px 之间图表宽度被压缩；窄屏（≤1250px）隐藏面板。
- ATR SSOT `compute_atr` 与历史 `atr_pine` 等价已通过自测验证，但未替换所有历史调用点（本次只新增 SSOT，不强制迁移，避免大范围回归风险）。
