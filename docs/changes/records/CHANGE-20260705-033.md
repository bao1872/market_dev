## 任务名称

时序特征 V1（Temporal Features V1）+ 个股详情页结构状态面板隐藏开关

## 需求出处

- 用户 `/plan` 指令：在 PR #21（V1.8 结构状态因子）+ PR #22（MANIFEST baseline）已合并的基础上，新增 `Temporal Features V1` 后端服务/API 与前端结构状态面板隐藏开关。
- 关键约束（`/root/web_dev/advice.md`）：
  - 复用 V1.8 structural factors，不重写 DSA/SQZMOM/BB/ATR/Node；
  - V1 只支持 `as_of=latest`，不实现历史回放截断；
  - 单股票 1d + 15m，不新增 worker/大表/全市场循环；
  - 字段无法计算返回 null，组级异常写入 `degraded_reasons`，warmup 写入 `warmup_notes`；
  - 所有 anchor 取值必须 point-in-time，不得使用未来 bar；
  - 15m 不使用 15m DSA 位置类字段作为核心输入；
  - 前端结构状态面板默认隐藏，用户开关 + localStorage + 截图/参数强制隐藏。
- 设计文档：`.trae/documents/temporal-features-v1-and-hide-toggle.md` + `.trae/documents/temporal-features-v1-stage-6-9-completion.md`（用户确认）。

## 修改前行为

- V1.8 结构状态因子面板在 `/stock/:symbol` 默认显示，仅在 `capture=feishu` 截图模式和窄屏时隐藏。
- 不存在 Temporal Features API，无法获取时序变化量/持续度/派生关系。

## 修改后行为

### 后端 `temporal_feature_service.py`（新建）

- `compute_temporal_features(session, instrument_id, primary_timeframe="1d", secondary_timeframe="15m", adj="qfq", as_of="latest")` 返回：
  - `daily_context`（9 字段）：`daily_dsa_dir`、`daily_dsa_segment_duration_percentile`、`daily_dsa_slope_atr_per_bar`、`daily_dsa_efficiency_0_1`、`daily_price_position_in_swing_0_1`、`daily_distance_to_swing_high_atr`、`daily_distance_to_node_above_atr`、`daily_sqzmom_change_since_segment_start`、`daily_volume_percentile_change_since_segment_start`；
  - `m15_response`（9 字段）：`m15_price_position_in_swing_0_1`、`m15_position_change_since_swing_anchor`、`m15_distance_to_swing_high_atr`、`m15_distance_to_swing_low_atr`、`m15_sqzmom_change_since_swing_anchor`、`m15_sqzmom_abs_percentile`、`m15_sqz_off`、`m15_bb_bandwidth_change_since_swing_anchor`、`m15_volume_percentile_change_since_swing_anchor`；
  - `derived_relation`（3 字段）：`m15_position_relative_to_daily`、`m15_response_direction_relative_to_daily`（aligned/counter/null）、`m15_response_intensity`（mean(abs)）；
  - `meta`：`as_of`、`primary_timeframe`、`secondary_timeframe`、`degraded_reasons`、`warmup_notes`。
- 复用 V1.8 `compute_structural_factors` 获取 primary/secondary factors；调用 `_compute_all_factors_for_bars` + `_fetch_bars` 在 temporal service 内部使用。
- 历史序列重算（point-in-time，无未来函数）：
  - SQZMOM：`compute_sqzmom_lb` 取 `val_list[idx]`；
  - volume_percentile：`percentile_rank(volumes[idx], volumes[:idx+1], 120)`；
  - BB bandwidth：重算 bollinger bandwidth 序列再 `percentile_rank`；
  - DSA 历史 segments：`compute_dsa_bundle` + `_find_bar_index_by_time` 定位起止 bar index。
- swing anchor 规则：`bars_since_swing_low < bars_since_swing_high` → anchor=swing_low_bar；否则 anchor=swing_high_bar；`anchor_bar_idx = len(bars) - 1 - min(bsh, bsl)`。
- 每个组（daily/m15/derived）独立 try/except，单组失败返回 null + `degraded_reasons`，不阻塞整体返回。

### 后端 `temporal_features.py`（新建 API）

- `GET /api/v1/instruments/{instrument_id}/temporal-features`，参数 `primary_timeframe`/`secondary_timeframe`/`adj`/`as_of`；
- 非法参数返回 400；不存在 instrument 返回 200 + `degraded_reasons`（与 `/structural-factors` 一致，不报 404）；
- 在 `app/main.py` 注册 `temporal_features_router`。

### 前端 `StockDetailPage.tsx`（修改）

- 结构状态面板默认隐藏，用户点击「结构状态」开关后显示，`localStorage.showStructuralState` 持久化；
- URL 参数 `?hideStructuralState=1` / `?capture=1` / `?capture=feishu` 任意一个命中即强制隐藏面板且禁用开关按钮（`toggleStructuralState` 回调 early return），忽略 `localStorage`；
- 截图模式（`capture=feishu`）默认只渲染 K 线和基础信息；
- 新增 `.structural-state-toggle` 按钮样式 + `.hide-structural-state` 单列布局样式。

## 风险

- 低：仅新增只读 API + 前端 UI 开关，不修改任何现有 V1.8 字段计算逻辑。
