# CHANGE-20260709-002: 趋势选股批量加入修复 + change_pct 独立列 + 表格视图配置预设 + sticky 表头

## 变更编号

CHANGE-20260709-002

## 任务名称

修复趋势选股页批量加入自选无效；新增当日涨跌幅独立可筛选列；支持保存多种筛选排序配置；表头滚动悬浮完善。

## 需求出处

用户指令：趋势选股页批量加入自选无效（根因 rowKey=instrumentId 但 handleBatchAdd 按 resultId 匹配）；新增 change_pct 独立列；支持命名配置 CRUD；sticky 表头完善选择列 z-index。

## 修改前行为

- `ScreenerPage.handleBatchAdd` 用 `r.resultId` 匹配 selectedKeys，但 rowKey 是 `instrumentId`，selectedKeys 保存 instrumentId → selected 永远为空，批量加入无效；
- 全量 universe 下 skipped/failed 行 `resultId=''`，禁止用 resultId 做批量选择依据；
- change_pct 只在股票列内联显示，股票列 `filterable=false`，无法独立筛选/排序；
- 无命名配置能力，用户每次进入需重新设置筛选/排序；
- sticky 表头已存在但选择列 z-index 未处理，水平滚动时表头首列覆盖选择列。

## 修改后行为

- `handleBatchAdd` 改用 `r.instrumentId` 匹配 + 去重；选中后无可加入股票提示而非静默；
- 新增独立列 `change_pct`（dataType=percent, sortable=true, filterable=true, width≈86），render 用 `fmtChange` + A股涨红跌绿；后端 dsa_selector manifest 已支持 filterable（无需改后端白名单）；
- 新增 `user_table_view_presets` 表 + `/me/table-view-presets` API（GET/POST/PATCH/DELETE），JWT user_id 隔离，权限与趋势选股一致（active subscription + trend_selection feature），每 user+table_id+strategy_key 最多 20 个，config 只保存 keyword/sort/filters/hiddenColumns/pageSize；
- **唯一约束用两个 partial unique index 实现**（解决 PostgreSQL NULL!=NULL 问题）：
  - `strategy_key IS NOT NULL` → unique(user_id, table_id, strategy_key, name)
  - `strategy_key IS NULL` → unique(user_id, table_id, name)
  - 当 strategy_key 为 NULL 时同名也会被拦截（普通 UNIQUE 因 NULL!=NULL 无法拦截）
- **config 深度校验加强**：filters 每项必须是 dict 且含 key/op/value，op 限制白名单（contains/eq/gt/gte/lt/lte/between/empty/not_empty）；hiddenColumns 每项必须是 string；sort.key 必须是非空 string；
- 前端 StrategyDataTable 元信息栏新增"配置"入口：保存当前/另存为/应用/重命名/删除/设为默认；默认配置进入页面自动应用，切换策略/批次时清空选中股票；
- global.scss 补充选择列 sticky z-index：表头 z-index 4、sticky 列 z-index 3、角落单元格（表头+sticky 列）z-index 5；选择列也 sticky left:0，首列通过相邻兄弟选择器偏移选择列宽度（40px）。
