## Runbook 模板

```markdown
# Runbook: <场景标题>

- **触发条件**: <什么情况下使用本手册？>
- **前置条件**: <执行前必须确认的状态/权限>
- **影响范围**: <涉及的系统模块与数据>
- **预计恢复时间**: <分钟级/小时级>

## 症状识别

<如何识别该问题？典型日志/告警/用户反馈是什么？>

## 排查步骤

1. <步骤 1：检查什么？期望结果是什么？>
2. <步骤 2：异常时如何处理？>
3. ...

## 修复操作

### 操作 1: <名称>

```bash
# 命令必须可复制粘贴执行
<command>
```

**预期输出**: <...>
**异常处理**: <...>

## 验证

<如何确认问题已解决？需要检查哪些指标/日志/用户反馈？>

## 防止复发

<长期改进措施？是否需要新增告警/测试/守护规则？>

## 关联

- CHANGE-YYYYMMDD-NNN
- ADR-XXXX（如适用）
- AGENTS §X.Y（如适用）
```

## 规则

- 命令必须可复制粘贴执行，禁止占位符未说明
- 修复操作必须包含预期输出与异常处理
- 任何破坏性操作（删除数据/重启容器/修改 DB）必须标注 **⚠️ 破坏性** 并要求二次确认
- 部署类 runbook 必须包含回滚步骤与回滚验证
- 严禁包含生产环境密码、token、密钥
