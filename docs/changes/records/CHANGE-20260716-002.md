# CHANGE-20260716-002: Atomic Fact Contract V1 个股状态观察（纯函数 + 快照/Context API + 前端面板）

## 变更编号
CHANGE-20260716-002

## 任务名称
实现 Atomic Fact Contract V1 个股状态观察：Canonical Registry 驱动的 14 Core + 10 Auxiliary 纯函数计算；复用 `/stocks/{symbol}/context` 与 `/admin/stocks/{symbol}/debug` 接口（替换旧 state/events 为原子事实结构）；前端 `/market` 右栏与 `/stock/:symbol` 右面板改为 `AtomicFactsPanel`；新快照注入 `summary_payload.atomic_fact_contract_v1`。

## 需求出处
用户指令（2026-07-16）：在分支 `feat/next-phase-20260716`（基线 `8257e27`）实施。真源为 `experiment/hierarchical-scene-state-v3` 的 V4.13 冻结文件（`atomic_fact_contract_v1.json` + `v4_13_atomic_fact_contract_freeze.py`），不凭记忆重写公式。硬约束见 AGENTS 项目规则与用户指令（资源门禁、阈值未确认项仅比值+「分类未启用」、V1 永不进 UI、T3/T6 feature flag 默认关、S3 严格 0.33/0.67、S7/S8 禁止负距离、M3 1e-6 零值容差、近期变化一次查≤10 已发布快照只读计算不写 stock_state_events、as_of 严格 point-in-time）。

## 修改前行为
1. `GET /api/v1/stocks/{symbol}/context` 返回 `StockState`（MACD/Boll 等旧状态向量）+ `StateEventDTO[]` 历史事件 + 数据质量
2. 前端 `EventStatePanel` 渲染旧 state/events（含 MACD 动量、布林位置、事件时间线）
3. 快照 `summary_payload` 不含原子事实
4. 管理员调试页渲染 `data.state`/`data.events` 表格

## 修改后行为

### 一、Canonical Registry（合同真源）
1. 新增 `backend/app/contracts/atomic_fact_contract_v1.json`：从 V4.13 冻结文件复制，校验 core=14/aux=10/rejected=1、ID 唯一、V1 在 rejected 不在 core/aux、ui_sections 顺序 trend→momentum→structure→volume
2. thresholds：s3_position(lower=0.33,upper=0.67,engineering_confirmation_required=false)；t5_slope_ratio/v3_ratio/m3_zero_tolerance 均为 null + engineering_confirmation_required=true（分类未启用）

### 二、纯函数服务（生产实现，不依赖实验脚本运行）
1. 新增 `backend/app/services/atomic_fact_contract_service.py`：导入即校验计数（断言 14/10/1、ID 唯一、V1 在 rejected）；`compute_atomic_facts(structural_payload, temporal_payload)` 只读 `structural_payload.primary.1d.{dsa_segment,volatility_momentum,swing_position}` + `temporal_payload.daily_context`，不查库/联网/复制底层公式/用未来数据
2. 关键规则落地：T2/M2/M3 显示真实原始值（不伪造成 [-1,1]）；T5/V3 阈值未确认 → 仅比值 + 显示「分类未启用」；M3 统一 1e-6 零值容差分类（增加/减少/基本不变）；S3 严格 0.33/0.67（0.33/0.67 归属中间区间）；S7/S8 禁止负距离（d>=0「尚未到达|d|ATR」/d<0「已越过|d|ATR」）；V1 累计成交量比永不进 UI/摘要/可用性；T3/T6 feature flag 默认关（普通用户完全不显示）；缺失事实直接省略
3. `compute_recent_changes(snapshots)`：接收升序≤10 快照列表，逐对对比 category/value，限制 30 条，不写任何表
4. 模块自测 `__main__` 覆盖 S3 边界、S7/S8 正负距离、T2/M2/M3 真实值、T5/V3 分类未启用、V1 缺席、T3/T6 隐藏、分母 14

### 三、Schema
1. 新增 `backend/app/schemas/atomic_fact_contract.py`：`AtomicFactItem`/`AtomicFactAvailability`/`AtomicFactChange`/`AtomicFactsContextResponse`/`AdminAtomicFactDebugItem`/`AdminStockDebugResponse`（camelCase 契约，N815 noqa）

### 四、快照（feature_snapshot_service）
1. `build_summary_payload` 返回 dict 末尾追加 `"atomic_fact_contract_v1": compute_atomic_facts(...)`（仅新快照写入；旧已发布快照受 upsert WHERE 保护不覆盖）

### 五、Context API（复用接口，不新增平行端点）
1. `backend/app/api/stock_context.py`：`_build_stock_context` 返回原子事实结构（contractVersion/asOf/core/auxiliary/availability/recentChanges/dataQuality）；删除旧 `build_stock_state`/`strip_internal_fields_for_user`/`Evidence`/`StateEventDTO`/`_event_to_dto` 等死代码；as_of 无 run → `_empty_atomic_response(no_published_full_run)`（core 全缺失 + 分母 14）；无 snapshot → `_empty_atomic_response(snapshot_missing)`；`_find_recent_published_snapshots` 一次查询≤10 已发布 full scope 快照升序只读计算
2. `get_stock_context` 返回 `AtomicFactsContextResponse`；`get_admin_stock_debug` 返回 `AdminStockDebugResponse`（含 rawDebug + atomicFactsDebug 可追溯 Fact ID/真实路径/raw value/阈值来源/feature flag）

### 六、前端
1. `endpoints.ts`：用 `AtomicFactsContextResponse`/`AdminStockDebugResponse`/`AtomicFactItem` 等替换旧 `StockContextResponse`/`StockState`/`StateEventDTO` 等；`getStockContext` 返回新契约
2. 新增 `AtomicFactsPanel.tsx` + `AtomicFactsPanel.module.scss`：compact（/market 右栏：固定四组顺序 趋势4/动量4/结构5/成交1，分母14，面板内滚动不压缩小K线）/expanded（/stock/:symbol：概览 + 近期变化 + 默认收起更多）；复用 `useStockContext`（同一 query key `['stock-context', symbol, params]`，面板收起 enabled=false → 0 请求）；通俗中文，禁综合分/反转概率/买卖/成熟衰竭/便宜昂贵/止损/安全/放量缩量
3. `MarketRightPanel.tsx`：`MiniKlineCard` + `AtomicFactsPanel compact`（symbol null 不渲染面板）
4. `StockDetailPage.tsx`：右栏改为 `AtomicFactsPanel expanded`，按钮文案「显示状态观察/隐藏状态观察」
5. `AdminStockDebugPage.tsx`：消费原子事实 debug（Fact ID/真实路径/raw value/阈值来源/feature flag）+ 近期变化，保留 raw payload/数据质量
6. 删除旧 `EventStatePanel.tsx`（已被 `AtomicFactsPanel` 替代；`EventStatePanel.module.scss` 仍被管理员页作为调试样式使用，保留）
7. 新增前端契约测试 `src/features/research-context/__tests__/atomic-facts.test.ts`（校验后端 Registry 14/10/1、顺序、ID 唯一、V1 缺席、T3/T6 隐藏、前端类型定义），加入 `package.json` test:contract
