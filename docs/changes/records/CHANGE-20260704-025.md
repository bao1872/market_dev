CHANGE-20260704-025

## 任务名称

Admin Jobs 可观察性补齐 - Worker 心跳 Tab + 只读 admin API

## 需求出处

- ALIGN-012（管理页面 E2E）：AdminJobsPage 缺 Worker 心跳观察区块
- 边界审计 AUDIT-20260704-worker-notification-capture-boundaries.md 第 11 节 PR-D：Admin Jobs 可观察性补齐
- 用户于 2026-07-04 要求：合并 PR #9 后启动第一个功能性小改造，只补可观察性，不做大重构

## 修改前行为

- AdminJobsPage 只有 2 个 Tab（定时任务 / 策略计算），无 Worker 心跳区块；
- 后端 24 个 admin 端点中无任何 worker-heartbeats 暴露；
- `system_overview_service` 仅返回聚合 `worker_health`（healthy/degraded/unknown），不返回 raw 心跳记录；
- `worker_heartbeats` 表只能通过系统概览聚合查看，admin 无法定位具体 Worker 实例状态；
- 前端 `SchedulerJobRunItem` TS 类型缺 `worker_instance_id` / `last_cycle_at`（后端 Pydantic 已有）；
- `JOB_NAME_CN_MAP` 缺 `watchdog` 中文映射。

## 修改后行为

- 新增只读 admin API `GET /admin/worker-heartbeats`，返回 `worker_heartbeats` 表 raw 记录，附加后端计算的 `heartbeat_age_seconds` 和 `health_state`；
- `health_state` 分类阈值与生产一致：fresh（running + age<120s）/ stale（running + 120s≤age<600s）/ stopped（stopped 或 age≥600s）；
- AdminJobsPage 新增第 3 个 Tab "Worker 心跳"，展示 worker_name/instance_id/status/health_state/heartbeat_at/age/build_sha/current_job_id；
- 10 秒轮询，loading/error/empty 状态完整；
- 前端 `SchedulerJobRunItem` TS 类型补齐 `worker_instance_id` / `last_cycle_at`；
- `JOB_NAME_CN_MAP` 追加 `watchdog: '看门狗'`。
