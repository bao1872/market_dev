# CHANGE-20260707-039: Developing Swing Current State

## 变更编号
CHANGE-20260707-039

## 任务名称
Developing Swing 当前状态修复（V1.10）

## 需求出处
用户发现 PR #28 (CHANGE-20260706-038) 实现的 active swing 仍不代表当前状态。000100 案例：close=5.19，active_high=6.26、active_low=4.45、active_pos=0.4088。active_low=4.45 仍是大段起点（confirmed_low），不是从 6.26 回落后的当前 developing low。

## 修改前行为
- swing_position 只有 confirmed pivot + active major leg 两层语义
- active major leg 描述从 confirmed pivot anchor 到当前价格的整段区间
- 但 active major leg 未反映"自高点回落后"或"自低点反弹后"的当前 developing 结构
- 案例 000100：active_low=4.45 是大段起点，不是当前回落后的 low
- 前端 Swing 摘要卡显示 active 字段
- Temporal derived_relation 使用 active swing

## 修改后行为
- swing_position 三层语义：confirmed pivot + active major leg + developing swing
- developing swing 表示"当前正在发生的回落/反弹结构"
- major up leg + active_high_bar_idx < current_idx → developing_low = min(lows[active_high_bar:now])
- major down leg + active_low_bar_idx < current_idx → developing_high = max(highs[active_low_bar:now])
- major up leg + active_high_bar_idx == current_idx → developing = active (仍在创新高)
- major down leg + active_low_bar_idx == current_idx → developing = active (仍在创新低)
- fallback (无 confirmed pivot) → developing = active, dir=None
- 前端 Swing 摘要卡改用 developing 字段（active/confirmed 移到明细 JSON）
- Temporal derived_relation 改用 developing swing，不回退 active/confirmed raw

## 风险
- developing swing 计算依赖 active major leg，fallback 模式（无 confirmed pivot）developing = active，`developing_swing_dir=None`；
- 旧前端如果仍依赖 active swing 字段，V1.10 改为 developing 后会显示 developing 值（V1.10 已同步前端，无遗留）；
- active swing / confirmed raw 字段保留在 API 响应中（向后兼容），但 derived_relation 不再使用；
- 性能：仅单只股票已有 lookback O(n) 扫描，不新增表、worker、全市场循环。
