# CHANGE-20260716-003: AFC V1 双合同分离 + 前端 Compact/Expanded 重构与契约对齐

## 变更编号
CHANGE-20260716-003

## 任务名称
AFC V1 正确性/UI 对齐修复（非新研究实验）：冻结研究合同与产品展示合同分离；前端 `AtomicFactsPanel` 重构为 Compact 四组卡 + Expanded 右侧 overlay Drawer；用户 DTO 不泄露内部字段、缺失事实由后端省略、M3 不声称 1e-6 已确认、近期变化按展示精度比较、persisted-first/fallback 一致；补充后端双合同结构测试与 API 集成测试；更新文档与 maps。

## 需求出处
用户指令（2026-07-16 续）：分支 `feat/next-phase-20260716`，在 CHANGE-20260716-002 基础上继续。对齐 PROMPT.md / `/tmp/panji_afc_fix_review.md` 的 10 项后端问题 + 6 项前端问题。硬约束：冻结合同不得混入产品层语义；普通用户 DOM 不得出现 factId/路径/英文术语；四组配色只用 `variables.scss` token；近期变化非 Core Fact；worker 仍为旧镜像，当前靠 fallback。

## 修改前行为
1. 冻结研究合同内联 `public_key`/`public_label`，与生产层语义耦合；
2. 前端 `AtomicFactsPanel` 用单一 `factRow` 打印所有事实，无 Compact/Expanded 信息层级，详情页把 Expanded 塞进窄 `tv-side-column`；
3. 用户 DTO 含 `factId`/`sourcePath`，内部字段泄露；
4. 缺失事实以 `missing=true` 灰显而非从数组省略；
5. M3 硬编码 1e-6 容差；
6. 近期变化按浮点精确相等产生噪声，返回原始 value 而非 text。

## 修改后行为

### 一、双合同分离
1. 新增 `backend/app/contracts/atomic_fact_presentation_v1.json`：仅按 Fact ID 映射 `publicKey/publicLabel/visualKind/valuePrecision/groupTitle/secondaryLabel`（14 Core + 8 Auxiliary，排除 T3/T6/V1）；不含量/公式/阈值/路径；
2. 冻结合同 `atomic_fact_contract_v1.json` 移除全部 `public_key`/`public_label`，恢复纯 V4.13 原字段；
3. `atomic_fact_contract_service.py` 从 presentation 合同加载 `publicKey/publicLabel`；T3/T6 因不在 presentation 中用字面量 `"trend_efficiency"`/`"efficiency_delta"` 输出；
4. 测试 `test_atomic_fact_contracts.py` 校验：frozen 无产品字段、presentation 恰好 14+8、Fact ID 一一对应、T3/T6/V1 不进 presentation、V1 无映射。

### 二、后端 API 对齐（前序已完成，本轮纳入验证）
1. `PublicAtomicFactItem` 无 `factId`/`sourcePath`；`AdminAtomicFactDebugItem` 保留追溯字段；
2. 缺失事实由 `compute_atomic_facts` 从 Core 数组省略，`availability.coreMissing` 用 publicKey，分母固定 14；
3. M3 按 `raw>0→增加/raw<0→减少/raw==0→基本不变`，`thresholdEnabled=false`，不声称 1e-6；
4. `recentChanges` 按各事实展示精度比较（T2/M2 4 位、M3 6 位、S3/S7/S8/T5/V3 2 位），返回 `fromText/toText/deltaText`；
5. Context API persisted-first：优先读取 `summary_payload.atomic_fact_contract_v1`（校验后直接返回），否则同纯函数 fallback（不回写）；
6. `as_of` 近期变化 SQL 直接 `trade_date <= as_of` 过滤再 `DESC LIMIT`；GET 零写入；
7. 测试 `test_stock_context_atomic_facts.py`（14 项）：覆盖字段不泄露、缺失省略、M3 未确认、M5 双 true 异常、S1 未知枚举缺失、S3 越界省略、S7/S8 动态 sourcePath、summary 优先+fallback、as_of SQL 顺序、精度过滤、admin 完整追溯、普通用户 403。

### 三、前端重构
1. `AtomicFactsPanel.tsx` 重写为组件树（`Header/CoreFactGroup/FactMetricRow/RelationBadge/PositionRail/BoundaryRow/AuxiliaryAccordion/RecentChangesStrip`）：
   - Compact（`/market` 右栏）：标题「个股状态观察」、右侧「日线 · V4.13」、日期与 `N/14`；四张组卡（趋势 info / 动量 brand / 结构 purple / 成交 warning），每项一行（标签左/主值右/弱说明下）；S3 用 0–1 轨道（0.33/0.67）、T5/V3 比值+「分类未启用」、S7/S8「尚未到达/已越过」；面板内滚动不压小 K 线；
   - Expanded：四组响应式网格（宽屏 4 列/普通 2 列/小屏 1 列）+ 全宽近期变化 + 默认收起 Auxiliary（展开 8 项，T3/T6/V1 不出现）；
2. 新增 `AtomicFactsDrawer.tsx`：右侧 overlay 抽屉，宽度 `min(1080px, calc(100vw - 48px))`，固定 overlay 不压缩 K 线；Escape/遮罩/关闭按钮可关，`role="dialog"` `aria-modal`；
3. `StockDetailPage.tsx`：详情页「显示状态观察」打开 Drawer（移除内联窄 aside，StockResearchWorkspace 不再渲染 inline 右栏）；
4. `AdminStockDebugPage.tsx`：近期变化列改用 `publicKey/fromText/toText/asOf`（旧 `factId/fromCategory` 字段已不存在）；
5. `AtomicFactsPanel.module.scss` 重写：四组维度色（仅 `variables.scss` token）、位置轨道、徽章、近期变化、Drawer 样式，无硬编码十六进制；
6. 前端契约测试扩展：presentation 14+8 排除 T3/T6/V1、frozen 无产品字段、面板源码无内部术语（整词匹配）。
