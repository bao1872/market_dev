# 事实源清单（Source of Truth Map）

> 适用范围：后端、前端、文档、CI、AI Agent 与人类开发者。
> 关联文档：`AGENTS.md`、`docs/architecture/ADR-001-postgresql-only.md`、`docs/测试规范.md`。
> 原则：**一个业务概念只能有一个事实源**。

---

## 规则优先级

当事实源定义发生冲突时，按以下优先级裁决：

1. ADR 架构决策（`docs/architecture/ADR-*.md`）
2. AGENTS.md 项目硬规则
3. docs 项目规范（含本文档）
4. 公共基础设施（`backend/tests/conftest.py`、`backend/app/db.py` 等）
5. 模块文档字符串
6. 文件头注释

**文件头注释、局部 docstring 不得推翻 ADR、AGENTS.md 与本文档。**

## roles（用户角色）

| 项 | 事实源文件 | 消费方式 | 禁止事项 |
|---|---|---|---|
| 角色名枚举 | `backend/alembic/versions/001_users_table.py`（或最早创建 roles 表的迁移） | 通过 Alembic 迁移写入 `roles.name`，配合 `Role.name` 唯一约束 | 禁止在代码/测试中新增未迁移的角色名 |
| 普通用户角色 | `member` | 迁移 seed、测试 factory、API 校验统一使用 `"member"` | 禁止使用 `"user"` 作为普通用户角色名；禁止 `Role(name="user")` / `_ensure_role(db, "user")` |
| admin 角色 | `backend/app/scripts/bootstrap_admin.py` | 首位管理员 bootstrap 时创建 `admin` 角色并绑定 | 禁止为 admin 创建 subscription 或绑定套餐 |
| 角色检查依赖 | `backend/app/core/deps.py` | 通过 `require_roles(...)` 校验请求上下文角色 | 禁止绕过 deps 直接根据 `user.roles` 字符串硬编码判断 |

## plans（套餐）

| 项 | 事实源文件 | 消费方式 | 禁止事项 |
|---|---|---|---|
| 套餐定义 | `backend/alembic/versions/048_plans_table.py` | 迁移中初始化 `observe_20`、`research_50` 等套餐的完整字段（含 features、monitor_limit、price 等） | 禁止在测试 seed SQL、常量文件、API 中复制套餐 features / monitor_limit |
| 套餐 ORM 模型 | `backend/app/models/plan.py` | 通过 SQLAlchemy ORM 读写 `plans` 表 | 禁止绕过模型直接手写 SQL 操作套餐 |
| 套餐查询服务 | `backend/app/services/plan_service.py` | 统一使用 `get_plan`、`get_monitor_limit`、`list_all_plans` 查询套餐 | 禁止在 service 外部直接使用 `plan_code` 字面量推导限额 |
| 套餐代码字符串常量 | `backend/app/constants/plan_codes.py`（已精简） | 仅保留 `DEFAULT_PLAN_CODE` 作为默认套餐代码引用 | 禁止保留历史管理员套餐代码常量；禁止新增其他硬编码套餐代码常量 |
| admin 套餐状态 | `AGENTS.md` 规则 8 | admin 的 `plan_code=None`，无 subscription | 禁止返回 `research_50` 或任何套餐代码给 admin |

## schema（数据库结构）

| 项 | 事实源文件 | 消费方式 | 禁止事项 |
|---|---|---|---|
| 唯一 DDL 真源 | `backend/alembic/versions/` 下所有迁移文件 | 通过 `alembic upgrade head` 应用到测试/生产库 | 禁止测试手写 `CREATE TABLE`、手写 ALTER、注册 SQLite JSONB compiler |
| ORM 映射 | `backend/app/models/*.py` | 业务代码通过 SQLAlchemy ORM 操作数据 | 禁止 ORM 模型与最新迁移不一致 |
| 一致性校验 | `tools/check_architecture.py`（已创建，standalone，无 DB 连接） | 扫描 backend/frontend/src/docs/tools 无 SQLite/aiosqlite/手写 schema/自定义 db_session/角色与套餐违规等 | 禁止新增与迁移冲突的模型定义 |
| 数据库连接配置 | `backend/app/config.py` | `get_settings().database_url` 经硬校验后供 `backend/app/db.py` 创建 engine | 禁止直接调用 `create_async_engine` 绕过 config |

## worker（服务名与启动方式）

| 项 | 事实源文件 | 消费方式 | 禁止事项 |
|---|---|---|---|
| 服务名 | `docker-compose.prod.yml` | 所有 worker 服务名以该文件为准 | 禁止在文档、脚本、运维手册中手写不存在或过期的 worker 名称 |
| 启动命令 | `docker-compose.prod.yml` 各 service 的 `command` | 统一为 `python -m app.worker` 或对应 `uvicorn` 命令 | 禁止新增 worker 而不更新 `docker-compose.prod.yml` |
| 服务清单 | `docker-compose.prod.yml` | `worker-bars-scheduler`、`worker-strategy-scheduler`、`worker-calendar`、`worker-monitor`、`worker-strategy-batch`、`worker-outbox`、`worker-delivery`、`worker-after-close`、`worker-capture` | 禁止文档中自行命名或缩写 |

## test database（测试数据库）

| 项 | 事实源文件 | 消费方式 | 禁止事项 |
|---|---|---|---|
| 测试库配置 | `backend/tests/conftest.py` | 读取 `APP_ENV=test` 与 `TEST_DATABASE_URL`，校验库名含 `_test`，转换为 `postgresql+asyncpg://` | 禁止在测试模块内自行解析环境变量并创建 engine |
| Alembic 升级 | `backend/tests/conftest.py` 中 `init_test_db` fixture | session 范围自动调用 `alembic upgrade head` | 禁止测试模块本地执行 DDL 或手写 seed schema |
| 测试 session | `backend/tests/conftest.py` 中 `db_session` fixture | 提供事务隔离的 AsyncSession，测试后回滚 | 禁止模块级 `db_session` fixture 覆盖 |
| 测试工厂 | `backend/tests/conftest.py` | 提供 `user_factory`、`role_factory`、`subscription_factory`、`invite_code_factory`、`instrument_factory` 等 | 禁止在测试模块内复制相似构造逻辑 |
| 测试目标库 | `bz_stock_test` | `TEST_DATABASE_URL` 必须指向该库 | 禁止指向非 `_test` 库或 SQLite |

## docs（文档生成规则）

| 项 | 事实源文件 | 消费方式 | 禁止事项 |
|---|---|---|---|
| 操作手册 | `docs/操作手册.md` | 必须由 `tools/update_docs.py` 从事实源生成 | 禁止手工编辑 |
| 数据结构 | `docs/数据结构.md` | 必须由 `tools/update_docs.py` 从事实源生成 | 禁止手工编辑 |
| 安全规范 | `docs/安全规范.md` | 当前为手工维护；Last verified commit 由 `tools/check_architecture.py` 门禁检查，后续纳入生成脚本统一 | 禁止同时写互相矛盾的账号权限/测试库部署方式 |
| 生成脚本 | `tools/update_docs.py` | 尚未实现，当前为可选；事实源/脚本参数/输出/表结构/语义变更后建议运行并提交 diff | 禁止修改事实源后不运行生成脚本 |
| 规则优先级 | `AGENTS.md` + `docs/architecture/ADR-001-postgresql-only.md` | 所有文档、注释、代码必须服从该优先级 | 禁止文件头注释推翻 ADR / AGENTS.md |
