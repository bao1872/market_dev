## 任务名称
DetailSourceContextV2 — 详情页来源列表同源同序合同 V2（行情来源失真 + 自选排序跳变根治）

## 需求出处
- V3 已知问题彻底收口：1) 行情筛选进入详情后来源变成自选；2) 自选列表进入详情后排序跳变
- `ref/PROMPT.md`：DetailSourceContextV2 必须实现 `origin + sourceRunId + canonicalQuery + returnTo + stableContextId`
- AGENTS.md §七.4：有效会员添加自选后自动进入盘中监控；到期用户保留历史但不能读取/修改/监控

## 修改前行为
1. **行情来源失真**：V1 详情页 `useStockDetailActions` 用 fresh `usePublishedRuns` 推导 activeRunId，当 DSA published run 列表变化或请求时序不同时，详情左栏可能取到不同 run 或回退到自选列表，导致"行情筛选进入详情后来源变成自选"
2. **自选排序跳变**：V1 详情页不接收 canonicalQuery（sort/filters/page），用默认排序请求，导致"自选列表进入详情后排序跳变"
3. **stableContextId 合同违反**：V2 初版 `computeStableContextIdV2` 接收 `returnTo` 参数，而 `returnTo` 含 `selected=入口symbol`，切股后 returnTo.selected 变化导致 stableContextId 变化，违反"切股不变"合同

## 修改后行为

### V2 来源同源同序合同
- `buildStockDetailUrl` 编码 `originScope` + `sourceRunId`（入口快照 activeRunId）+ `cq`（canonicalQuery = `JSON.stringify(resultParams)`，含 universe/sort_by/sort_desc/keyword/industry/concept/metric_filters/page/page_size）+ `returnTo`
- `MarketWorkspacePage.handleNavigateToStock` 从当前内存 `marketListCtx` 构建 returnTo（`buildMarketReturnToUrl`），透传 `sourceRunId` + `canonicalQuery`
- `useStockDetailActions` V2 用固定 `sourceRunId` + `canonicalQuery` 调 `useStrategyRunResults`（**禁止 fresh `usePublishedRuns`**），保证列表页与详情左栏请求同一 run + 同一 query → 有序 symbol 数组完全一致
- `useWatchlistMonitorStatus` 仅用于 `inWatchlist` 状态，**禁止充当来源列表数据源**
- `resolveDetailSourceContextV2` 返回 `origin + sourceRunId + canonicalQuery + canonicalQueryRaw + returnTo + stableContextId + sourceContextInvalid + invalidReason`

### stableContextId 不变性（合同纠正）
- `computeStableContextIdV2(origin, sourceRunId, canonicalQueryRaw)` 只由这三者计算
- **不含 selectedSymbol**、**不含 returnTo**（returnTo 含 `selected=入口symbol`，纳入会间接破坏切股不变性）
- 切股/刷新后 stableContextId 不变

### 缺 sourceRunId 不静默回退
- `origin=market` 但缺 `sourceRunId` → `sourceContextInvalid=true`，显示 invalid 占位，禁止静默回退自选
- `origin=direct` → 无来源列表（不伪造行情来源）
