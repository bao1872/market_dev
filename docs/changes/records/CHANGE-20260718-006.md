## 2. 修改前理解

- **当前产品行为**：
  - 12 个算法族（node_cluster/dsa/smc/bollinger/macd/sqzmom/breakout/participation/temporal_features/structural_features/primary_secondary_relation/snapshot_derived_features）散落在 `indicator_service`/`feature_snapshot_service`/`monitor_batch_service`，无统一注册表
  - 四条调用链（详情/盘后/盘中/Capture）各自调用算法 kernel，无统一调度入口
  - 飞书状态机中，要求图片的投递 `card_status=success` 但 `image_status!=success` 时整体标记 `partial_failed`，掩盖了图片失败
  - 前端周期切换时，bars 与 indicators 是独立 React Query，可能短暂不一致（上一周期残留指标 + 新周期 bars）
  - 图表纵轴候选包含所有 Node/trailing，远端 Node 会撑爆 y-range
- **发现的缺陷**：
  - 缺少算法合同注册表和统一调度服务，无法保证"相同输入相同输出"
  - 飞书图片失败被 `partial_failed` 掩盖，无法准确反馈图片投递状态
  - 前端周期切换非原子，可能渲染不一致的 bars+indicators 帧
  - 远端 Node/trailing 撑爆纵轴，影响 K 线可读性
  - 文档 `docs/current/08` 仅覆盖 Node Cluster，未覆盖全部 12 个算法族
  - 文档 `docs/maps/indicator-computation-map.md` 仅三链，未覆盖 Capture/飞书链
- **已阅读的当前设计文档**：`docs/current/01-08`、`docs/maps/*`、`AGENTS.md`、`backend/app/services/indicator_service.py`、`backend/app/services/feature_snapshot_service.py`、`backend/app/services/stock_detail_feishu_service.py`、`backend/app/api/stock_detail_feishu.py`、`frontend/src/components/StrategyChart.tsx`、`frontend/src/features/stock-research/StockResearchWorkspace.tsx`、`frontend/src/features/stock-research/useStockResearchData.ts`、`frontend/src/components/smcRendering.ts`、`frontend/src/utils/dsaOverlayPolicy.ts`
