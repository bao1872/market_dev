## 2. 修改前理解

- `feishu_webhook_adapter.py` 与 `feishu_platform_app` 并行存在，运行时仍有 Webhook 遗留（runtime/schema/ORM/admin/tests/docs）。
- 截图服务访问普通 `/stock/:symbol?capture=feishu` 路由，与普通页面混用，无专用 Capture API。
- `create_capture_token` 只校验 `type=capture + event_id`，无 scope/instrument_id 隔离。
- Capture Token 可访问普通 API（`get_current_user` 未拒绝 type=capture）。
- 截图失败时仍返回 `status="pending"`，前端无法识别失败。
- `capture_resp.raise_for_status()` 丢弃 502 响应体详情。

## 3. 修改目标

- 永久删除 `feishu_webhook_adapter.py`，统一 `feishu_platform_app`。
- 新增专用 Capture 链路：`/capture/stock/:symbol` 前端路由 + `/api/v1/capture/stocks/{instrument_id}/snapshot` 后端 API。
- 增强 `create_capture_token`：新增 `scope=stock_detail_capture`、`instrument_id`、`user_id` 参数。
- Capture Token 隔离：普通 API 拒绝 `type=capture`；Capture API 拒绝 `type=access`。
- 状态机统一：截图失败返回 `status="partial_failed"` + `failed_step` + `error_code` + `error_message`。
- 解析 502 响应体详情（不丢弃 worker 返回的错误信息）。
- migration 055 添加 CHECK 约束永久禁止 `feishu_webhook`。
