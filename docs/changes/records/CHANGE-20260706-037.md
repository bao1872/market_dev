# CHANGE-20260706-037: 前端不覆盖后端 1d partial bar

## 变更编号
CHANGE-20260706-037

## 任务名称
前端不覆盖后端 1d partial bar

## 需求出处
PR #26 后端已把 `/bars?timeframe=1d&include_realtime=true` 固化为个股详情 K线实时唯一契约，交易时段会返回 `is_partial=true` 的 partial daily bar。但前端 `StockDetailPage` 仍无条件调用 `mergeRealtimeQuoteIntoBars(baseBars, quoteQuery.data, timeframe)`，当 quote 为 pytdx 实时价时，会覆盖后端 partial bar 的 close/high/low，导致 K线显示事实源变成 `/quote` 而非 `/bars`，违反刚写入 AGENTS.md 的 K线实时契约。

## 修改前行为
- `StockDetailPage.tsx` 中 `displayBars = mergeRealtimeQuoteIntoBars(baseBars, quoteQuery.data, timeframe)` 无条件执行；
- 交易时段后端返回 `is_partial=true` 的 partial bar 后，若 quote 为 pytdx 实时价，`mergeRealtimeQuoteIntoBars` 会用 quote.current_price 更新最后一根 bar 的 close/high/low；
- 顶部行情卡片与 K线实时状态的事实源混同，用户无法区分“quote 实时”与“K线 partial”。

## 修改后行为
- `mergeRealtimeQuoteIntoBars` 新增第 4 个参数 `backendIsPartial: boolean = false`；当 `backendIsPartial === true` 时直接返回原 bars，不做任何 quote 合并；
- `StockDetailPage.tsx` 计算 `backendIsPartial = barsQuery.data?.is_partial === true`，并传入 `mergeRealtimeQuoteIntoBars`；
- 后端 `/bars` 已返回 partial bar（`is_partial=true`）时，`displayBars` 等于 `baseBars`，quote 仅用于顶部行情卡片；
- 后端未返回 partial bar（`is_partial=false`）时，仍允许 `mergeRealtimeQuoteIntoBars` 作为兜底视觉增强；
- 指标计算继续沿用 `baseBars`，不受 displayBars 影响。
