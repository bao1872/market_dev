## 修改前行为
### 后端
- M5 `_squeeze_state` 使用 `if on is None and off is None`（双缺失才缺失），单侧缺失（如 `on=None, off=False`）会进入 Core 伪装 NORMAL。
- `compute_recent_changes` 统一 `round(...,4)` 量化所有 Fact（忽略 per-fact `valuePrecision`），事实消失时 dimension 默认 `trend`（S3 消失→错误标 trend）。
- `_is_valid_stored_afc` 手写校验（四版本 + 四组 + publicKey），未用 Pydantic schema 严格校验 availability 一致性 / 维度归属 / 重复 key / 未知 key / T3/T6/V1 禁止。
- `as_of` 语义为精确匹配 `trade_date == as_of`，周末/无批次日期返回空态（不返回之前最近发布状态）。
- Legacy snapshot 存在但 `source_run_id` 缺失/歧义时，`reasonCode` 被清除（不进入 `degradedReasons`）。
- 公共响应无 `meta` 字段；前端硬编码 `AFC_RESEARCH_VERSION = 'V4.13'`。
- secondaryText 散落硬编码常量（`ATR / 根日K`、`个交易日`、`分类未启用`），未统一从 presentation `secondaryLabel` 映射。

### 前端
- `.factRow` `grid-template-areas: "label value" / "secondary secondary"`（secondary 跨第二行整行，非右列右对齐）。
- `PositionRail` 复用 `.factRow` 布局，轨道在右列（非横跨整组宽度）；刻度 低位/高位 在轨道上方，0.33/0.67 在轨道下方，caption 在轨道下方（可能与刻度重叠）。
- `RecentChangesStrip` 不渲染 `c.deltaText`（仅 label/from→to/asOf）。
- `Drawer` 正向 Tab 仅检查 `active === last`，不处理焦点离开 drawer 的情况（`!drawer.contains(active)`）。
- `Header` 硬编码 `AFC_RESEARCH_VERSION = 'V4.13'`，不从 API `meta` 读取。

## 修改后行为
### 后端
1. **M5 单侧缺失**：`_squeeze_state` 改为 `if on is None or off is None: return None`（任一缺失即缺失）。新增四个单侧缺失组合测试：`(None,False)`/`(False,None)`/`(None,True)`/`(True,None)` 均不进入 Core、在 coreMissing、无 m5_inconsistent warning。
2. **RecentChanges per-fact 精度**：`_quantize_fact_value(fact_id, v)` 按 presentation `valuePrecision` 量化（禁止统一 `round(...,4)`）；`FACT_DIMENSION_BY_ID` 从冻结合同导出 `fact_id → dimension` 映射，事实消失时仍返回正确维度（禁止默认 trend）；`_combine_text` 组合短值和 category（避免丢失 M3 双文本状态）。
3. **PersistedAtomicFactsPayload Pydantic schema**：`extra="forbid"` + `model_validator` 严格校验：四版本完全匹配、core 键恰好 `trend/momentum/structure/volume`、每一项通过 `PublicAtomicFactItem`、publicKey 属于正确维度且无重复/未知、T3/T6/V1 不存在、availability 与实际数组及固定分母 14 一致、不含 debug。不兼容必须 fallback，不得 500。
4. **as_of 截止语义**：`_find_run_by_trade_date` 改为 `trade_date <= as_of` + `desc(trade_date)` + `desc(published_at)` + `desc(finished_at)` 取最新 1 条；周末和无批次日期返回之前最近发布状态。
5. **Legacy degradedReasons**：legacy snapshot 存在但 `source_run_id` 缺失/歧义时，`reasonCode` 加入 `dataQuality.degradedReasons`（不清除原因）；无 snapshot 才使用 `reasonCode` 作为唯一空态原因。
6. **公共响应 meta**：`AtomicFactsMeta` schema（`payloadVersion`/`researchFreezeVersion`/`presentationVersion`），`AtomicFactsContextResponse` 新增 `meta` 字段；前端禁止硬编码 V4.13。
7. **presentation secondaryLabel 真源**：`_secondary_text_for(fact_id, value_present, threshold_enabled)` 统一从 presentation `secondaryLabel` 映射生成 secondaryText；分类未启用使用 `unclassifiedLabel`（presentation 合同新增顶层字段）；移除服务中散落的 `ATR / 根日K`/`个交易日`/`分类未启用` 硬编码常量。

### 前端
1. **factRow secondary 右列**：`.factRow` `grid-template-areas: "label value" / ". secondary"`（secondary 仅占右列）；`.factSecondary { text-align: right }`。
2. **PositionRow 独立布局**：`.positionRow` `grid-template-areas: "label caption" / "track track"`（第一行 label 左 / caption 右；第二行轨道横跨整组宽度）；`.railScale` `space-between` 均匀分布四刻度 低位/0.33/0.67/高位；`.railTrackWrap` `min-height: 28px` 预留刻度高度禁止与 caption 重叠。
3. **RecentChanges deltaText**：`RecentChangesStrip` 渲染 `c.deltaText`（`changeDelta` class）；`changeItem` grid 改为 4 列 `minmax(0,1fr) auto auto auto`；禁止显示 publicKey。
4. **Drawer Tab 焦点双向**：正向 Tab 分支也检查 `!drawer.contains(active)`（`if (active === last || !drawer.contains(active))`），与 Shift+Tab 分支对称。
5. **Header meta 读取**：移除 `AFC_RESEARCH_VERSION` 常量；`Header` 从 `data.meta.researchFreezeVersion` 读取研究版本。
6. **endpoints.ts**：新增 `AtomicFactsMeta` interface；`AtomicFactsContextResponse` 新增 `meta: AtomicFactsMeta` 字段。
