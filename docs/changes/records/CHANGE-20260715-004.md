# CHANGE-20260715-004

## 变更编号
CHANGE-20260715-004

## 任务名称
Bug 1 修复：行情筛选后进入详情左栏不显示 loading 占位 + Pine 真源文件入 Git 跟踪

## 需求出处
用户要求修复详情页 Bug 1：行情筛选后进入详情，左栏不能变自选；来源列表加载中应显示 loading 占位，避免空白后突然出现列表；并将用户原创 Pine 源码纳入 Git 跟踪。

## 修改前行为
1. `useStockDetailActions` 未暴露 `sourceListLoading` 字段
2. `StockDetailPage` 左栏渲染条件为 `sourceStocks.length > 0`，DSA results 加载中（`sourceStocks=[]`）时不显示任何 loading 占位
3. 用户从 `/market` 进入 `/stock/:symbol` 后，左栏空白一段时间，DSA 数据返回后突然出现列表
4. `ref/smc_user_source.pine`（用户原创 Pine 源码，SHA256 0bd3d2ad，843 行）未被 Git 跟踪（`.gitignore` 排除 `ref/`）

## 修改后行为
1. `useStockDetailActions` 新增 `sourceListLoading: boolean` 字段：
   - `hasMarketContext=true` 时：`publishedRunsQuery.isLoading || !activeRunId || sourceResultsQuery.isLoading`
   - `hasMarketContext=false` 时：`monitorStatusQuery.isLoading`
2. `StockDetailPage` 新增 loading 占位渲染分支：
   - `sourceListLoading=true` → 渲染 `<aside data-testid="detail-source-list-loading">` + `<div class="tv-source-list-placeholder">加载中…</div>`
   - `!sourceListLoading && sourceStocks.length > 0` → 渲染正常列表
3. CSS 新增 `.tv-source-list-placeholder { padding: 16px 10px; font-size: 12px; color: #778287; text-align: center; }`
4. `ref/smc_user_source.pine` 使用 `git add -f` 强制跟踪，仅跟踪该单文件（`.gitignore` 仍排除 `ref/` 其他文件）
