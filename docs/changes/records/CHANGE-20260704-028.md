# CHANGE-20260704-028

## 变更编号

CHANGE-20260704-028

## 任务名称

趋势选股页全量 Universe 展示

## 需求出处

- PR #14 部署后前端仍显示 804 命中/4391 失败。
- 根因：`/strategy-runs/{run_id}/results` 以 `strategy_results` 为主表（仅 succeeded 行），`strategy_run_items` 中的 skipped/failed 行不参与查询。
- 前端 `activeRunId` 为旧 published run `b4cd9d4d`（trade_date=2026-07-02，succeeded=804, failed=4391）。
- 新 run `f0c15e1c`（trade_date=2026-07-03，succeeded=5106, failed=0, skipped=187）已 publish，但前端无法展示全量 universe。

## 修改前行为

- 后端 `/strategy-runs/{run_id}/results` 主表为 `strategy_results`，仅返回 succeeded 行。
- `source_total = count(strategy_results where run_id=...)`，不是 `run.total_instruments`。
- skipped/failed 行不参与查询，前端无法展示全量股票。
- 前端 `StrategyResult.id`/`payload` 为必填字段，不支持 null。
- 前端 `ScreenerPage` 行 key 使用 `resultId`，skipped/failed 行无 result_id 无法选中。
- 前端标签 "命中" 不准确（`filtered_total` 是当前筛选条件下的数量，不是命中数）。

## 修改后行为

- 后端 `/strategy-runs/{run_id}/results` 以 `strategy_run_items` 为主表 LEFT JOIN `strategy_results` + `instruments`。
- 返回全量 universe（含 succeeded/skipped/failed），skipped/failed 行 `id`/`payload` 为 null。
- 新增 `item_status`/`reason_code`/`error_message` 字段。
- `source_total = run.total_instruments`（fallback 到 `count_run_items_by_run`）。
- `filtered_total` = 全量 items 数（无筛选时等于 `source_total`）。
- `metric_filters` 只筛有对应指标的行（skipped/failed 行 `result_id=None` 自动不命中）。
- 前端 `StrategyResult.id`/`payload` 改为可空（`string | null`、`Record<string, unknown> | null`）。
- 前端 adapter 支持 null id/payload 降级（`resultId=''`、`payload={}`）。
- 前端 `ScreenerPage` 行 key 改用 `instrumentId`（skipped/failed 行也能选中加入自选）。
- 前端标签 "命中" 改名 "筛选结果"。
- AGENTS.md 新增第 12 条 Docker 镜像保护规则。
