### 3.7 交易日历排查（Mootdx 口径）

**事实源**：`backend/app/services/mootdx_calendar_provider.py` + `backend/app/services/calendar_seed.py` + `backend/app/services/calendar_service.py` + `backend/app/models/calendar.py`

**症状**：前端显示"非交易日"但实际是交易日（如周五显示非交易日），或盘后编排任务因"非交易日"跳过所有步骤；`/market/status` 显示"交易日历待确认"。

**字段含义**：

| 字段 | 说明 |
|------|------|
| `status` | `OPEN`=交易日 / `CLOSED`=非交易日 / `UNKNOWN`=待确认 |
| `source` | `MOOTDX_HOLIDAY` / `MOOTDX_HISTORICAL` / `MANUAL_OVERRIDE` |
| `is_trading_day` | 由 `status` 派生：`OPEN`→true，`CLOSED`→false，`UNKNOWN`→NULL |
| `verified_at` | Mootdx 校验时间戳 |
| `validation_error` | 校验冲突/失败原因 |

**排查**：
```bash
# 查询指定日期的交易日标记

SELECT trade_date, is_trading_day, status, source, verified_at, validation_error
FROM trading_calendar
WHERE trade_date BETWEEN CURRENT_DATE - 7 AND CURRENT_DATE + 7
ORDER BY trade_date;"

# 查询市场状态诊断字段
```


**修复**：

- Tushare 已彻底删除，唯一外部数据源为 Mootdx `mootdx.utils.holiday`。
- 历史日期使用 `holidays()` 返回的交易日集合核验；今天/未来使用 `holiday(YYYY-MM-DD)` 单日期判断。
- Pytdx 行情是否存在 K 线不再参与交易日判断。
- `MANUAL_OVERRIDE` 不会被自动同步覆盖，除非带 `--force`。

### 3.8 覆盖率卡在 75% 无法触发 DSA

**症状**：盘后任务 `refreshing_daily` 完成后，覆盖率检查始终 < 90%，无法触发 DSA 选股计算。

**根因**：覆盖率分母包含指数/基金/ETF（约 2020 个），这些标的 pytdx 不写入 `bars_daily`，导致覆盖率永远卡在 75.5%。

**排查**：
```bash

WHERE status='active' AND (
  (market='SH' AND symbol LIKE '6%')
```

## 4. 飞书投递失败

### 4.1 错误码分类

事实源：`backend/app/services/feishu_webhook_adapter.py`

| HTTP 状态码 | 错误类型 | 处理建议 |
|------------|---------|---------|
| 429 | 限流 | 自动重试（指数退避） |
| 500/502/503/504 | 服务端错误 | 自动重试 |
| 400/401/403/404 | 配置错误 | 标记渠道 invalid，进入 dead |
| 其他 | 未知 | 重试到上限后 dead |

### 4.2 排查步骤

```
1. 查询 message_deliveries 表找 failed/dead 记录
2. 查看 last_error_code 与 provider_response 字段
3. 检查渠道配置（app_secret / sign_secret 是否正确）
4. 手动验证渠道：POST /notification-channels/{id}/verify
5. 手动测试投递：POST /notification-channels/{id}/test
6. 重试投递（admin）：POST /admin/message-deliveries/{id}/retry
```

### 4.3 常见问题

**问题 1：飞书 Webhook 签名错误**
- 原因：`sign_secret` 配置错误或时间漂移
- 解决：核对 `sign_secret`，确保服务器时间准确（NTP 同步）

**问题 2：飞书平台应用 token 获取失败**
- 原因：`app_id` / `app_secret` 错误或应用被禁用
- 解决：核对应用凭证，检查飞书开放平台应用状态

**问题 3：消息未投递**
- 原因：静默时段（22:00-08:00）仅站内可见
- 解决：检查 `next_attempt_at` 字段，等待静默结束后投递
