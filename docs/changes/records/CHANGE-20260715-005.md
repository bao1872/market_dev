# CHANGE-20260715-005

## 变更编号
CHANGE-20260715-005

## 任务名称
详情左栏来源状态拆分 + 表格 sticky 列和工具栏对齐根治

## 需求出处
用户要求：1) 详情左栏来源列表拆分 loading/error/empty/invalid 四态，尊重显式 source 参数；2) normalizeInternalReturnTo 上限从 500 提升到 4096；3) 表格 sticky 列只允许 stock 列，结构改为 table-shell > meta-bar + table-scroll > table + pager，工具栏/分页器移出横向滚动容器。

## 修改前行为
1. `useStockDetailActions` 只有 `sourceListLoading` 单一布尔状态，无法区分 error/empty/invalid
2. `source` 参数在某些路径下被 returnTo 推断覆盖，不尊重显式传入的 source
3. `normalizeInternalReturnTo` 上限 500，复杂筛选 URL（多 filters + industry + concept + keyword + sort）编码后可能超过 500
4. 表格使用 `.table-wrap` 单层结构，meta-bar 和 pager 在横向滚动容器内，需 `position:sticky;left:0;width:100%` 补丁保持可见
5. sticky 列判断分散在 header 和 body 各自实现，且允许 change_pct 列为 sticky（死 CSS `.sticky-col-change-pct`）
6. `AdminAfterClosePipelinePage` 仍使用旧 `.table-wrap` 结构

## 修改后行为
1. `useStockDetailActions` 拆分为四态：
   - `sourceListLoading: boolean` — 加载中（publishedRunsQuery.isLoading || !activeRunId || sourceResultsQuery.isLoading）
   - `sourceListError: boolean` — 错误（publishedRunsQuery.isError || sourceResultsQuery.isError）
   - `sourceListEmpty: boolean` — 空状态（非 loading 非 error 但 sourceStocks 为空）
   - `sourceContextInvalid: boolean` — 上下文失效（source=selection 但 returnTo 解析失败或非 /market 前缀）
2. `source` 参数优先级：显式 source > returnTo 推断；source=selection → sourceListKind=market（即使 returnTo 无效也不回退 watchlist）
3. `normalizeInternalReturnTo` 上限 500 → 4096
4. 表格结构改为 `table-shell > meta-bar + search-bar + table-scroll > table + pager`：
   - 只有 `table-scroll` 设置 `overflow-x: auto`
   - meta-bar/search-bar/pager 移出横向滚动容器，右边界自然等于 table-scroll 右边界
   - 删除 `position:sticky;left:0;width:100%` 补丁（不再需要）
5. `isStickyColumn(col)` 统一判断函数：只允许 `col.key === 'stock'` 为 sticky 列；header 和 body 共用同一判断
6. 删除死 CSS `.sticky-col-change-pct`
7. `AdminAfterClosePipelinePage` 迁移到 `table-shell` + `table-scroll` 结构
