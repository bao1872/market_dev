## 修改后行为

### 一、SMC 指标模块（纯函数，无外部依赖）
1. 新增 `backend/app/strategy_assets/algorithms/features/smc_indicator.py`
2. 仅依赖 stdlib（logging, dataclasses, typing），不引入 numpy/pandas/plotly/pytdx
3. 默认参数来自 ref/smc.py build_parser() 默认值：
   - swings_length=50, equal_length=3, equal_threshold=0.1
   - internal_filter_confluence=False, internal_ob_size=5, swing_ob_size=5
   - order_block_filter="Atr", order_block_mitigation="High/Low"
   - show_internal_order_blocks=True, show_swing_order_blocks=False
   - show_equal_hl=True, show_high_low_swings=True, show_swings=False
4. 输出结构：events(BOS/CHoCH) / order_blocks(internal OB) / equal_highs_lows(EQH/EQL) / trailing(strong/weak high/low) / pivots / time
5. 每个事件含 anchor_index/anchor_time 与 confirmed_index/confirmed_time 因果契约
6. events 新增 bias 字段（BULLISH=1/BEARISH=-1），与 OB 输出一致
7. 未来 bar 不修改已确认事件（事件一旦写入即不可变）
8. FVG 完全排除：不计算、不返回、不缓存、不渲染

### 二、indicator_service 按需计算
1. compute_all_indicators 新增 include_smc: bool = False 参数
2. include_smc=False 时跳过 SMC 计算（不消耗 CPU），响应无 smc layer
3. include_smc=True 时计算 SMC 并注入 smc 图层
4. SMC 输入使用当前 timeframe 对应 macd_bars（与 MACD/SQZMOM 同源）
5. SMC 是独立图层，不进入 DSA、Node 监控、Capture 或右栏 context

### 三、indicator_cache 缓存隔离
1. ALGORITHM_VERSION 从 v5 bump 到 v6（旧 v5 缓存自动失效）
2. include_smc=True 时缓存键追加 :smc 后缀
3. SMC 与非 SMC 结果独立缓存，同 symbol/timeframe 切换开关不返回旧缓存
4. SMC 默认关闭（不带 :smc 后缀）

### 四、API 层
1. `/api/v1/instruments/{id}/indicators` 新增 include_smc 查询参数（bool，默认 false）
2. include_smc 缺省/false 时响应无 smc layer 且不计算
3. include_smc=true 时返回 smc layer（renderer/direction_colored/颜色/data）

### 五、前端图层系统 7→8
1. ChartLayerKey 新增 'smc'（trend/node/boll/volume/macd/sqzmom/breakout/smc）
2. CHART_LAYER_MANIFEST 新增 smc 条目（name="智能资金", kind="main", 默认 false）
3. defaultChartLayerVisibility：watchlist 和 selection 的 smc 默认 false
4. indicatorPreferences 迁移：旧 localStorage 偏好缺少 smc 时迁移为 false
5. StockResearchWorkspace 新增 onSmcToggle 回调
6. StockDetailPage + AdminStockDebugPage 新增 smc 状态管理
7. StrategyChart 新增 SMC Canvas 渲染（BOS/CHoCH 事件 + OB 矩形 + EQH/EQL 标记）
8. useStockResearchData 新增 includeSmc 传递
9. endpoints.ts 新增 include_smc 参数

### 六、MiniKline viewport P0 修复
1. 新增 `frontend/src/features/market-workspace/miniKlineViewport.ts` 纯函数
2. 新增 `frontend/src/features/market-workspace/__tests__/miniKlineViewport.test.ts`（12 项测试）
3. per-timeframe clamp（全部在 [30, 64] 区间内）：
   - 15m/60m: 50–64
   - 日线: 48–58
   - 周线: 40–52
   - 月线: 30–40
4. visibleBars = floor((contentWidth - 56) / 5)，clamp 到 per-timeframe 区间
5. from = max(0, dataLength - visibleBars)
6. to = dataLength - 1 + 3（右侧 3 bar 留白，最新 K 线不紧贴右轴）
7. MiniKlineCard 使用 computeMiniKlineViewport 替代 fitContent
8. 切周期不沿用旧 range（每次重新计算）
9. 价格轴宽度固定 56，effectivePlotWidth = floor(contentWidth) - 56
