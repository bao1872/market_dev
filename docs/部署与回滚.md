- **版本**：PostgreSQL 15+

- **数据库**：`bz_stock`（生产）+ `bz_stock_test`（测试）
- **监听**：`127.0.0.1:5432`
- **测试库容器**：`trading-postgres-test`（端口 5433）

**创建业务角色**（首次部署）：

## 2. 首次部署

### 2.1 克隆代码

```bash
cd /root
git clone <repo-url> web_dev
cd web_dev
```

### 2.2 配置 env 文件

```bash
sudo mkdir -p /etc/market-dev
sudo tee /etc/market-dev/market.env <<EOF
DATABASE_URL=postgresql+psycopg://bz:***@127.0.0.1:5432/bz_stock
JWT_SECRET=***
SECRET_MASTER_KEY=***
APP_ENV=production
EOF
sudo chmod 600 /etc/market-dev/market.env
```

### 2.3 执行数据库迁移

```bash
cd /root/web_dev/backend
source .venv/bin/activate
export DATABASE_URL="postgresql+psycopg://bz:***@127.0.0.1:5432/bz_stock"
alembic upgrade head
```

### 2.4 构建并启动生产服务

```bash
cd /root/web_dev
GIT_SHA=$(git rev-parse --short HEAD) \
BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ") \
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml build

docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d
```

### 2.5 验证

```bash
# 健康检查
curl http://localhost:8000/health
curl http://localhost:8000/health/ready
curl http://localhost:8000/version

# 容器状态
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

## 3. 发布流程

### 3.1 拉取最新代码

```bash
cd /root/web_dev
git pull origin main
```

### 3.2 执行数据库迁移（如有）

```bash
cd /root/web_dev/backend
source .venv/bin/activate
export DATABASE_URL="postgresql+psycopg://bz:***@127.0.0.1:5432/bz_stock"
alembic upgrade head
```

### 3.3 重建受影响的服务

```bash
cd /root/web_dev
GIT_SHA=$(git rev-parse --short HEAD) \
BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ") \
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml build

# 仅重启受影响的服务（如 backend 改动）
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d backend

# 如 Worker 改动，重启所有 worker
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d \
  worker-bars-scheduler worker-strategy-scheduler worker-calendar \
  worker-monitor worker-strategy-batch worker-outbox worker-delivery worker-capture

# 如前端改动
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d frontend
```

### 3.4 验证发布

```bash
# 健康检查
curl http://localhost:8000/health/ready
curl http://localhost:8000/version | python3 -m json.tool

# 验证 git_sha 已更新
GIT_SHA=$(git rev-parse --short HEAD)
VERSION_SHA=$(curl -sS http://localhost:8000/version | python3 -c "import sys,json; print(json.load(sys.stdin)['git_sha'])")
[ "$GIT_SHA" = "$VERSION_SHA" ] && echo "OK: git_sha 一致" || echo "FAIL: git_sha 不一致"
```

## 4. 回滚流程

### 4.1 代码回滚

```bash
cd /root/web_dev
# 查看最近 5 次提交
git log --oneline -5

# 回滚到指定 commit
git reset --hard <commit-sha>
```

### 4.2 数据库迁移回滚

```bash
cd /root/web_dev/backend
source .venv/bin/activate
export DATABASE_URL="postgresql+psycopg://bz:***@127.0.0.1:5432/bz_stock"

# 回滚一个版本
alembic downgrade -1

# 回滚到指定版本
alembic downgrade <revision-id>

# 查看当前版本
alembic current
```

> **注意**：数据库回滚可能造成数据丢失，仅在迁移有 down 方法时可用。生产回滚前必须先备份。

### 4.3 服务回滚

```bash
cd /root/web_dev
GIT_SHA=$(git rev-parse --short HEAD) \
BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ") \
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml build

docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d
```

### 4.4 数据库备份与恢复

```bash
# 备份
PGPASSWORD=*** pg_dump -U bz -h 127.0.0.1 -d bz_stock -F c -f /backup/bz_stock_$(date +%Y%m%d_%H%M%S).dump

# 恢复
PGPASSWORD=*** pg_restore -U bz -h 127.0.0.1 -d bz_stock -c /backup/bz_stock_YYYYMMDD_HHMMSS.dump
```

## 5. Alembic 迁移管理

### 5.1 迁移目录

`backend/alembic/versions/`，当前最新迁移：`034_message_delivery_group_id`

### 5.2 创建新迁移

```bash
cd /root/web_dev/backend
source .venv/bin/activate
alembic revision --autogenerate -m "add xxx table"
```

### 5.3 迁移规范

- **文件命名**：`<序号>_<描述>.py`（如 `035_add_user_preference.py`）
- **必须包含 down 方法**：支持回滚
- **数据迁移与 schema 迁移分离**：避免单次迁移过大
- **生产迁移前**：先在测试库验证

### 5.4 查看迁移状态

```bash
# 当前版本
alembic current

# 迁移历史
alembic history

# 待执行迁移
alembic heads
```
