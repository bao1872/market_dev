## 修改后行为
1. **ConsensusZone 真实实现**（PRD §7.4）：
   - 纯函数 `compute_consensus_zones(df)` — 接受 pandas DataFrame，返回 `ConsensusZoneResult`
   - 因果性过滤 `filter_bars_by_as_of(bars, as_of)` — 严格 `timestamp <= as_of`
   - 峰识别 `identify_peaks()` — 局部最大值（prominence 阈值）
   - 谷底分簇 `segment_clusters_by_valleys()` — 相邻峰间最低谷分割
   - 成交量加权百分位 `compute_volume_weighted_percentiles()` — P10/P50/P90
   - Redis 版本化缓存键：`consensus_zone:{symbol}:{as_of}:{timeframe}:{algo_version}:{data_version}`，TTL 3600s
   - 支持日线主结构和 15 分钟细化
   - 22 个测试覆盖：单峰/多峰/空分布/重叠/因果性/缓存键/纯函数

2. **qstock 板块同步**（PRD §7.5）：
   - 新增 `market_boards` + `market_board_memberships` 表（migration 062）
   - `BoardFetcher` Protocol + `QStockFetcher` 适配器（延迟导入 qstock）
   - `StagingData` dataclass — 内存暂存完整快照
   - `validate_staging_data()` 纯函数 — 4 项校验：空集合/目录数下限(100)/成分数下限(3000)/异常降幅(>20%)
   - `sync_boards()` 异步服务 — 拉取 → 暂存 → 校验 → 原子切换（TRUNCATE+INSERT）
   - 失败保留上一成功版本，不删除旧关系，不阻塞盘后
   - 只保存最新关系，不存板块历史/行情/资金流
   - 11 个测试：7 纯函数 + 3 DB 集成 + 1 migration 循环

3. **industry/concept 筛选接入**：
   - market API 移除 422，通过 `filter_instruments_by_board()` 查询 market_boards 表
   - 无匹配股票时返回空列表（不报错）
   - industry + concept 组合筛选取交集（AND 语义）

4. **migration 061 冗余索引修复**：
   - 删除单列索引 `ix_feature_snapshot_source_run_id`
   - 组合索引 `ix_feature_snapshot_run_instrument(source_run_id, instrument_id)` 最左前缀已覆盖纯 run 查询

5. **sourceField/idempotencyKey 字段完全排除**：
   - `strip_internal_fields_for_user` 改为返回 dict（`model_dump()` + 递归 `pop()`）
   - JSON 序列化后字段**完全消失**（不是 null）
   - 管理员路径保留完整 Pydantic 模型
