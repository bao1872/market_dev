## 2. 修改前理解

### 2.1 Phase 1：只读复现 display_frame mismatch（已完成）

`ChartRenderFrame` 在 V1（CHANGE-20260721-001）中虽然分离了 `displayHash`/`displayRangeKey`，但仍存在三类问题：
- 展示窗口固定 100 根硬编码（`_display_window=100`），不随请求 bars 参数变化；
- display_frame 字段不完整，缺失 `requested_count`/`actual_count`/`first_time`/`last_time`/`include_realtime`/`is_partial`/`adjustment_as_of`；
- indicators API 缺少 `include_realtime`/`completed_only`/`adjustment_as_of` 参数，无法与 bars API 同款对齐。

### 2.2 Phase 2：Display Frame Contract V2

需删除 `_display_window=100` 硬编码，改用请求 bars 参数；display_frame 必须包含 V2 全字段；indicators API 必须新增 `include_realtime`/`completed_only`/`adjustment_as_of` 参数与 bars API 同款，缓存键追加 `DisplayWindowSpec` 后缀。

### 2.3 Phase 3：Canonical Node DTO V2

`node_cluster` DTO 结构需升级为 `node_regions` + 独立 `price_state`，四链统一消费。`ALGORITHM_VERSION` 从 v11 bump 到 v12，强制旧缓存失效。

### 2.4 Phase 4：移动飞书舞台 1440×2560

需移植为 1440×2560 9:16 移动竖屏舞台（`MobileIndicatorStage`），新增 `renderDensity` 集中管理 Canvas 字号/线宽/节点标记尺寸（desktop 保持不变，mobile_capture 放大）。Playwright 截图选择器为 `[data-testid="stock-detail-capture"]`。

### 2.5 Phase 5：复权+Snapshot 生产闭环

`_invalidate_downstream_caches` 需扩展为四层（新增 Capture 缓存精确清理）；`_audit_and_rebuild_factors` summary 需补全 PROMPT.md §5.4.2 真实生产验证字段（`trade_date`/`audit_rebuilt`/`failed_symbols`）。
