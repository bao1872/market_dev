## 修改前行为

### 详情来源列表
- 详情页 URL 存在第二套手工拼接（`detailNavigation.ts:buildStockDetailUrl`），与 `stockDetailNavigation.ts` 的统一 builder 并存。
- 来源由 `returnTo.scope` 推断，显式 `originScope` 不存在；`returnTo` 滞后时可能静默回退自选。
- `MarketWorkspacePage` 生成 returnTo 时直接复制可能滞后的 `location.search`。
- 上一只/下一只和左栏切换可能丢失来源上下文。

### 产品观察
- Context 响应无 `productObservations` 字段；confirmed swing 位置无独立展示项。
- `price_position_in_confirmed_swing_raw` 存在于底层 payload 但未在用户侧展示。
- 本地误改 `structural_factor_service.py` 新增 `price_position_in_confirmed_swing_0_1` 持久化字段（修改 worker 持久化链）。

### 近期变化
- `compute_recent_changes` 查询 ≤10 个已发布快照，返回多条历史变化表达。
- 响应无 `latestChangesFrom` / `latestChangesAsOf` 字段。
- `compute_recent_changes` 有 30 条上限。

### 详情右边界
- `.tv-detail-layout` 使用 `display: flex`，子元素可能撑宽导致右边界错位。
- `ResizeObserver` 使用 120ms 纯防抖，快速切周期/全屏时中间宽度绘制导致右边界错位。
- 缺少 `width:100% / min-width:0 / max-width:100%` 统一约束。

## 修改后行为

### 一、详情来源列表（originScope 唯一真源）
1. **新增 `stockDetailNavigation.ts`**：统一导航模块，提供 `buildStockDetailUrl` / `resolveStockDetailOrigin` / `sourceForOriginScope` / `strategyForOriginScope`。
2. **originScope 优先级**：显式 `originScope` 参数（market|watchlist）为来源第一真源，不被 `returnTo.scope` 覆盖；旧 URL 无 `originScope` 时兼容解析 `returnTo.scope`；无任何来源的直接 URL 默认 watchlist。
3. **冲突检测**：`originScope` 存在且 `returnTo.scope` 不同 → `contextMismatch=true`，禁止静默回退自选。
4. **三入口统一**：`MarketWorkspacePage` 点击股票、`useStockDetailActions` 上一只/下一只、`StockDetailPage` 左栏来源列表点击，全部调用同一 `buildStockDetailUrl`，删除手工字符串拼接。
5. **returnTo 生成**：`MarketWorkspacePage` 使用当前 `searchParams` 副本并强制写入 `scope` 和 `selected`，禁止直接复制可能滞后的 `location.search`。
6. **删除旧 builder**：`detailNavigation.ts` 旧 `buildStockDetailUrl` 已删除（禁止第二套 URL 拼接）。

### 二、最近确认区间位置（产品观察扩展）
1. **独立合同**：新增 `atomic_fact_product_observations_v1.json`，sourceFields 仅 `raw`/`confirmed_high`/`confirmed_low`，`inside` 由 raw 派生（不读取 worker 持久化的 `_0_1` 字段）。
2. **撤销 worker 改动**：`structural_factor_service.py` 不新增 `price_position_in_confirmed_swing_0_1` 持久化字段，不修改 worker 持久化链。
3. **compute_product_observations**：`inside = raw if 0<=raw<=1 else None`；范围内显示 0–1 轨道，<0 显示「低于确认区间」，>1 显示「高于确认区间」；区间外数值不静默 clip。
4. **Context 响应**：新增 `productObservations.structure` 字段；观察项 `publicKey=confirmed_swing_position`、`visualKind=confirmed_position`、`scope=product`。
5. **不计入 Core 14**：产品观察不进入 `core`/`auxiliary`/`availability`，不影响 14/14 统计。
6. **前端 UI**：`ConfirmedPositionRow` 组件，0.33/0.67 边界与 S3 一致；插入结构组「价格与已确认区间关系」之后；保留现有 S3「价格在当前主要波段的位置」。

### 三、近期变化只保留最近交易日
1. **查询 ≤2 个快照**：`_find_recent_published_snapshots` limit 10→2，只比较最新交易日与前一发布交易日。
2. **响应新增字段**：`latestChangesFrom`（前一发布交易日）、`latestChangesAsOf`（最新发布交易日）。
3. **空状态**：无变化显示「最近交易日（{asOf}）无状态变化」。
4. **删除 30 条上限**：`compute_recent_changes` 不再截断。
5. **产品观察变化**：confirmed_swing_position 变化可纳入产品观察变化，但标 `scope=product`，不伪装 V4.13 Core。

### 四、详情右边界
1. **grid 布局**：`.tv-detail-layout` 从 `display: flex` 改为 `display: grid; grid-template-columns: 200px minmax(0,1fr)`。
2. **统一宽度约束**：`.tv-workspace`/`.strategy-chart-wrap`/`.tv-chart-column`/`.tv-canvas-wrap` 统一 `width:100%; min-width:0; max-width:100%; box-sizing:border-box`。
3. **ResizeObserver**：改为 `requestAnimationFrame` 立即 draw + 120ms trailing draw；保留 window resize 补偿（等几何测试通过后再判断是否删除）。

## Known Gaps
- worker 仍旧镜像（当前生产依赖 fallback 重算）；产品观察扩展不修改 worker 持久化链，confirmed_swing_position 由 API 层从已有 raw 派生。
- Active Swing 公式（当前主要波段）：`(active close-active_low)/(active_high-active_low)`，用户值 clip 到 [0,1]。
- Confirmed Swing 公式（最近确认区间）：`(close-confirmed_low)/(confirmed_high-confirmed_low)`，raw 不 clip，inside 仅 0≤raw≤1 时有效。
