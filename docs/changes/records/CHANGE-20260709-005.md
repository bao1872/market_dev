# CHANGE-20260709-005: 飞书配置权限隔离 + 趋势选股返回状态恢复

## 变更编号

CHANGE-20260709-005

## 任务名称

hotfix：普通用户飞书配置误触 admin-only 实测按钮；趋势选股进详情后返回筛选条件重置

## 需求出处

用户指令：
- `/settings` 页面普通 member 也能看到「发送最近事件实测」按钮，但后端接口 admin-only，member 点击必然 403；
- 趋势选股页进入个股详情后返回，筛选/排序/关键词/页码全部重置。

## 修改前行为

1. **飞书配置权限错误**：
   - `SettingsPage` 中「发送最近事件实测」按钮对所有用户可见；
   - 后端 `POST /notification-channels/{channel_id}/test-latest-event` 使用 `require_roles("admin")`，普通用户点击 403；
   - 普通用户没有可用的「发送测试消息」渠道卡片按钮；
   - 表单文案「验证并保存」暗示保存即启用，且 `receive_id_type` 下拉缺少 `chat_id`/`union_id`。
2. **趋势选股返回状态丢失**：
   - `ScreenerPage` 的 strategy/keyword/sort/filters/page/pageSize 仅在组件 state；
   - `goDetail` 只 navigate 到 `/stock/${symbol}?source=selection&strategy=...`，未保存当前 screener URL；
   - `StockDetailPage` 返回按钮固定 `navigate('/screener')`，返回后重置。

## 修改后行为

1. **飞书配置权限隔离**：
   - `SettingsPage` 引入 `useAuthStore` 读取 `user?.is_admin`；
   - 普通用户看不到「发送最近事件实测」按钮；admin 可见，文案改为「管理员实测最近事件」；
   - 所有飞书渠道卡片增加「发送测试消息」/「测试并启用」按钮，调用 `POST /notification-channels/{id}/test`；
   - 测试成功后 toast「测试成功，飞书渠道已启用」并刷新 `useNotificationChannels` 列表，状态从 `pending` 变为 `active`；
   - 测试失败展示后端 `delivery.error_code` / `error_message`；
   - 表单 `receive_id_type` 下拉补充 `chat_id`、`union_id`，帮助文案区分个人/群；
   - 「验证并保存」改为「保存配置」，保存后仍为 `pending`。
2. **后端 403 提示友好化**：
   - `/notification-channels/{channel_id}/test-latest-event` 保持 admin-only；
   - 普通用户调用时返回 403，detail 为「最近事件实测仅管理员可用，普通用户请使用发送测试消息」。
3. **趋势选股 URL 状态持久化**：
   - `ScreenerPage` 将 strategy key、keyword、sort、filters、page、pageSize 同步到 URL query；
   - `StrategyDataTable` 抽出 `screenerUrlState.ts` 纯函数负责 encode/decode；filters 只保存 `key/op/value/value2`；decode 丢弃当前 columns 中不存在的陈旧 key；
   - 切换策略时更新 `strategy=` 并重置 `page=1`；
   - `goDetail` 把当前 `location.pathname + location.search` 作为 `returnTo` 通过 `navigate(..., { state: { returnTo } })` 传入个股详情；
   - `StockDetailPage` 返回按钮优先使用 `location.state?.returnTo`，没有时按 `source` fallback 到 `/screener` 或 `/watchlist`；所有返回按钮（加载中/未找到/正常页）走同一 `handleBack()`。
4. **可测试性**：
   - 新增 `frontend/src/pages/detailNavigation.ts` 纯函数，抽离 URL/state 构建与返回路径解析逻辑。
