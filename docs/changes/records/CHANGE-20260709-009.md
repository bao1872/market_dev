## 需求出处

mypy baseline 债务治理最终阶段：PR #58（22→0）和 PR #59（20→0）后，剩余 186 个 baseline 错误需全量清零。

## 修改前行为

- `tools/quality_baselines/mypy.json` 记录 186 个错误（141 unique），分布于 ~50 个 `app/` 文件；
- 错误类型包括：attr-defined（84）、arg-type（49）、assignment（19）、name-defined（9）、list-item（5）、operator（5）、return-value（4）、valid-type（4）、misc（3）、union-attr（2）、var-annotated（2）。

## 修改后行为

- `tools/quality_baselines/mypy.json` total=0, unique=0, diagnostics=[]；
- `mypy app` 零错误；
- `mypy app tests` 中 `app/` 零错误（`tests/` 有 300 个 pre-existing 错误，不在 baseline 管控范围）；
- 新增 `backend/app/models/_table_meta.py`：SQLAlchemy `__table__` metadata 类型安全访问 helper（`table_indexes`/`table_constraints`/`table_columns`）；
- 新增 `backend/app/core/route_utils.py`（PR #59 已 merge）：API 路由类型收窄 helper；
- 修改 `backend/app/constants/indicator_contract.py`：`VP_TROUGHS_SHOW` 从 `str` 改为 `Literal["troughs", "clusters", "none"]`；
- 全部修复使用 isinstance 收窄、显式 None 检查、Protocol 类型、TypedDict/NamedTuple、TYPE_CHECKING guard；
- 未使用 `type: ignore`、`cast`、`Any` 掩盖问题；
- 未改变运行时行为。
