# CHANGE-20260713-004: PR #74 阶段四 — /market DSA 列表恢复 + 列对齐修复 + 列设置 CRUD + columnOrder

## 变更编号
CHANGE-20260713-004

## 任务名称
恢复 /market 的 DSA 列表能力，修复字段和值错位，保留列设置与配置 CRUD，保持 Node Cluster/盘中监控边界不变。

## 需求出处
用户指令（ref/PROMPT.md）：在 `fix/pr74-context-layers-remove-consensus` 分支上继续未完成工作，不回滚、不重开分支、不合并 main。

## 修改前行为
1. `/market` 页面使用 `MarketStockTable` 组件（基于 `useMarketStocks` → `GET /market/stocks` API），不显示 DSA 数值字段（趋势/DSA 持续时间、VWAP、offset_mean/offset_std/offset_percentile 等）。
2. 表头和值错位：`MarketStockTable` 的列定义与数据映射依赖数组下标和对象遍历顺序，隐藏列/列排序后表头 th 与表体 td 不从同一 visibleColumns 派生。
3. 列设置不支持 `columnOrder`（调整顺序）；`/me/table-view-presets` 的 `TableViewPresetConfig` 白名单不含 `columnOrder`。
4. `/market` URL 契约复杂（scope/query/page/page_size/sort/selected/industry/concept/state），与 DSA 列表实际需求不匹配。
5. `MarketToolbar` 包含搜索词输入和行业/概念筛选，但 DSA API 不支持这些筛选维度。

## 修改后行为
1. **/market DSA 列表恢复**：`MarketWorkspacePage` 改用 `StrategyDataTable`（复用趋势选股的 DSA 列表能力），数据来自 `usePublishedRuns` + `useStrategyRunResults`（最新 published DSA run）；列定义复用 `getTrendSelectionColumns`（`frontend/src/features/trend-selection/columns.tsx`），包含 stock/change_pct/dsa_dir_bars/vwap_ret_avg/vwap_ret_total/offset_mean/offset_std/offset_percentile/dsa_vwap/dsa_vwap_dev_pct/offset_variance_rate/price/action；`scope=market` → `universe=all`，`scope=watchlist` → `universe=watchlist`。
2. **P0 列对齐修复**：提取 `reorderVisibleColumns` 纯函数到 `frontend/src/components/columnOrdering.ts`；表头 th、表体 td、colgroup col 三者从同一 `visibleColumns` 派生；每行 td 数 = 可见 th 数；单元格按 `col.key` 取值，禁止依赖数组下标；action/select 列固定 id 不参与重排。
3. **列设置与配置 CRUD**：`StrategyDataTable` 支持列显示/隐藏、调整顺序（`columnOrder`）、恢复默认、刷新保留（localStorage `table-columns:${tableId}` + `table-column-order:${tableId}`）；配置复用 `/me/table-view-presets`（`TablePresetMenu`），保存 `keyword/sort/filters/hiddenColumns/columnOrder/pageSize`；股票名称/代码和操作列不可全部隐藏；旧配置包含已删除字段时忽略未知项。
4. **后端 columnOrder 支持**：`TableViewPresetConfig` Pydantic schema 新增 `columnOrder: list[str] | None = None` 字段（`extra="forbid"` 白名单）；50 项 preset API 测试全通过（含 columnOrder 持久化、跨 session 读取、NULL strategy_key 场景）。
5. **URL 状态简化**：`/market` URL 契约简化为 `scope/selected`（`MarketWorkspacePage` 管理）+ `sort/dir/keyword/filters/page/page_size`（`StrategyDataTable` 内置 `screenerUrlState` 管理）；`marketWorkspaceUrlState.ts` 简化为仅 `scope/selected`。
6. **MarketToolbar 简化**：仅保留 scope 切换（watchlist/market）分段按钮；搜索词输入和列筛选由 `StrategyDataTable` 内置 UI 承载。
7. **右栏不变**：首次默认收起，收起时 0 context 请求；展开并选中股票后才请求 `useStockContext`；列表不显示形态状态/DSA状态/事件列。
8. **Node Cluster 边界保护**：本轮未修改 `indicator_contract.py`/`indicator_service.py`/`monitor_batch_service.py`/`volume_node_monitor.py`/`watchlist_monitor` 配置/`capture.py`/`stock_capture_service.py`；只读核查确认 Node 数学公式、Worker 调度、事件触发、冷却去重、通知、Capture 均相对基线未变。
