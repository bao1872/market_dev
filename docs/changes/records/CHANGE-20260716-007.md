## 修改前行为

### 数据源
- 板块同步使用 qstock（`qstock_fetcher.py`）+ THS 适配器（`ths_adapter.py`），依赖 py-mini-racer。
- 生产 `BOARD_SYNC_ENABLED=false`，THS 成分股端点 403 被反爬拦截，板块数据不可用。
- pywencai 0.13.1 曾探测但未加入生产依赖（CHANGE-20260714-001）。

### 同步服务
- `board_sync_service.py` 逐板块抓目录/成员，非原子快照。
- 完整性门禁为旧阈值（板块数 <100、成分数 <3000）。
- `code_uniqueness_rate` 计算为 `unique_stock_count / total_symbol_refs`（每股票多次引用导致比率极低）。

### 编排
- `worker.py` 注册独立 17:00 `scheduled_board_sync` cron job（qstock 语义）。
- `after_close_orchestrator.py` 无 `syncing_boards` 步骤。
- 板块同步与盘后流程完全独立，无断点恢复。

### 前端
- 行业/概念输入每敲一字即触发 API 请求。
- 无 stale 数据契约，`available=false` 时直接禁用。
- `/market/boards` 响应仅含 `available` + `reason_code`。

## 修改后行为

### 一、问财唯一数据源（pywencai）
1. **新增 `wencai_board_provider.py`**：固定查询 `同花顺概念，行业分类`，pywencai headers 补 `Referer: http://www.iwencai.com/`，`wc.get(query=..., loop=True, sleep=2, cookie=os.getenv("WENCAI_COOKIE") or None)`，最多 3 次有限重试。
2. **同步调用包装**：`asyncio.to_thread` 包装 pywencai 同步调用，不阻塞事件循环；不记录 Cookie 或完整原始响应。
3. **必需字段**：仅 `股票代码、股票简称、所属概念、所属同花顺行业`；忽略价格、市值、DDE、PE、market_code、数值 code。
4. **股票代码规范化**：取 `.SH/.SZ/.BJ` 前六位并保留前导 0。
5. **概念处理**：按 `;` 拆分、NFKC、trim、单股去重；超过 `MAX_CONCEPTS_PER_STOCK=100` 截断。
6. **行业规范化**：必须为「一级-二级-三级」完整路径，每股仅一个行业。
7. **stable external_code**：`wc:c:`/`wc:i:` + 规范化名称 SHA256 前 24 位；哈希冲突立即失败。
8. **兼容嵌套**：DataFrame/dict/list/tuple 嵌套，选择包含必需字段且行数最大的主表。

### 二、BoardSnapshot 原子快照同步
1. **完整快照构造**：`board_sync_service.sync_boards` 接收完整 `BoardSnapshot`（内存中全部 boards + memberships），不再逐板块抓取。
2. **绝对门禁（V1.1）**：原始行 ≥5000、代码唯一率 ≥99.9%（`unique_stock_count / raw_rows`）、行业 ≥200、概念 ≥300、关系 ≥60000、解析率 ≥95%、行业覆盖 ≥99%、每股行业恰好 1、单股概念 ≤100。
3. **相对门禁**：股票/行业/概念/关系任一下降 >20% 拒绝切换（首次同步 prev=0 不检查）。
4. **原子切换**：校验后一次批量解析 instrument，再在单一事务中差异 upsert/delete；任何异常 rollback 保留上次成功数据。
5. **刷新 updated_at**：成功同步时即使内容未变化也统一刷新有效 board 的 `updated_at`。
6. **未解析股票**：只记录总数和前 50 个脱敏样本，不把全量列表写 metadata。

### 三、盘后编排（soft-failure）
1. **新状态**：`AfterCloseRunStatus.SYNCING_BOARDS`，位于 `REFRESHING_DAILY` 与 `WAITING_DSA_WORKER` 之间。
2. **正常流程**：`refreshing_daily → syncing_boards → waiting_dsa_worker → quality_gate → feature_snapshot → publishing → succeeded`。
3. **位置**：板块同步在日线刷新后、任何「DSA 未触发提前结束」之前执行；非交易日跳过；`mode=dsa_only` 默认跳过，避免人工 DSA 重跑访问问财。
4. **软失败**：失败不覆盖旧数据、不阻断 DSA/快照/发布；写 `job_run_event` 和 metadata（status/source/completed_at/raw_rows/resolved/unresolved/industry_count/concept_count/membership_count/duration_ms/error_code/reused_previous_snapshot）。
5. **断点恢复**：无论成功或 degraded 均把该步骤记为已处理，断点恢复不重复请求问财。
6. **Redis 状态**：`record_sync_status()`/`get_sync_status()` 写入 key `board_sync:status`，TTL 7 天。

### 四、worker.py 清理
1. **删除 17:00 独立 qstock 板块任务**：移除 `scheduled_board_sync` 函数、cron 注册、`_resolve_instruments` 嵌套函数。
2. **保留 `BOARD_SYNC_ENABLED` 开关**：语义改为 pywencai；`false` 时步骤 `skipped`。
3. **不新增并行板块 scheduler**。

### 五、API 契约（/market/boards）
1. **扩展响应**：`source`（str|null）、`stale`（bool）、`last_attempt_status`（str|null）。
2. **stale 语义**：旧数据存在而最新同步失败时 `available=true, stale=true`，仍允许筛选；从未成功才 `available=false`。
3. **请求链禁联网**：`/market/boards` 继续读数据库 + Redis 状态，不在用户 API 请求链访问问财。

### 六、前端
1. **行业/概念提交修复**：各用本地输入，Enter/blur/选中时才提交；只接受当前目录精确值；清空立即提交并重置 `page=1`。
2. **stale 显示**：stale 时控件仍可用并显示简短「沿用上次板块数据」；loading 与从未可用分开。
3. **行业显示**：可将 `-` 渲染为 `/`，API 值不变。
4. **不新增前端依赖**。

### 七、依赖清理
1. **pyproject.toml**：移除 `py-mini-racer`；新增 `pywencai==0.13.1`；保留 `qstock==1.3.1`（strategy_assets 仍用日线和交易日历）。
2. **删除文件**：`qstock_fetcher.py`、`ths_adapter.py`、`test_qstock_fetcher.py`。
