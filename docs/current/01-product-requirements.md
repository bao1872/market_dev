# 01 当前产品需求（PRD）

## 1. 产品目标

盘迹帮助普通投资者完成：

1. 从全市场特征中发现值得继续研究的股票；
2. 在个股详情中验证趋势、成交参与、节点和其他研究信息；
3. 将股票加入自选并在盘中接收结构化事件提醒。

产品不是信号售卖器，不把单一指标解释为确定性买卖建议。

## 2. 信息架构

### 2.1 公开区域

- `/`：门户页
- 内测申请弹窗
- `/login`：登录和邀请码注册
- `/subscription-expired`：订阅到期与续期的 canonical 路由
- `/membership-expired`：仅作为兼容重定向，不承载独立业务逻辑

### 2.2 有效会员区域

- `/overview`：服务总览
- `/screener`：趋势选股
- `/watchlist`：我的自选
- `/stock/:symbol`：个股详情
- `/messages`：消息中心
- `/settings`：账户和通知渠道

### 2.3 到期或无有效订阅会员区域

允许：登录、账户信息、套餐查询、邀请码续期、账户安全、历史消息只读。

禁止：趋势选股、自选股读取和修改、个股详情、盘中监控、新事件、新通知和新飞书投递。原有自选股、历史消息和历史事件不删除，续期成功后自动恢复。

### 2.4 管理员区域

- `/admin`、`/admin/overview`：管理总览
- `/admin/users`：用户、订阅、邀请码和审计
- `/admin/beta-applications`：内测申请
- `/admin/strategies`：策略、版本和运行
- `/admin/jobs`：后台任务、Worker、投递和恢复

## 3. 功能需求

### PRD-PORTAL-001 门户页

设计状态：`CURRENT`  
实现核对：未发现本轮阻断差异，仍需最终 E2E 验收。

使用盘迹品牌和深色研究型视觉；清楚表达“发现—验证—追踪”；展示趋势特征、产业逻辑链、个股详情和飞书通知；不承诺收益；无效或未实现能力不得作为已上线功能宣传。

### PRD-BETA-001 内测申请

设计状态：`CURRENT`  
实现核对：本轮未专项核验完整通知与重试链路。

访客可以提交联系方式、关注股票数量、申请原因和补充说明。后台数据必须真实保存，管理员通知失败不得影响申请成功，但必须记录可重试状态。

### PRD-AUTH-001 登录、注册与续期

设计状态：`EXPERIMENTAL`  
实现核对：基础角色、Plan、Subscription 已存在；管理员完整订阅管理仍为 `KNOWN_GAP`。

采用邀请码注册、`member` 角色、`plans` 套餐和 `subscriptions` 订阅。管理员无套餐、无订阅。到期规则以本文件和权限文档为准，不能由前端或单个 API 自行改变。

### PRD-OVERVIEW-001 服务总览

设计状态：`CURRENT`  
实现核对：需要以 Worker 心跳、任务记录和生产运行态完成验收。

展示实际数据日期、最新完整发布批次、用户监控资格、最近事件和真实任务状态。不得使用静态假数据冒充 Worker 或任务结果。

### PRD-SCREENER-001 趋势选股

设计状态：`CURRENT`  
实现核对：`KNOWN_GAP`，部分 API 未要求有效订阅，100ms 超时和宽松发布门禁导致结果不完整。

- 每个批次对全部可计算股票生成 DSA 特征；
- 计算阶段不按方向、强弱、`matched` 或用户筛选条件删除股票；
- 不可计算股票只能按明确允许原因跳过，并保存 reason code；
- `partial_failed` 批次不得自动发布；
- 用户仅查询最新完整 `published` 批次；
- 前端筛选、排序、关键词和分页只作用于已发布全量结果；
- 页面显示 universe、成功、失败、跳过、结果总数和覆盖率；
- 当前不提供多策略组合。

### PRD-WATCH-001 我的自选

设计状态：`CURRENT`  
实现核对：`KNOWN_GAP`，接口未全部接入统一订阅依赖，额度检查函数存在未收口调用。

- 有效会员添加股票后自动进入盘中监控，不创建 MonitoringPlan；
- 套餐额度由后端事务内校验；
- 到期或无有效订阅用户不能查看、添加、删除、恢复或读取监控状态；
- 到期不删除已有自选，续期后恢复；
- 页面展示股票、当前价、涨跌幅、上下节点、POC、最近事件和详情入口。

### PRD-STOCK-001 个股详情

设计状态：`CURRENT`  
实现核对：`KNOWN_GAP`，统一实时行情聚合和尾部补齐尚未完成。

- 历史已完成 Bar 来自数据库；
- 数据库尾部缺失时补齐已完成 Bar；
- 交易时段聚合最新 1m 数据形成当前 15m、1h、1d、1w、1mo partial Bar；
- 页面返回并展示 `as_of`、`data_source`、`is_partial` 和降级状态；
- K 线、指标和飞书截图必须使用同一聚合行情和时间键；
- DSA 与 Node Cluster 图层可开关；
- 到期会员不能访问。

### PRD-MESSAGE-001 消息中心与飞书

设计状态：`CURRENT`  
实现核对：`KNOWN_GAP`，飞书图片投递、部分失败状态和仅重试图片尚未完成真实 E2E。

- 消息包含股票、事件时间和个股详情入口；
- 文字和图片分别记录状态；
- 文字成功、图片失败显示 `partial_failed`，允许仅重试图片；
- 不得因图片失败返回笼统“发送成功”；
- 到期会员可只读历史消息，但不产生新消息和外部投递。

### PRD-SETTINGS-001 设置页

设计状态：`CURRENT`  
实现核对：基础渠道配置已存在，Secret 与所有权仍需专项验收。

用户管理自己的账户安全和通知渠道。Secret 只可写入、验证、掩码展示和替换，不能明文回显。

### PRD-ADMIN-001 管理后台

设计状态：`CURRENT`  
实现核对：`KNOWN_GAP`，已实现邀请码、列表、部分任务与审计；用户启停及完整订阅变更 API 尚不完整。

管理员可以查看和管理用户状态、订阅、邀请码、策略、任务、投递和审计日志。界面中没有真实后端能力的控件必须删除或标记不可用，禁止只显示成功 Toast。

## 4. 非功能需求

- 所有关键数据带业务时间、生成时间和版本；
- 生产算法参数不可由普通用户修改；
- 用户数据严格隔离；
- Worker 可观察、可恢复、幂等；
- 文档、代码、测试和部署配置保持一致；
- 任何发布批次和消息都可追溯到源数据、策略版本和 Git SHA。
