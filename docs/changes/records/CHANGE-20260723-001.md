## 修改前行为
1. **MDAS**：`_DEFAULT_INTRADAY_LOOKBACK_DAYS=180` 导致 15m 不足 4000 根（1872 根），无 `history_exhausted` 字段区分"DB 历史不足"与"系统未取满"
2. **实时 K 线**：`/quote` 实时与 `/bars?timeframe=1d&include_realtime=true` 实时语义混淆；前端 `mergeRealtimeQuoteIntoBars()` 可能替代后端 partial bar
3. **SMC 渲染**：使用 index fallback（`smcIdx` 直接映射 displayIdx），viewport 从 250→90 时标签错位；anchor/confirmed time 缺失或匹配失败时仍渲染
4. **因子版本**：`instruments.factor_algorithm_version`/`factor_reconciliation_version`/`factor_reconciled_at` 三列已存在（migration 065）但从未写入；无影响集识别机制
5. **Auto-resume**：`auto_resume_interrupted_after_close_runs` 已实现但缺乏受控测试验证状态机闭环
6. **AGENTS.md**：289 行，超过 180-220 行目标

## 修改后行为
1. **MDAS count-aware 回补（CP-V3-A2）**：daily required_count=250、15m required_count=4000；实际不足时自动向前扩展直到达到 required_count、到达真实上市起点或安全边界；返回 `history_exhausted: bool` 区分两种不足
2. **实时 partial bar 收口（CP-V3-B2）**：交易时段内 `/bars?timeframe=1d&include_realtime=true` 必须返回今日 partial daily bar（`is_partial=true`、`data_source=hybrid`、`last_live_bar_time` 非空）；前端 `mergeRealtimeQuoteIntoBars()` 只作兜底视觉增强
3. **SMC 严格 time-key（CP-V3-C2）**：`strictTimeKey=true` 时 time 缺失→`missing_time`+skip，time 匹配失败→`match_failed`+skip，禁止 index fallback；events/EQH/EQL 使用 OR 逻辑；详情链与 Capture 共用同一 SMC 坐标映射核心
4. **因子版本追踪（CP-V3-D）**：`stamp_factor_reconciliation_version` 成功重建后写入三字段；`find_stale_version_instruments` 识别版本过期影响集；`_rebuild_single` 成功路径接入 stamp
5. **Auto-resume 受控测试（CP-V3-D）**：9 个测试覆盖 refreshing_daily/waiting_dsa/feature_snapshot 三步骤中断恢复、attempt_no 递增+max=3、lease_epoch fencing、唯一活跃记录、无僵尸 running、非 after_close 排除、完整状态机闭环
6. **AGENTS.md 压缩**：289→181 行，新增 §7.12 MDAS count-aware、§7.14 SMC 严格 time-key、§7.22 因子版本追踪与 auto-resume
