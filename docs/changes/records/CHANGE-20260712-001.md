# CHANGE-20260712-001: PR #74 两项架构纠偏 — board_sync 合并进 bars_scheduler + ConsensusZone 数据源修正

## 变更编号
CHANGE-20260712-001

## 任务名称
PR #74（`refactor/final-market-workspace`）两项架构纠偏：
1. arch1：board_sync 合并进 bars_scheduler（移除独立 Docker 服务和 WORKER_TYPE）
2. arch2：ConsensusZone 数据源修正（从 `macd_bars` 改为固定窗口 `daily_bars`）

## 需求出处
PR #74 架构纠偏指令：
- board_sync 不应是独立 worker，应合并进 bars_scheduler 同一调度器
- ConsensusZone 不应使用随显示 timeframe/count 变化的 `macd_bars`，应使用固定窗口日线

## 修改前行为
1. **board_sync**：曾设计为独立 `worker-board-sync` Docker 服务 + `run_board_sync_scheduler_worker()` 函数 + `WORKER_TYPE=board_sync_scheduler`；`docs/current/03-jobs` 标记为 "TODO 未接入调度"。
2. **ConsensusZone**：`indicator_service` 使用 `macd_bars`（随显示 timeframe/count 变化）作为数据源；`timeframe` 跟随显示周期；`as_of` 取 `macd_bars` 末根时间；缓存键随显示周期切换而变化；文档描述 "主结构使用日线，细化分布使用 15 分钟数据"。

## 修改后行为

### arch1: board_sync 合并进 bars_scheduler
- `worker-board-sync` Docker 服务从 `docker-compose.prod.yml` **移除**；
- `run_board_sync_scheduler_worker()` 独立函数从 `backend/app/worker.py` **删除**；
- `board_sync_scheduler` **不再是**有效 `WORKER_TYPE`（不在 dispatch 列表 / assert 列表）；
- board_sync job **注册在 `run_bars_scheduler_worker()` 内**的同一个 `AsyncIOScheduler`：
  - `CronTrigger(day_of_week="mon-sun", hour=17, minute=0, timezone=Asia/Shanghai)`
  - `max_instances=1`（单并发）
  - `id="board_sync_daily"`
  - 与 `bars_refresh`（16:00）共用同一 scheduler，两 job 独立调度、失败互不影响；
- `job_name="board_sync_scheduler"` 字符串仍用于 `SchedulerJobRun` 跟踪（内部 job 名，**非 WORKER_TYPE**）；
- qstock 同步调用通过 `asyncio.to_thread()` 包装在 `QStockFetcher` 中，不阻塞事件循环。

### arch2: ConsensusZone 数据源修正
- `indicator_service` ConsensusZone 数据源从 `macd_bars` **改为 `daily_bars`**（固定 250 根日线窗口，独立于显示 timeframe/count）；
- `timeframe` 参数固定为 `"1d"`（不是显示周期）；
- `as_of` 取最后一根日线 bar 时间（不是 `macd_bars` 末根时间）；
- V1 算法：仅日线成交分布；15 分钟细化分布是未来工作，bump 算法版本后才启用；
- 缓存键 `consensus_zone:{symbol}:{as_of}:1d:{algo_version}:{data_version}` 在显示周期切换时保持稳定；
- 前端 manifest：`consensus_zone` 为 `enabled=true, defaultVisible=true, name="筹码共识区"`。
