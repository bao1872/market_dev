## 变更编号

CHANGE-20260709-001

## 任务名称

研究特征矩阵 DB 主存储 + 因果口径 registry + compute + writer + CLI + 5 个 Blocker 修复

## 需求出处

用户指令（PR #51 review 反馈）：在启动全量后台回补前，必须先修 5 个 blocker，再分阶段验证，最后才允许后台逐月跑。

## 修改前行为

- registry 27 字段，无 `db_column` 映射；
- CLI 输出 parquet 文件为主，DB 未设计；
- DSA hindsight 字段用 causal segment_ids/directions/age_bars 近似冒充；
- Node Cluster 字段全 NULL 但未在 metadata 标记；
- 失败率统计只计 `failed + 1`（股票级），不计 `failed_rows`（行级）；
- upsert 异常未 `await db.rollback()`，污染后续事务；
- 无进程锁，可能重复启动后台任务。

## 修改后行为

- registry 扩展到 33 字段（causal 16 + confirmed_delay 4 + hindsight 6 + label 7），新增 `FeatureSpec.db_column` 把 dotted key 映射为下划线列名；
- DB 为主存储，新增两张专用表（`research_feature_matrix_runs` + `research_feature_matrix_rows`）；
- **Blocker 1（DSA hindsight 置 NULL）**：`hindsight_dsa_finalized_*` 3 列在 Phase 1 全 NULL，不得用 causal 近似冒充；`compute_dsa_dual_track_features` 保留占位但写入 NaN；run metadata_json 记录 `dsa_hindsight_status=not_implemented`；
- **Blocker 2（Node Cluster metadata 标记）**：`hindsight_node_cluster_*` 3 列全 NULL；run metadata_json 记录 `node_cluster_status=not_implemented` + `feature_version=phase1_no_node_cluster`；PR body 不得说 Node Cluster 已完成；
- **Blocker 3（失败率统计修正）**：`failed_count` 列存 `failed_rows`（行级，用于失败率 = `failed_rows / expected_rows`）；`metadata_json.failed_instruments` 存股票级失败数；`_process_instrument` 失败返回 `(0, expected_rows)`；
- **Blocker 4（DB 异常 rollback）**：`_process_instrument` 捕获异常时 `await db.rollback()` 后继续下一只股票；rollback 失败只记日志不 crash；
- **Blocker 5（进程锁）**：双保险 `pg_advisory_lock` + lock file；同 `month/scope` 已有 running run 或 lock 存在时拒绝启动；CLI 主流程 try/finally 释放锁。

## 风险

- **Phase 1 字段不完整**：hindsight 6 列全 NULL，不可用于真实回测；run metadata 明确标记 `feature_version=phase1_no_node_cluster`；
- **migration 058 未应用**：PR merge 前无法进行 B/C/D 阶段验收；
- **单进程性能瓶颈**：compute 是主要瓶颈（PR #41 profile 数据），全市场 2026-01 可能耗时较长；
- **磁盘占用**：单月预估 ~0.2GB（5293 股 × 20 交易日），远低于 3GB 阈值，但 6 个月累计可能 ~1.2GB；
- **进程锁不是分布式锁**：lock file 只在单机有效，多机部署需要改用分布式锁。

## 遗留问题

- **DSA raw hindsight 未实现**：后续 PR 需要绕过 `_remove_dsa_lookahead` 取 raw DSA full series；
- **Node Cluster 未集成**：后续 PR 需要集成 `VolumeNodeMonitor`；
- **`--workers` 多进程未实现**：当前单进程，后续可加 multiprocessing；
- **migration 058 未应用**：PR merge 后需要手动 `alembic upgrade head`；
- **A/B/C/D/E 阶段验证未执行**：需要 PR merge + migration 应用后按顺序执行。
