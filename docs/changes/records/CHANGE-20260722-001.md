# CHANGE-20260722-001: docs/记忆系统真正收口 — AGENTS.md v3 压缩 + ADR/Runbook/Evidence 模板 + MANIFEST baseline 新鲜度门禁

## 变更类型

docs / 治理工具 / 记忆系统

## 关联

- CP-19（分支 `fix/production-pipeline-stability-v1`）
- PROMPT.md §4（docs/记忆系统没有完全完成）
- ADR-0001 / ADR-0002
- Runbooks: after-close-recovery / feishu-image-issues / branch-deployment-rollback

## 背景

PROMPT.md §4 指出 Phase 4 收尾时仍存在两个 docs/记忆系统问题：

1. CP-14 名称含"docs/AGENTS"，但实际未压缩 AGENTS.md（仍 909 行），新文档结构已建立但旧记忆系统未真正替换。
2. `check_docs_consistency.py` 的 MANIFEST baseline 校验仅要求 SHA 是 HEAD 祖先（规则 4），即使 baseline `18049da` 落后 HEAD 88 个 commit 仍能通过，导致文档与代码严重脱节。

## 变更内容

### CP-19.1：AGENTS.md v3 收口

- AGENTS.md 从 909 行压缩到 290 行
- 移除内联 clause 39-64 变更历史，保留硬规则与必读入口
- 变更历史指向 `docs/changes/CHANGELOG.md` + `docs/changes/records/CHANGE-*.md`
- 保留所有硬规则：Node Cluster 250/4000/2、飞书 platform_app、Capture Token 隔离、ref/ 彻底隔离、Migration、MDAS SSOT、Atomic Snapshot 单 MDAS、SMC FVG 排除、Canonical 四链、AFC Core 14、三链五周期一致性、K 线实时契约、板块同步降级保护、文档目录与 CI 门禁、提交安全

### CP-19.2：ADR/Runbook/Evidence 模板

- `docs/decisions/README.md`：ADR 模板（状态/日期/关联/Context/Decision/Consequences）+ ADR-0001（Atomic Snapshot 单 MDAS）+ ADR-0002（Node Cluster 输入契约隔离）
- `docs/runbooks/README.md`：Runbook 模板（触发条件/症状识别/排查步骤/修复操作/验证/防止复发）+ 3 个 Runbook（after-close-recovery / feishu-image-issues / branch-deployment-rollback）
- `docs/evidence/README.md`：生产验收证据模板（部署日期/分支/Merge SHA/镜像 SHA/部署范围/Migration/关键功能验收/性能与资源/回滚验证/遗留问题）

### CP-19.3：MANIFEST baseline 新鲜度门禁

- `tools/check_docs_consistency.py` 新增规则 16：baseline SHA 必须在 HEAD 的最近 `BASELINE_FRESHNESS_WINDOW=50` 个 commit 内
- 新增 `count_commits_ahead_of_baseline(sha)` 使用 `git rev-list --count HEAD ^<sha>` 计算落后 commit 数量
- 新增 `check_baseline_freshness(shas)` 校验函数，集成到 `check_manifest_baseline()`
- `docs/current/MANIFEST.md` baseline 从 `18049da` 同步到 CP-18 HEAD `2c4ad50`
- `tools/tests/test_check_docs_consistency.py` 新增场景 12（落后 88 commit 失败）、13（窗口内通过）、13b（边界 50 通过）
