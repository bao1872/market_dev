## 修改后行为

### 一、/market 列表产品回归

1. **action 列改名"自选"**：渲染"加入自选/移除自选"按钮（替代旧"详情"按钮），title 在 `onToggleWatchlist` 模式下动态为"自选"，否则为"操作"（兼容 ScreenerPage）；按钮 `onClick` 调用 `e.stopPropagation()` 防止冒泡到 `<tr onClick>`；按 `instrument_id` 维护 `pending` Set 防重复点击。
2. **股票名称链接**：股票名称/代码为可点击 `<a>` 链接，点击进入 `/stock/:symbol?returnTo=<编码后的当前 /market URL>`；`onClick` 调用 `e.stopPropagation()` + `e.preventDefault()`，使用 React Router `useNavigate` 导航；行其他区域仍只负责 `selected` 和右栏。
3. **股票单元格不再显示行内涨跌幅**：`renderStock` 只显示名称/代码/市场，独立 `change_pct` 列保留（涨红跌绿 + fmtChange）。
4. **watchlist 单次请求 + Set**：`MarketWorkspacePage` 调用 `useWatchlist` 一次，`useMemo` 按 `instrument_id` 建 Set 判断 watched 状态；`useAddToWatchlist`/`useRemoveFromWatchlist` 成功后 invalidate `['watchlist']` 和 `['watchlist', 'monitor-status']`；watchlist scope 移除自选后该行消失（`universe=watchlist` 结果重新请求）；禁止逐行查询/N+1。
5. **批次信息 admin-only**：数据日期/批次/状态属于调试信息，普通用户 DOM 中完全不渲染（不是 CSS 隐藏）；仅 `useAuthStore(s => s.user?.is_admin === true)` 为真时渲染，默认折叠为"批次信息"区块，点击展开后显示 `run_trade_date`/`run_published_at`/`run_status` 等字段。

### 二、/market 单一搜索 SSOT

6. **顶部搜索框**：`MarketToolbar` 渲染唯一全文搜索框（占位"搜索股票代码/名称/拼音首字母"），`inputValue` 本地 state 避免逐字符 API；Enter/失焦提交，清空立即提交（onChange 检测空值）。
7. **StrategyDataTable 隐藏内置搜索**：`/market` 传 `searchable={false}`，避免与顶栏搜索并存。
8. **受控 keyword 模式**：`StrategyDataTable` 新增 `externalKeyword`/`onKeywordChange` 受控 props；`effectiveKeyword = externalKeyword !== undefined ? externalKeyword : globalQuery`；URL hydration 时回调 `onKeywordChange(state.keyword)` 同步到 parent；`applyPresetConfig` 应用 preset keyword 时调用 `onKeywordChange`；URL sync 使用 `effectiveKeyword`；`onQueryChange` 回调包含 `keyword: effectiveKeyword.trim()`。
9. **单一状态真源**：`MarketWorkspacePage` 持有 `keyword` state（初始值从 URL `keyword` 读取）→ `StrategyDataTable` 通过受控 props 接收 → URL `keyword` 同步；预设/筛选/排序/分页/列顺序继续保留。

### 三、后端 keyword pinyin_initials 匹配

10. `strategy_result_repository.query_results` 的 `keyword` 参数（非空时）ILIKE 同时匹配 `Instrument.symbol`/`Instrument.name`/`Instrument.pinyin_initials`（3 处 or_ 分支同步：keyword 过滤、filtered_total 子查询、source_total 子查询）；支持股票代码/中文名称/拼音首字母搜索；前端不做全量过滤，不增加新表。
11. `total` 字段为该 keyword + filters 下的真实总数（不是 items.length）。

### 四、消息数量 SSOT

12. `MessagesPage` 使用 `useUnreadCount`（`GET /messages/unread-count`，queryKey `['messages', 'unread-count']`）作为未读权威数量。
13. "全部"显示后端列表 `messagesQuery.data?.total`（不用 `items.length`）。
14. 页头显示"共 X 条 · 未读 Y 条"。
15. 分段按钮仅 `all`/`unread` 显示计数；`selection`/`price`/`system`/`process` 不显示误导数字（除非后端实现单条 GROUP BY counts API，禁止新增表/缓存/Worker）。
16. `AccountMenu` 消息项显示未读数 badge（`>99` 显示 `99+`），`unread>0` 时点击进入 `/messages?filter=unread`。
17. 标记单条/全部已读后 `useMarkMessageRead`/`useReadAllMessages` 的 `onSuccess` invalidate `['messages']`，自动刷新列表 + unread-count + 菜单角标。
18. "全部已读"按钮 `disabled` 条件增加 `|| unreadCount === 0`。

### 五、消息跳转目标

19. 单只股票消息点击进入 `/stock/:symbol?event_id=...&returnTo=/messages`（不再 `/market?symbol=`）。
20. `selection_composite` 类型消息进入 `/market`（不再 `/screener`）。
21. 抽屉内单只标的跳转同样进入 `/stock/:symbol?event_id=...&returnTo=/messages`。
22. `returnTo` 必须经 `normalizeInternalReturnTo` 校验。

### 六、K 线 Pointer Events 拖拽

23. `StrategyChart` 使用 Pointer Events（`pointerdown`/`pointermove`/`pointerup`/`pointercancel`）替代旧 mouse 事件。
24. `pointerdown` 调用 `setPointerCapture`，`pointerup`/`pointercancel` 调用 `releasePointerCapture`。
25. `dragRef` 保存 `{startClientX, startViewport, pointerId}`，`pointermove` 从 `startViewport` 计算总位移（禁止在 stale viewport 上累计）。
26. 向右拖查看更早数据，向左拖回到最新。
27. `dragMovedRef` 4px 阈值抑制 click（避免拖动误触节点/事件点击）。
28. cursor 为 `grab`/`grabbing`；鼠标移出 canvas 后仍可继续拖动（依赖 setPointerCapture）。
29. 保留滚轮锚点缩放、双击复位和移动端双指缩放。
