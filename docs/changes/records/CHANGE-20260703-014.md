# CHANGE-20260703-014

## 变更编号

CHANGE-20260703-014

## 任务名称

删除独立管理员飞书渠道配置，管理员内测申请通知复用管理员用户自己的 feishu_platform_app NotificationChannel

## 需求出处

用户于 2026-07-03 明确要求：
"管理员已经是权限体系中的用户，应复用管理员账号在 /settings 配置的 feishu_platform_app NotificationChannel，不应再维护第二套管理员飞书 App 和接收人配置。"

## 当前问题

1. `backend/app/constants/system_channel.py` 维护独立管理员飞书凭证（`ADMIN_FEISHU_APP_ID/APP_SECRET/RECEIVE_ID/RECEIVE_ID_TYPE`），与"飞书唯一接入为 feishu_platform_app"规则冲突；
2. `docker-compose.prod.yml` 传递 `ADMIN_FEISHU_*` 环境变量；
3. `outbox_relay.py` 存在"系统级渠道不入库"的旧实现和陈旧 Webhook 注释；
4. `feishu_card_builder.py` 残留 `mask_webhook_url` 函数；
5. `tools/pre_deploy_check.py` 要求 `SECRET_KEY`、`FEISHU_APP_ID` 等不必要的环境变量；
6. 生产环境 `/etc/market-dev/market.env` 存在 `ADMIN_FEISHU_WEBHOOK_URL` 等 Webhook 残留变量。

## 当前设计理解

- 管理员身份由 `users` / `roles` / `user_roles` 表决定，不依赖独立飞书配置；
- 管理员通知应复用标准 `NotificationMessage` → `MessageDelivery` → `FeishuPlatformAppAdapter` 链路；
- 不能走普通 `notification.message.created` 路径，因为该路径调用 `eligible_user_service`，会排除 admin；
- 需要专用 Outbox 事件 `beta_application.admin_notification.created`，由 outbox_relay 专用分支扩张；
- 无管理员渠道时不得阻塞部署、不得无限重试。

## 风险

- 管理员未在 /settings 配置 active Platform App 渠道时，通知无法投递；已通过 `feishu_delivery_status='failed'` + `feishu_last_error='ADMIN_PLATFORM_CHANNEL_NOT_CONFIGURED'` 处理；
- outbox_relay 专用分支需确保不进入普通 `notification.message.created` 路径，避免 `eligible_user_service` 过滤 admin；
- 幂等键需包含 `application_id + admin_user_id + channel_id`，防止重复投递。

## 验收标准

1. active admin + active Platform App 渠道收到通知；
2. admin 无 subscription 仍能收到；
3. inactive admin 不接收；
4. 普通 member 不接收管理员通知；
5. 多管理员、多渠道正确扩张；
6. 无管理员渠道不影响申请提交；
7. 无管理员渠道不无限重试；
8. 幂等键阻止重复投递；
9. delivery worker 复用 Platform App Adapter；
10. runtime、Compose、current docs 零 ADMIN_FEISHU_*；
11. runtime、Compose 零 Webhook；
12. admin 可访问 /settings；
13. 用户普通通知、Capture、Node、DSA 不回归。

## 实际修改内容

1. 删除 `backend/app/constants/system_channel.py`；
2. `outbox_relay.py`：新增 `_expand_beta_application_admin_notification` 函数，查询 active admin 用户的 active `feishu_platform_app` 渠道，为每个渠道创建 `NotificationMessage` + `MessageDelivery`；无渠道时标记 `feishu_delivery_status='failed'` + `ADMIN_PLATFORM_CHANNEL_NOT_CONFIGURED`；删除未使用的 `NotificationMessage` 导入；
3. `beta_application_notifier.py`：事件类型改为 `beta_application.admin_notification.created`，payload 仅含 `application_id`；
4. `beta_application_service.py`：`_BETA_APPLICATION_ADMIN_EVENT` 同步更新；
5. `delivery_worker.py`：新增 `_ADMIN_NOTIFICATION_SOURCE_TYPES` 集合，管理员通知跳过 `eligible_user_service` 资格检查和静默期限制；
6. `feishu_card_builder.py`：删除 `mask_webhook_url` 函数；
7. `docker-compose.prod.yml`：删除 `ADMIN_FEISHU_APP_ID/APP_SECRET/RECEIVE_ID/RECEIVE_ID_TYPE` 环境变量；
8. `tools/pre_deploy_check.py`：仅检查 `DATABASE_URL`、`POSTGRES_PASSWORD`、`JWT_SECRET`、`REDIS_URL`、`SECRET_MASTER_KEY`，不再要求 `SECRET_KEY`、`FEISHU_APP_ID` 等；
9. 测试：新增 `test_beta_application_notifier.py`（11 个测试覆盖管理员通知链路）；更新 `test_feishu_platform_app_only.py` 验证运行时零 `ADMIN_FEISHU_*` 和零 Webhook；删除 `test_notification.py` 中 `mask_webhook_url` 相关测试；更新 `test_beta_application.py` 和 `test_admin_beta_applications.py` 事件类型；
10. 文档：更新 `docs/current/11-jobs-integrations.md` 和 `docs/current/18-code-doc-alignment.md`。
