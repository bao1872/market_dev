# CHANGE-20260709-008: API 路由 BaseRoute.path mypy baseline 债务清零

## 变更编号

CHANGE-20260709-008

## 任务名称

app/api/* + capture_main.py BaseRoute.path mypy baseline 20 个错误类型收窄修复

## 需求出处

mypy baseline 债务治理第二阶段：`app/api/*` 和 `app/capture_main.py` 有 20 个 `attr-defined "BaseRoute" has no attribute "path"` baseline 错误，分布于 15 个文件。

## 修改前行为

- `router.routes` 类型为 `list[BaseRoute]`，`BaseRoute` 没有 `path`/`methods` 属性；
- 15 个文件中直接 `r.path for r in router.routes` 触发 mypy `[attr-defined]` 错误；
- 部分文件使用 `getattr(r, "path", None)` 绕过（安全但不统一），部分文件使用 `isinstance(r, APIRoute)` 但 mypy 仍报错。

## 修改后行为

- 新增 `app/core/route_utils.py`：
  - `iter_api_routes(routes: list[BaseRoute]) -> Iterator[APIRoute]`：isinstance 收窄 helper；
  - `get_route_paths(routes: list[BaseRoute]) -> list[str]`：安全获取 path 列表；
- 15 个文件中的 `[r.path for r in router.routes]` 替换为 `get_route_paths(router.routes)`；
- `admin_after_close.py` 中需要 `r.path` + `r.methods` 的位置使用 `iter_api_routes` + `if r.methods` 过滤；
- `notifications.py` 中已有的 `isinstance(r, APIRoute)` 替换为 `get_route_paths`，统一风格；
- 不使用 `type: ignore` / `cast`；不改变 API 行为。
