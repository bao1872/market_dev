# CHANGE-20260704-026: 修复 indicators 旧 strategy_key 残留与 ALIGN-030 扫描证据

## 变更编号

CHANGE-20260704-026

## 任务名称

修复 indicators 旧 strategy_key 残留与 ALIGN-030 扫描证据

## 需求出处

生产部署（main HEAD `4a5278f`）后发现 `/api/v1/instruments/{id}/indicators` 仍会触发 `volume_node_monitor`/`bb_monitor` 策略不存在的 warning/error：

- `backend/app/services/indicator_service.py` 遍历 `StrategyLoader._registry` 计算指标；
- `_registry` 仍注册旧 key，但生产 DB 中已无对应 released `StrategyVersion`，manifests 目录也无对应 YAML；
- 每次请求都会记录策略计算失败日志并返回 `errors` 字段。

## 修改前行为

- `StrategyLoader._registry` 包含 `volume_node_monitor`、`bb_monitor`、`dsa_selector`、`watchlist_monitor`；
- `indicators` API 对旧 key 查询不到 released 版本，抛异常并记录 warning；
- 前端 `StrategyChart.tsx` 在提取 Volume Profile 数据时保留对 `volume_node_monitor` 的迁移期 fallback；
- `tests/verify_p0_fixes.py` 断言已不存在的 `bb_monitor.yaml`/`volume_node_monitor.yaml`；
- ALIGN-030 仅记录 TCL 科技单标回补，未执行全市场低覆盖扫描。

## 修改后行为

- `StrategyLoader._registry` 仅保留 `dsa_selector` 与 `watchlist_monitor`；
- `watchlist_monitor` 内部继续委托 `BollingerMonitor`/`VolumeNodeMonitor` 计算并输出 `bb`/`volume_node` chart layers；
- `indicators` API 不再遍历旧 key，日志不再报策略不存在；
- 前端 `StrategyChart.tsx` 仅通过 `watchlist_monitor` 读取 VP 数据；
- `verify_p0_fixes.py` 更新为断言 `dsa_selector.yaml`/`watchlist_monitor.yaml` 存在；
- `test_node_cluster_consistency.py` mock manifest 的 `strategy_id` 改为 `watchlist_monitor`；
- `indicator_service.py` 自测断言验证 `_registry` 仅含生产策略；
- ALIGN-030 更新为 PARTIAL/OPEN，附 2026-07-05 全市场只读扫描证据。
