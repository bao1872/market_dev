## 修改前行为

### SMC trailing 顺序
1. `smc_pine_core.py` 的 `_SMCPineState.run()` 中 `update_trailing_extremes` 位于 `display_structure` 之后（第6步）
2. Pine 源码 lines 766-807 中 `updateTrailingExtremes` 在任何 `getCurrentStructure` 之前执行（第1步）
3. 顺序颠倒导致：trailing 极值被当前 bar 的 high/low 二次覆盖，与 Pine 不一致；新 swing pivot 检测后 trailing.top/bottom 无法正确更新

### Pine 真源文件命名
1. 生产代码和文档引用 `ref/smc_ref.txt` 作为 Pine 真源路径
2. 用户上传文件命名为 `smc_user_source.pine`，更符合 Pine 源码命名约定

### Bug 2：sticky 列横向滚动重叠
1. `.sticky-col` 只有 `position: sticky; left: 0` 但无固定宽度
2. 长股票名称/代码撑开列宽，导致 sticky 列与后续列在横向滚动时重叠
3. 单元格内容溢出，无 `overflow: hidden` / `text-overflow: ellipsis`

1. `.table-meta-bar`（配置/列设置/清除/导出）和 `.table-pager`（分页器）无 sticky 定位

3. 用户横向滚动查看右侧列时无法操作工具栏

## 修改后行为

### 一、SMC trailing 执行顺序修复（`smc_pine_core.py`）
1. `_SMCPineState.run()` 方法中 `update_trailing_extremes` 移到循环体最前面（第1步），在任何 `get_current_structure` 之前
2. Pine 条件：`if showHighLowSwingsInput or showPremiumDiscountZonesInput`（`show_high_low_swings` 默认 true，`show_premium_discount_zones` 默认 false 不实现）
3. 执行顺序对齐 Pine lines 766-807：
   - 1. `update_trailing_extremes(i)` — trailing 极值更新（FIRST）
   - 2. `get_current_structure(i, swings_length=50, False, False)` — swing pivot
   - 3. `get_current_structure(i, 5, False, True)` — internal pivot
   - 4. `get_current_structure(i, equal_length=3, True, False)` — equal H/L pivot（if show_equal_hl）
   - 5. `display_structure(i, True)` — internal BOS/CHoCH + OB 创建（if show_internal_order_blocks）
   - 6. `display_structure(i, False)` — swing BOS/CHoCH + OB 创建
   - 7. `delete_order_blocks(i, True)` — internal OB mitigation（if show_internal_order_blocks）
   - 8. `delete_order_blocks(i, False)` — swing OB mitigation（if show_swing_order_blocks）
4. trailing 必须在最前面：Pine 中 `updateTrailingExtremes` 用当前 bar 的 high/low 更新 `trailing.top/bottom`，然后 `getCurrentStructure` 检测到新 swing pivot 时会覆盖 `trailing.top/bottom` 为新 pivot level。若顺序颠倒，trailing 会被当前 bar 的 high/low 二次覆盖
