## 五、应用层拆分清单（app/）

### 5.1 monitoring.py 拆分

当前 `app/monitoring.py` 是单体应用，混合了策略计算 + 通知 + 调度。按 V1.1 架构拆分：

| 当前功能 | V1.1 组件 | 改造方式 |
|---|---|---|
| `detect_bb_signals` | monitor 插件（BBMonitor） | 提取为独立 monitor |
| `detect_node_cluster_signals` | monitor 插件（VolumeNodeMonitor） | 提取为独立 monitor |
| 冷却逻辑（600s） | Monitoring Correlator | 移至组合状态机 |
| `generate_monitoring_card` | Message Builder | 移至消息构建服务 |
| `send_card_notification` | Channel Adapter | 移至飞书 Adapter |
| `start_scheduled_monitor` | Scheduler/Beat + Monitor Worker | 移至 worker |

### 5.2 feishu_notifier.py 改造

- **当前**：`FeishuNotifier` 类，支持 text/image/markdown/card/post
- **V1.1 改造**：改为 Channel Adapter，只负责格式转换+发送，不做业务逻辑
- **问题**：`scheduler.py::send_feishu_notification` 重复实现了 token 获取+发送，需统一使用 `FeishuNotifier`

### 5.3 scheduler.py 改造

- **当前**：APScheduler 单进程，4 个任务
- **V1.1 改造**：拆分为 Selector Worker / Monitor Worker / Composite Worker / Notification Worker
