## 修改前行为

K线最新 bar 紧贴右轴（plotRight），bars 占满整个绘图区宽度（plotW），没有右侧留白。用户无法直观看到最新 bar 后方的空间，视觉上最新 K 线被右轴截断。

## 修改后行为

- 引入 `RIGHT_PADDING_RATIO = 0.20` 常量（20% 留白，落在 18%-22% 要求区间内）。
- `drawTrading` 中 `step = effectivePlotW / display.length`，其中 `effectivePlotW = plotW * (1 - RIGHT_PADDING_RATIO)`。
- bars 只占据绘图区前 80%，最新 K 线位于绘图区约 80% 位置（即画面 78%-82% 区域）。
- 所有交互坐标映射统一使用 `step`，自动同步到压缩后的 bar 分布：
  - 十字线 mouse X → bar index
  - 滚轮缩放锚点
  - Pointer 拖拽 deltaBars
  - 双击复位（viewport 定义不变，渲染时自动留白）
  - 节点/事件命中坐标
- 网格线和十字线水平线仍延伸到 `g.plotRight`（保持全宽），只压缩 bar 分布区域。
- 时间轴标签使用 `effectivePlotW`（跟随 bar 分布，不延伸到留白区）。
- 不修改 Node/Profile/POC 算法、indicator_contract、盘中监控或 Capture 口径。
