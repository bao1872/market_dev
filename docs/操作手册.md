## 2. 调度任务配置

### 2.1 行情定时更新

- **触发时间**: 每个交易日（周一至周五）16:00
- **Worker 启动**: `WORKER_TYPE=bars_scheduler python -m app.worker`
- **调度框架**: APScheduler (AsyncIOScheduler + CronTrigger)
- **任务 ID**: `bars_refresh_daily`
- **拉取方式**: 串行（pytdx 不支持并发），通过 asyncio.to_thread 桥接
- **失败重试**: 最多 3 次，间隔 5 秒

### 2.2 拉取配置

**每日增量更新（DAILY_COUNTS，小 count，约 1.8 小时）**:

| 周期 | count |
|------|-------|
| d | 5 |
| 15m | 50 |
| 60m | 10 |

**历史回补（BACKFILL_COUNTS，大 count，约 11.1 小时）**:

| 周期 | count |
|------|-------|
| d | 500 |
| 15m | 15000 |
| 60m | 4000 |

**重试配置**: MAX_RETRIES=3, RETRY_DELAY=5秒

### 2.3 保留策略清理

- **触发时间**: 每日 02:00（避开交易时间）
- **调度方式**: APScheduler（在 bars_scheduler_service.run_retention_cleanup 中调用）
- **清理逻辑**: DELETE FROM ... WHERE trade_time < :cutoff（向量化删除）

## 6. 故障排查

### 6.1 行情数据不显示

1. 检查 `GET /api/v1/bars/health` 确认 DB/Redis 连通性
2. 检查对应周期的数据新鲜度（freshness_sla）
3. 若 DB 无数据，检查 pytdx 拉取日志（bars_fetch_total 指标）
4. 若 adj_factor=1.0，需重新拉取日线数据以触发复权因子计算

### 6.2 前复权价格异常

1. 检查 `bars_daily` 表中 adj_factor 是否为 1.0（默认值）
2. 若 adj_factor=1.0，执行 `POST /api/v1/admin/bars/refresh` 刷新日线
3. 验证 adj_factor 通过 Chanlunpro preclose 公式计算正确

### 6.3 定时任务未执行

1. 检查 worker 进程: `WORKER_TYPE=bars_scheduler python -m app.worker`
2. 检查 APScheduler 日志: `bars_refresh_daily` 任务是否注册
3. 检查交易日历: 非交易日不触发（is_trading_day_async 判断）
4. 手动触发: `POST /api/v1/admin/bars/refresh`

### 6.4 历史数据回补

1. 手动触发: `POST /api/v1/admin/bars/backfill?start_date=2023-01-01`
2. 耗时约 11.1 小时（全市场 8000+ 股票 × 5 周期）
3. 回补使用 BACKFILL_COUNTS（大 count）
4. 回补后执行对账验证数据完整性
