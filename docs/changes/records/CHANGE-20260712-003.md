## 2. 修改前理解

- 当前产品行为：快照归属在重跑时不更新 source_run_id；发布原子性在快照计算完成后提前写 succeeded/published_at；stock_context evidence 未映射到用户 DTO；前端事件面板默认隐藏且变量名语义混乱；MarketStockTable 名称列无 sticky；来源徽章从 market 进入却显示"自选监控"。
- 当前前端流程：StockDetailPage 使用 showStructuralState（默认隐藏）；EventStatePanel 未展示 state.evidence；MarketStockTable 内联 11px 字号。
- 当前后端流程：feature_snapshot_service upsert_snapshot 冲突时不更新 source_run_id；after_close_orchestrator 快照计算后即标记 succeeded；state_event_service 幂等键用日期+hash 而非 source_run_id；stock_context _event_to_dto 不映射 evidence；时区用 UTC 而非 Asia/Shanghai。
- 当前数据/API/Worker/第三方关系：StockFeatureSnapshot ORM 含冗余单列索引 ix_feature_snapshot_source_run_id（迁移中不存在）。
- 已阅读的当前设计文档：AGENTS.md、docs/current/MANIFEST.md、docs/current/02-data-api-contracts.md、docs/current/04-frontend-ux.md。
- 发现的代码—文档—生产冲突：ORM 索引与迁移不一致；幂等键设计与文档约束不一致；前端面板默认行为与用户期望不一致。

## 3. 修改目标

- 当前问题：快照归属错误、发布非原子、stock_context evidence 缺失、前端面板/表格/徽章行为错误。
- 修改后行为：
  1. upsert_snapshot 冲突更新 source_run_id；同日重跑归属新 run。
  2. 快照计算完成后不提前写 succeeded/published_at；DSA publish_run 成功后才发布。
  3. stock_context _event_to_dto 映射 evidence；时区用 Asia/Shanghai；cutoff 次日 00:00 exclusive；run 查询确定性倒序。
  4. 幂等键改为 symbol:source_run_id:algorithm_version。
  5. UserAppShell 删除角色预览；StockDetailPage eventPanelCollapsed 默认展开；EventStatePanel 展示 evidence + MACD；MarketStockTable 名称 sticky + 字号≥14px/辅助≥12px + 来源徽章修复。
  6. CORE_ONLY 部署不构建 worker-capture。
- 验收标准：pytest 全绿、tsc/eslint/build 通过、契约测试通过、Playwright 验收通过。
- 明确不修改：ConsensusZone 算法逻辑、Capture 路由、飞书投递、Migration 历史、DSA 计算逻辑。
