# CHANGE-20260708-050: 修复 frontend nginx 缓存 backend 旧 IP 导致 API 502

## 变更编号
CHANGE-20260708-050

## 任务名称
修复 frontend nginx 缓存 backend 旧 IP 导致 API 502（前端"数据加载失败，请刷新重试"）

## 需求出处
用户发现网页前端自选股监控页面提示"数据加载失败，请刷新重试"，要求暂停历史回补和性能开发，优先只读排查根因。

## 修改前行为
1. `frontend/nginx.conf` 在 `location /api/` 中使用静态 `proxy_pass http://backend:8000/`；
2. nginx 默认只在启动时解析一次 `backend` 主机名，将 IP 缓存到 worker 进程；
3. backend 容器重建（如 `docker compose build backend`、`--force-recreate`、部署升级）后，Docker 内部 DNS 给 `backend` 分配了新 IP；
4. frontend nginx 仍连接旧 IP，所有 `/api/*` 请求返回 502 Bad Gateway，前端 React Query 判定请求失败，WatchlistPage 显示"数据加载失败，请刷新重试"。

## 修改后行为
1. 在 `server` 块中声明 `resolver 127.0.0.11 valid=10s;`，使用 Docker 嵌入式 DNS 并限制缓存 TTL；
2. `/api/` location 改为使用变量：`set $backend_url backend:8000;` + `proxy_pass http://$backend_url;`；
3. 配合 `rewrite ^/api/(.*) /$1 break;` 正确剥离 `/api` 前缀（nginx 对变量形式的 proxy_pass 不会自动剥离 location 前缀）；
4. backend 容器重建后，frontend nginx 在 10s 内重新解析 `backend` 主机名，自动连接新 IP，/api 请求恢复。

## 风险
1. **Docker DNS 可用性**：`resolver 127.0.0.11` 依赖 Docker 嵌入式 DNS。若自定义网络配置禁用该 DNS，nginx 将无法解析 `backend`。生产当前使用默认 bridge/compose 网络，已验证可用；
2. **rewrite 规则误剥离**：`rewrite ^/api/(.*) /$1 break;` 与 `proxy_pass http://$backend_url` 配合，已将 `/api/watchlist/monitor-status` 正确转发为 `/watchlist/monitor-status`。若后续路径规则变化，需同步检查；
3. **resolver TTL**：`valid=10s` 在 backend IP 变化后最多 10s 内恢复，平衡了 DNS 查询开销与故障恢复速度；
4. **前端缓存**：index.html 已配置 `Cache-Control: no-store`，确保新 bundle 生效；assets 长期缓存不受影响。
