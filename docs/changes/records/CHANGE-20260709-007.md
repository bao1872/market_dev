# CHANGE-20260709-007: after_close_orchestrator mypy baseline 债务清零

## 变更编号

CHANGE-20260709-007

## 任务名称

after_close_orchestrator.py mypy baseline 22 个错误类型收窄修复

## 需求出处

mypy baseline 债务治理第一阶段：`after_close_orchestrator.py` 有 22 个 baseline 错误，全部源于 `db.get()` 返回 `T | None` 但调用方按非 Optional 使用。

## 修改前行为

- `execute_after_close_run` 中多处 `job_run = await db.get(SchedulerJobRun, job_run_id)` 返回 `SchedulerJobRun | None`，直接传给 `_update_orchestrator_status` / `_update_heartbeat_and_step`（参数要求 `SchedulerJobRun`）；
- `_get_or_create_job_run` 中 `acquire_job_run_lock` 返回 `tuple[SchedulerJobRun | None, bool]`，`is_new=True` 分支未显式收窄；
- `quality_gate` 阶段 `dsa_run = await db.get(StrategyRun, dsa_run_id)` 与前序 `create_batch_run` 返回的 `StrategyRun` 类型冲突；
- mypy baseline 记录 7 条 diagnostic、共 22 个错误。

## 修改后行为

- 新增 `_get_job_run_or_raise(db, job_run_id) -> SchedulerJobRun`：不存在时 raise RuntimeError；
- 新增 `_get_strategy_run_or_raise(db, run_id) -> StrategyRun`：不存在时 raise ValueError；
- `execute_after_close_run` 中所有裸 `db.get(SchedulerJobRun, ...)` 替换为 `_get_job_run_or_raise`；
- `_get_or_create_job_run` 在 `is_new=True` 后显式校验 `job_run is not None`；
- `quality_gate` 阶段 `dsa_run` 使用 `_get_strategy_run_or_raise`；
- 不使用 `cast` / `type: ignore`；不改变状态机语义、heartbeat/lease/repair 流程；
- mypy baseline 中 `after_close_orchestrator.py` 的 7 条 diagnostic 已删除。
