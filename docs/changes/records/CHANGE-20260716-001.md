# CHANGE-20260716-001

## 变更编号
CHANGE-20260716-001（合并原 001-007）

## 变更范围
盘后任务历史恢复 API + SMC 逐 Bar 对齐 Pine + MiniKline 真实留白 + indicator_service 按需加载 + 文档收口

## 主要修改

### 目标一：盘后任务历史恢复（after_close resume）
- **resume API 增强**（`admin_after_close.py`）：`SELECT FOR UPDATE` 行锁、同日 queued/running 互斥校验（409 SAME_DAY_ACTIVE_RUN）、幂等返回（queued 状态直接返回同一任务）、清 `worker_instance_id/heartbeat_at/lease_expires_at`、metadata 写 `resume_requested_at`、写唯一 `manual_resume` 事件
- **P0 修复**（`after_close_orchestrator.py`）：
  - P0-1：`repair_stale_after_close_snapshot_runs` 统计按 `source_run_id == snapshot_run.id`（禁止按 trade_date 聚合）
  - P0-2：DSA 未 published 时 repair 不标 succeeded；tracked run 返回 `resume_pending` 保持 running
  - P0-3：publishing 步骤 `snapshot_result=None` 时从 DB 读取实际 count，保留 expected/failed_count
  - P0-4：feature_snapshot 步骤复用已有 running snapshot run（不新建）
  - commit bug 修复：repair 调用后添加 `await db.commit()`
- **管理页**（`AdminAfterClosePipelinePage.tsx`）：交易日选择器（默认"最新"，可选历史日期如 2026-07-15）、"继续未完成任务"按钮（interrupted/failed 时显示）
- **测试**：新增 resume 幂等/同日互斥/manual_resume 事件/metadata 清空验证（3 个 endpoint 测试），保留原有 5 个 P0 测试

### 目标二：SMC 逐 Bar 对齐 Pine
- **核心语义修正**（`smc_pine_core.py`）：`ta.crossover/crossunder` 每 Bar 快照上一 Bar pivot level，结构更新后比较 `close[i]` 对当前 level 和 `close[i-1]` 对上一 level（禁止 current_level 传两次）
- **三个 leg() 独立状态**：swing50、internal5、equal3 各自持久状态，精确复现 `high[size]>ta.highest(size)` 首个合法检测
- **EQH/EQL DTO 三时间点**：anchor=前pivot，second_pivot=i-size，confirmed=i；线画到 second_pivot，因果时间用 confirmed；阈值用确认 Bar 的 `0.1*ATR200` 和严格 `<`
- **OB 统一字段**：`anchor_index/time`，slice end-exclusive，相同极值 first index，unshift 最多 100，新建 OB 参与同 Bar mitigation；前端只画数组头部最近 5 个 `internal&&!mitigated`
- **swing_bias 后端返回**：`swing_bias=self.swing_trend.bias`（1/-1/0），前端禁止从可见事件猜
- **view adapter**（`smc_view_adapter.py`）：核心用完整历史，API 只输出展示窗口有界 DTO 并统一重基准索引；跨左边界的 event/EQ/OB 保留并 `clipped_left=true`
- **纵轴范围**：主图纵轴加入可见 event.level、OB high/low、EQ level 和 trailing
- **缓存版本提升**：SMC/non-SMC 隔离，cache key 含算法版本
- **输入门禁**：API 返回 `smc_source_bar_hash/first_time/last_time/bars/adj`；输入不一致写 `INPUT_BAR_MISMATCH`
- **TV 导出 fixture**：`ref/smc_user_export.pine` 保留导出代码；无 TV CSV 时 `PINE_PARITY_PENDING`
- **FVG 彻底排除**：不计算、不返回、不缓存、不渲染、无 toggle

### 目标三：MiniKline 真实留白
- **visibleData 切片**（`miniKlineController.ts`/`miniKlineViewport.ts`）：每周期取最后 48/44/40/36/30 根，只将 visibleData 传给 series
- **setVisibleLogicalRange 真实留白**：setData 后 rAF 执行 `{from:-2, to:dataLength-1+3}`，禁止 fitContent/reset/scrollToRealTime
- **autoscale 基于 visibleData**：上 12%、下 15%；价格轴 minimumWidth=56，chart 固定 190px
- **tabs 五等分 grid**：card/chart 宽 100%，无多余 min-height/底部空白/横向 overflow
- **cleanup**：symbol/timeframe 变化立即清旧 data、取消旧 rAF；ResizeObserver 读取最新 refs

### 整体复核
- **detailSource 统一**（`detailSourceContext.ts`）：`normalizeResearchSource/defaultStrategyForSource` 唯一定义点，模块从此导入
- **stock sticky 列**：固定宽度，不透明背景，z-index 高于普通列
- **indicator_service 按需加载**：只加载当前周期和实际可用策略 required_inputs
