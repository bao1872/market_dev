### 2.7 图表渲染契约（Pine 标签与 regime 分段）

事实源：`backend/app/strategy/selectors/dsa_selector.py::compute_indicators` +
`frontend/src/components/StrategyChart.tsx`

个股详情页调用 `compute_indicators` 获取按 bar 对齐的 DSA 数据，字段如下：

| 字段 | 类型 | 说明 |
|------|------|------|
| `dsa_vwap` | list[float\|None] | 每 bar 的 DSA VWAP 值 |
| `dsa_dir` | list[int] | 每 bar 的方向（1/-1） |
| `regime_id` | list[int] | 每 bar 所属 regime 编号，dir 第一次切换时递增 |
| `anchor_time` | list[str\|None] | 每 bar 的锚点时间（仅锚点 bar 非空） |
| `pivot_type` | list[str\|None] | 每 bar 的 pivot 类型（`HH`/`HL`/`LH`/`LL`） |
| `pivot_price` | list[float\|None] | 每 bar 的 pivot 价格 |

前端渲染规则（SSOT）：

1. **regime 分段断线**：`renderIndicatorLine` 在 `regime_id` 变化处断开连线，不跨 regime 绘制 VWAP 线。
2. **Pine 标签**：在 `pivot_type` 非空的 bar 位置绘制文本标签；`HH`/`LH` 为波段高点，标签画在 K 线上方；`HL`/`LL` 为波段低点，画在下方。标签颜色与 A 股涨跌色对齐（高点用红色，低点用绿色）。
3. **禁止前端重算**：DSA 算法、Pine 标签坐标、regime 编号全部由后端 `compute_indicators` 生成，前端只负责布局和绘制。

## 3. Bollinger Band 监控

事实源：`backend/app/strategy/monitors/bollinger_monitor.py`

### 3.1 算法调用

调用 `features.bollinger_features_plotly.bollinger`（严格不修改）。

### 3.2 监控参数

| 常量 | 值 | 说明 |
|------|----|----|
| `BB_WIN_DEFAULT` | 20 | 布林带窗口 |
| `BB_K_DEFAULT` | 2.0 | 布林带标准差倍数 |
| `NOTIFY_COOLDOWN_SECONDS` | 600 | 冷却时间 10 分钟 |

### 3.3 事件类型

| event_type | 检测条件 | direction |
|-----------|---------|-----------|
| `bb_upper_touch` | `prev_close < ref_upper ≤ cur_close`（从下方穿越上轨） | negative |
| `bb_mid_touch` | `prev_close` 和 `cur_close` 分列中轨两侧 | neutral |
| `bb_lower_touch` | `prev_close > ref_lower ≥ cur_close`（从上方穿越下轨） | positive |

### 3.4 状态输出

- `bb_upper` / `bb_mid` / `bb_lower`：布林带参考线（取上一根已完成 bar）
- `current_price`：当前 1m 收盘价
- `prev_close`：上一根 1m 收盘价
- `bb_width`：带宽
- `bb_pos`：位置（0-1）

### 3.5 已完成 bar 判断

`_get_completed_bar_index`：
- 日线：最后一根 bar 是今天且未收盘（< 15:05）→ 取 `iloc[-2]`
- 否则取 `iloc[-1]`

## 10. MACD 指标（按 timeframe 后端真源）

事实源：`backend/app/services/indicator_service.py::compute_macd`

MACD 由后端 `indicator_service` 按当前 timeframe（`1d`/`15m`/`1h`/`1w`/`1mo`）真源计算，作为全局图层注入个股详情页指标响应。前端 `renderer=macd` 仅负责布局与绘制，禁止在 React/Canvas 中重新计算 EMA、DIF、DEA 或 histogram。

### 10.1 计算公式（A 股 2× 版本）

使用当前 timeframe 对应 bars 的 close 序列计算：

```text
DIF  = EMA(close, 12) - EMA(close, 26)
DEA  = EMA(DIF, 9)
MACD(Hist) = 2 * (DIF - DEA)
```

参数默认值：`fast=12`、`slow=26`、`signal=9`。

### 10.2 输出字段

| 字段 | 说明 |
|------|------|
| `macd_dif` | DIF 快线序列 |
| `macd_dea` | DEA 慢线序列 |
| `macd_hist` | MACD 柱序列（2× 版本） |

### 10.3 前端渲染契约

事实源：`frontend/src/components/StrategyChart.tsx::renderIndicatorMacd` +
`frontend/src/lib/strategy-manifest.ts`

1. **布局**：MACD 独占 `macd` pane，位于成交量 pane 上方，与价格/成交量共享同一个 X 轴可视范围。
2. **绘制内容**：
   - 零轴参考线；
   - 柱状图：`hist >= 0` 红色（涨），`hist < 0` 绿色（跌）；
   - DIF 快线（黄色 `#f4c430`）；
   - DEA 慢线（蓝色 `#2196f3`）。
3. **数据对齐**：指标数组取最后 `barsCount` 个值与 K 线 display 对齐，保证十字线索引一致。
4. **范围计算**：可见区域范围固定包含 0 轴，上下边界取 `max(abs(value)) * 1.08`。
