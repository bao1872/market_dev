# CHANGE-20260706-038: Swing Active State + Capture 布局 + Publish Auto-trigger

## 变更编号
CHANGE-20260706-038

## 任务名称
Swing Active State + Capture 布局修复 + 盘后 publish auto-trigger

## 需求出处
用户要求修复 swing 当前状态语义（confirmed raw >1 问题）、Temporal V1 位置污染、DSA age 不一致、capture 右侧空白，以及盘后流水线 publish 未自动触发

## 修改前行为
- swing_position 只有 confirmed pivot，强上涨段 close 突破 confirmed high 时 raw > 1（TCL 案例 1.997）
- temporal m15_position_relative_to_daily 用 confirmed raw 相减出现 -1.755 极端值
- DSA age_bars 与 current_dsa_segment_age_bars 不一致（差 2）
- capture=feishu 右侧空白（testid 在 tv-content，截图包含空右列）
- DSA scheduled 完成后不自动触发 after_close_orchestrator，导致 publish 未执行

## 修改后行为
- 新增 active swing 字段（clip [0,1]），反映当前正在发展的结构区间
- confirmed raw 保留（可能 <0 或 >1），用于突破状态判断
- temporal derived_relation 改用 active swing
- DSA age 统一为 +1 口径
- capture 模式隐藏所有按钮和侧列，testid 在 tv-chart-column
- worker.py 在 DSA scheduled+completed 后自动触发 after_close_orchestrator

## 风险
- active swing 计算依赖 confirmed pivot，fallback 模式（无 pivot）使用最近 120 根 bar 的 high/low，`active_swing_dir=None`；
- auto-trigger 失败不传播异常，需要监控日志（`logger.exception`）确保触发失败可见；
- 前端字段切换后，旧 confirmed 字段仍保留在 API 响应中（向后兼容），但 derived_relation 不再使用；
- capture 模式 CSS 修改 `.tv-side-column` 和 `.tv-chart-column` 可能影响非 capture 模式布局，已通过 contract test 保证非 capture 模式保持原双列布局。
