## 2. 修改前理解

- **当前产品行为**：Node Cluster（筹码分布/Volume Profile）由三链独立计算：
  - 盘后链 `feature_snapshot_service._compute_cost_position_factors` 调用 `compute_unified_volume_profile(bars)` 只传单一周期 bars
  - 详情链 `VolumeNodeMonitor.compute_indicators` 调用 `compute_unified_volume_profile(bars_daily, profile_df=bars_15min)` 正确传双周期
  - 监控链 `monitor_batch_service._compute_node_cluster_profile` 调用 `compute_unified_volume_profile(bars_daily, profile_df=bars_15min)` 正确传双周期
- **发现的缺陷**：盘后链只传单一周期 bars，导致 Profile 与详情链/监控链不一致（三链 hash 不一致）
- **ref/ 隔离缺陷**：
  - `ref/smc_user_export.pine` 仍被 git 跟踪（应只跟踪 `ref/smc_user_source.pine`）
  - 文档中多处把 `ref/` 文件称为"真源"/"视觉真源"/"运行依赖"（应改称"参考源（人工阅读）"）
  - 缺少 ref/ 隔离架构守护测试和文档文本扫描门禁
- **文档缺陷**：缺少指标计算合同文档（`docs/current/08`）和指标计算地图（`docs/maps/indicator-computation-map.md`）
- **已阅读的当前设计文档**：`docs/current/01-07`、`docs/maps/*`、`AGENTS.md`、`backend/app/constants/indicator_contract.py`、`backend/app/services/structural_factor_service.py`、`backend/app/services/feature_snapshot_service.py`、`backend/app/services/monitor_batch_service.py`、`backend/app/strategy/monitors/volume_node_monitor.py`、`backend/app/services/indicator_cache.py`
