## 修改前行为
1. **公开根路径 `/`**：由 React SPA 的 `LandingPage` 组件渲染，包含 HeroMarketDemo（动画市场演示）、OpportunityWorkflow（工作流动画）、PricingSection（定价表）、FeishuNotificationDemo（飞书通知演示）、LandingFooter、LegalModal、BetaApplicationModal（内测申请弹窗）等多个专用门户组件
2. **旧门户入口**：`LandingPage.tsx` 约 153 行，依赖 `landingData.ts` 数据、`useHeroMarketAnimation.ts`/`useWorkflowAnimation.ts` 两个动画 hooks、`LandingPage.module.scss` 423 行样式
3. **业务路由**：`/login`、`/market`、`/stock/:symbol`、`/capture/stock/:symbol`、`/messages`、`/settings`、`/admin/*`、`/api/*` 通过 SPA fallback `try_files $uri $uri/ /index.html` 处理
4. **缺陷**：旧 React 门户与业务 SPA 耦合在同一 React bundle，门户访问需要加载完整 SPA JS（约 800KB），且 BetaApplicationModal 等内测申请流程未通过管理员 active `feishu_platform_app` NotificationChannel（违反 AGENTS.md §七.6）

## 修改后行为

### 静态门户替换公开根路径
- `/`（精确匹配）由 Nginx 直接返回 `/portal/index.html`，**不进入 React SPA**，地址栏保持 `/`
- 门户为纯静态 HTML/CSS/JS（无 React），独立于业务 SPA bundle
- 业务路由 `/login`、`/market`、`/market?scope=watchlist`、`/stock/:symbol`、`/capture/stock/:symbol`、`/messages`、`/settings`、`/admin/*`、`/api/*` **全部保留原 SPA fallback 行为**，不抢占

### 门户页面清单（11 个 HTML，PRD §5.1）
- `index.html`（使用说明首页）
- `pages/quick-start.html`（快速开始）
- `pages/market.html`（行情查看）
- `pages/watchlist.html`（自选管理）
- `pages/stock-detail.html`（个股详情）
- `pages/alerts.html`（提醒记录）
- `pages/messages.html`（消息通知）
- `pages/data.html`（指标原理）
- `pages/faq.html`（常见问题）
- `pages/boundaries.html`（使用边界）
- `pages/customization.html`（量化定制）

### 门户静态资源
- `assets/css/site.css`（门户样式）
- `assets/js/site.js`（门户交互：搜索、菜单、FAQ 折叠、定制摘要生成）
- `assets/data/factors.public.json` + `assets/data/factors.public.js`（公开因子目录：240 个，verified_current_core=24，extended_catalog=216，15 个 categories，ID 唯一）
- `assets/images/logo_symbol_128.png` + `assets/images/wechat-qr-placeholder.svg`
- `content/site.json`（站点配置）

### LandingPage 兜底
- `LandingPage.tsx` 重写为 SPA 内部导航兜底：mount 后 `window.location.replace('/')`，让浏览器重新请求 `/` 由 Nginx 返回静态门户首页
- 不使用 react-router `Navigate`（Navigate 只在 SPA 内切换，无法离开 SPA 进入静态 HTML）

### Nginx 路由配置
- `location = /` 精确匹配 → `try_files /portal/index.html =404`（优先级高于 `location /` 前缀，因此只有精确 `/` 走门户）
- `location = /portal/index.html` → no-cache
- `location ~ ^/portal/pages/.*\.html$` → no-cache
- `location /portal/` → 静态资源正常访问
- `/api/`、`/api/v1/health`、`/index.html`、`/assets/`、`location /`（SPA fallback）、`resolver`、WebSocket headers 全部保留
