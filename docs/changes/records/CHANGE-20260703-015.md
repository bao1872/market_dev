## 当前问题

4. `docs/current/*.md` 和 `docs/README.md` 的实现核对基线不统一，无机器可解析格式。

## 当前设计理解

- `outbox_relay._expand_notification_message_created` 是 `notification.message.created` 事件的扩张函数；
- ddca659 hotfix 修改该函数：payload 含 `target_channel_id` 时跳过 `eligible_user_service`，只扩张到指定渠道；无 `target_channel_id` 的自动通知仍走资格过滤；
- `target_channel_id` 场景用于用户主动触发/手动指定渠道的通知（如 `stock_detail_share`），不应受订阅状态限制；
- 无 `target_channel_id` 的自动通知仍必须过滤资格，避免 disabled/expired 用户被自动通知打扰，admin 不进入 universe；
- 管理员内测申请通知走专用事件 `beta_application.admin_notification.created`，不进入本函数；
- 当前飞书唯一接入方式为 `feishu_platform_app`，`feishu_webhook` 已由 migration 055 CHECK 约束永久禁止。
