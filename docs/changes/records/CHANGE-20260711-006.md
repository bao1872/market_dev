# CHANGE-20260711-006: PRD V1.1 纠偏 — 路由恢复 + EventStatePanel + StockState 字段剥离 + state 筛选

## 变更编号
CHANGE-20260711-006

## 任务名称
PRD V1.1 纠偏：恢复 /stock/:symbol 为唯一个股详情入口；EventStatePanel 替代 ResearchContextPanel；StockState/StateEvent 用户 DTO 剥离 sourceField/idempotencyKey；market state 筛选实现

## 需求出处
PR #74 纠偏指令（ref/PROMPT.md）：PRD V1.1 明确规定 /stock/:symbol 是唯一个股详情和 K线入口，不得重定向到 /market。

## 修改前行为
1. `/stock/:symbol` 被错误重定向到 `/market?symbol=...`（StockDetailRedirect 组件），违反 PRD V1.1。
2. 前端使用 `ResearchContextPanel`（含 EventExplanationCard/StructureSummaryCard/KeyRangeCard/LatestEventCard 等多个子组件），与 PRD V1.1 EventStatePanel 要求不符。
3. `StateValue.sourceField` 为必填字段（`Field(...)`），用户接口返回内部来源字段名。
4. `StateEventDTO.idempotencyKey` 为必填字段（`Field(...)`），用户接口返回内部幂等键。
5. market API `state` 参数未实现，传任意值返回 422。
6. 权限测试使用 `@pytest.mark.skipif` 跳过，无真实 HTTP 集成测试。
7. `state_event_service._batch_get_previous_snapshots` 使用 PostgreSQL 特有的 `DISTINCT ON`。
8. `feature_snapshot_service.upsert_snapshot` 的 ON CONFLICT DO UPDATE 会覆盖 `source_run_id`。

## 修改后行为
1. `/stock/:symbol` 恢复为 `StockDetailPage` 路由（删除 StockDetailRedirect），是唯一个股详情和 K线入口。
2. 新建 `EventStatePanel` 组件（`frontend/src/features/research-context/EventStatePanel.tsx`），market 和 detail 共用 `useStockContext` query key；删除 12 个旧 orphan 文件。
3. `StateValue.sourceField` 改为 `Optional`，默认 `None`；用户接口通过 `strip_internal_fields_for_user()` 剥离。
4. `StateEventDTO.idempotencyKey` 改为 `Optional`，默认 `None`；用户接口通过 `strip_internal_fields_for_user()` 剥离。
5. market API `state` 参数已实现（Phase 4）：合法值 `up`/`down`/`sideways`，对应 `daily_developing_swing_dir` > 0 / < 0 / == 0。
6. 权限测试改为 7 个真实 HTTP 集成测试（`httpx.AsyncClient` + `ASGITransport` + `dependency_overrides`），0 skip。
7. `_batch_get_previous_snapshots` 改用 `MAX(trade_date)` 子查询 + JOIN（标准 SQL，跨方言兼容）。
8. `upsert_snapshot` 的 `update_cols` 移除 `source_run_id`，冲突时不覆盖已发布快照的来源批次。
