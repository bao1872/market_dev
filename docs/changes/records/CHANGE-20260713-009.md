# CHANGE-20260713-009: 详情页来源上下文修复（P0）

## 变更编号
CHANGE-20260713-009

## 任务名称
P0：从 /market 进入个股详情后左侧来源列表错误显示自选股

## 需求出处
用户指令（2026-07-13）：从 /market 的"行情"进入个股详情后，左侧来源列表错误显示自选股。先修复该问题，再完成原任务收口。

## 修改前行为
1. MarketWorkspacePage 点击股票名称进入详情只传 returnTo，没有传 source/strategy
2. StockDetailPage source 缺失时默认 watchlist
3. parseMarketParamsFromReturnTo 只有存在 keyword/page/sort 才认为是 market context，纯 `/market?scope=market` 或只含 industry/concept 时返回 null
4. null 后 sourceListKind 回退 watchlist，左侧列表显示自选股
5. 即使识别为 market，详情左栏仍调用旧 useMarketStocks，与当前 published DSA results 不一致
6. normalizeInternalReturnTo 长度限制 200 字符，/market URL 含 filters JSON 编码后可能超过 200 被拒绝

## 修改后行为
1. MarketWorkspacePage handleNavigateToStock 根据 scope 明确传递 source/strategy：
   - scope=market → source=selection&strategy=dsa_selector
   - scope=watchlist → source=watchlist&strategy=watchlist_monitor
2. returnTo 保存完整当前 URL（scope/selected/keyword/industry/concept/filters/sort/dir/page/page_size）
3. 新增共享纯函数 decodeMarketListContext：任意合法 /market URL 都识别为 market context（不要求 keyword/page/sort 存在），scope=market 和 scope=watchlist 都解析，仅含 industry/concept 的 URL 也有效
4. 新增共享纯函数 buildStrategyResultQueryParams：将 MarketListContext 转换为 StrategyResultQueryParams（scope=market→universe=all，scope=watchlist→universe=watchlist，含 keyword/industry/concept/sort/metric_filters/page/page_size 完整转换）
5. useStockDetailActions 从 useMarketStocks 切换到 usePublishedRuns + useStrategyRunResults + adaptStrategyResultToTrendRow + getStockDisplay 链
6. sourceListKind 由 marketContext.scope 决定（market→"行情来源"，watchlist→"自选来源"）
7. sourceBadge 优先 sourceListKind=market→"行情来源"，否则 source=selection→"选股结果"，source=watchlist→"自选来源"
8. 左栏标题统一为"行情来源"/"自选来源"（不再显示"来源: 行情搜索"）
9. 上一只/下一只基于恢复后的同一页 DSA results
10. normalizeInternalReturnTo 长度限制从 200 提升到 500（/market URL 含 filters JSON 编码后可能超过 200 字符）
11. MarketWorkspacePage 和 useStockDetailActions 共用 decodeMarketListContext + buildStrategyResultQueryParams 避免筛选口径漂移
