## 当前问题

1. `CaptureStockPage.tsx` 仍分别调用 `/instruments/by-symbol/{symbol}`、`/api/v1/instruments/{id}/bars`、`/api/v1/instruments/{id}/indicators`、`/api/v1/instruments/{id}/quote` 四个普通端点，未真正使用 Snapshot API。
2. `stock_capture_service.py` 截图 URL 未携带 `instrument_id`，前端无法从 URL 读取并调用 Snapshot API。
3. `get_share_status` 返回字段缺少 `capture_status` / `image_upload_status`，`ShareStatusResponse.overall_status` 枚举未包含 `partial_failed`。
4. ALIGN-018 被错误标为 CLOSED，实际前端未完全切换到 Snapshot 链路。
5. 旧 `type-check` CI 任务因全仓库 252 个历史 mypy 错误直接阻断合并；本次新增 mypy 错误 `app/api/stock_detail_feishu.py:265` 与 `app/repositories/bar_repository.py` 重复定义需修复。

## 当前设计理解

- 专用 Capture 链路：`Capture Worker → /capture/stock/:symbol → CaptureStockPage → captureClient → GET /api/v1/capture/stocks/{instrument_id}/snapshot`。
- Capture Token 只能访问 `/api/v1/capture/*`，普通 access token 不能访问 Capture API。
- Snapshot API 一次返回 instrument/bars/indicators/events/quote 等截图所需完整数据。
- 图文状态机终态：`success`、`partial_failed`、`failed`；`partial_failed` 下支持仅重试图片。

## 实际修改内容

1. `CaptureStockPage.tsx` 改为仅通过 `captureClient` 调用 `GET /api/v1/capture/stocks/{instrument_id}/snapshot`，删除对普通端点的调用。
2. `stock_capture_service.py` 截图 URL 增加 `instrument_id={instrument_id}` 参数。
3. `stock_detail_feishu_service.py` / `stock_detail_feishu.py` 补齐 `capture_status` / `image_upload_status` / `overall_status`（含 `partial_failed`）状态机字段。
4. 重构 `CaptureStockPage.tsx` 与 `StockDetailPage.tsx` 的重复工具函数：提取 `MARKET_LABELS` / `formatAmount` 到 `src/utils/market.ts`，提取 `bars → BarData` 转换到 `src/utils/chart.ts`。
5. 修复本次新增 mypy 错误：`stock_detail_feishu.py` 自测代码使用 `getattr(route, "path", None)` 安全取 path；`bar_repository.py` 删除第 1232 行重复定义的 `_query_minute_bars`。
6. CI 替换旧 `type-check` 为三层 Mypy 门禁：`Mypy New Files`（阻断）、`Mypy Baseline Regression`（阻断）、`Mypy Full Repository Report`（非阻断上传 `mypy-full-report` artifact）。
7. 新增 mypy 工具脚本：`generate_mypy_baseline.py`、`compare_mypy_baseline.py`、`check_mypy_new_files.py`；新增基线文件 `tools/quality_baselines/mypy.json`（基线 commit `64ed75cce80f5b3f2b5ab95f96b52aac11475e3e`，mypy 2.1.0，诊断总数 242，唯一诊断 175）。
8. `backend/pyproject.toml` 将 `mypy` 从 `>=1.8` 固定为 `==2.1.0`，避免 CI mypy 版本波动导致基线比较失效；为兼容 mypy 2.1.0 的 stub 解析，将 `numpy` 从 `>=1.26` 收紧为 `>=1.26,<2.5.0`，防止 numpy 2.5.0+ 的 stub 在 mypy 2.1.0 下产生虚假的 site-packages 语法错误并中断分析。
9. `.github/workflows/ci.yml`：`Mypy New Files` 与 `Mypy Baseline Regression` 的 `Run mypy on backend` 步骤使用 `set +e` 捕获报告，避免 mypy 因历史错误返回非零退出码而跳过后续基线/新文件检查。
10. 同步更新 `AGENTS.md`、`docs/current/14-deployment-operations.md`、`docs/current/15-testing-acceptance.md`、`docs/current/18-code-doc-alignment.md` 与 `docs/changes/CHANGELOG.md`。
