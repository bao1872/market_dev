# CHANGE-20260707-045: 修复 MDAS live 1m 时区不一致导致 monitor 无事件

## 变更编号
CHANGE-20260707-045

## 任务名称
修复 `MarketDataAggregationService` 拉取 live 1m 时 `live_start`/`live_end` 时区不一致，恢复盘中监控与个股详情 1d partial bar。

## 需求出处
生产诊断：2026-07-07 A 股交易时段，用户未收到盘中监控飞书消息。

## 修改前行为
- `MarketDataAggregationService.get_bars()` 在交易时段拉取 live 1m 时构造：
  - `live_start = datetime.combine(now.date(), time(9, 30))` → naive datetime
  - `live_end = now` → aware Asia/Shanghai datetime
- 两者传入 `fetch_minute_bars()` → `pytdx_adapter.get_minute_bars()` 后，第 609 行 `(end - start).total_seconds()` 触发：
  - `can't subtract offset-naive and offset-aware datetimes`
- 导致：
  - `worker-monitor` 每个周期对全部 19 个自选股 1m 拉取失败；
  - `monitor_evaluations` / `monitor_states` / `strategy_events` 全天为 0；
  - `notification_messages` / `outbox` / `message_deliveries` 全天无 monitor 相关记录；
  - 个股详情 `/bars?timeframe=1d&include_realtime=true` 在交易时段也无法合成 partial daily bar。

## 修改后行为
- 新增统一时区处理：
  ```python
  now_cst = now if now.tzinfo else now.replace(tzinfo=SHANGHAI_TZ)
  live_start = now_cst.replace(hour=9, minute=30, second=0, microsecond=0)
  live_end = now_cst
  ```
- `live_start` 与 `live_end` 同为 `Asia/Shanghai` aware datetime，可正常相减；
- 修复两处：
  1. 日内周期（含 1m）实时分支；
  2. 1d partial daily bar 合成分支；
- 个股详情 K线实时与盘中监控触发仍保持为两条独立业务链路，仅共同依赖 MDAS live 1m 能力。
