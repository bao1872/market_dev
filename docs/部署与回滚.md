# 部署与回滚

## 1. 环境准备

### 1.1 外部 PostgreSQL（物理机）

- **版本**：PostgreSQL 15+
- **角色**：`bz`（业务读写）+ `postgres`（管理员）
- **数据库**：`bz_stock`（生产）+ `bz_stock_test`（测试）
- **监听**：`127.0.0.1:5432`
- **测试库容器**：`trading-postgres-test`（端口 5433）

**创建业务角色**（首次部署）：
```sql
-- 使用 postgres 管理员执行
CREATE ROLE bz WITH LOGIN PASSWORD 'OZDFLwrMZIfpPrJjaYxnaAoRDv5TQkJNtLlD3ENN5c';
CREATE DATABASE bz_stock OWNER bz;
CREATE DATABASE bz_stock_test OWNER bz;
GRANT ALL PRIVILEGES ON DATABASE bz_stock TO bz;
GRANT ALL PRIVILEGES ON DATABASE bz_stock_test TO bz;
```

### 1.2 外部 env 文件

生产环境 env 文件位置：`/etc/market-dev/market.env`

**必填变量**：
```bash
DATABASE_URL=postgresql+psycopg://bz:OZDFLwrMZIfpPrJjaYxnaAoRDv5TQkJNtLlD3ENN5c@127.0.0.1:5432/bz_stock
JWT_SECRET=<strong-random-secret>
SECRET_MASTER_KEY=<strong-random-secret>
```

**可选变量**：
```bash

JWT_REFRESH_TTL_SECONDS=604800

```

### 1.3 Docker Compose 文件

- **开发环境**：`docker-compose.yml`（仅 Redis 容器，PostgreSQL 用物理机）
- **生产环境**：`docker-compose.prod.yml`（包含全部服务）

### 1.4 生产环境配置挂载（config.production.py）

生产环境通过 `CONFIG_FILE` 环境变量指定额外配置文件，并挂载宿主机上的 `/etc/market-dev/config.production.py`：

```bash
sudo mkdir -p /etc/market-dev
sudo tee /etc/market-dev/config.production.py <<'EOF'
# 生产环境专属配置，覆盖代码默认项
# 例如：日志级别、外部服务地址、业务阈值等
EOF
sudo chmod 600 /etc/market-dev/config.production.py
```

所有后端与 Worker 服务统一设置：

```yaml
environment:
  CONFIG_FILE: /app/app/config.production.py
volumes:
  - /etc/market-dev/config.production.py:/app/app/config.production.py:ro
```

> 若 `/etc/market-dev/config.production.py` 不存在，需先创建空文件或移除对应挂载，否则容器启动会因挂载失败而报错。

## 2. 首次部署

### 2.1 克隆代码

```bash
cd /root
git clone <repo-url> web_dev
cd web_dev
```

### 2.2 配置 env 文件

```bash
sudo mkdir -p /etc/market-dev
sudo tee /etc/market-dev/market.env <<EOF
DATABASE_URL=postgresql+psycopg://bz:***@127.0.0.1:5432/bz_stock
JWT_SECRET=***
SECRET_MASTER_KEY=***
APP_ENV=production
EOF
sudo chmod 600 /etc/market-dev/market.env
```

### 2.3 执行数据库迁移

```bash
cd /root/web_dev/backend
source .venv/bin/activate
export DATABASE_URL="postgresql+psycopg://bz:***@127.0.0.1:5432/bz_stock"
alembic upgrade head
```

### 2.4 构建并启动生产服务

```bash
cd /root/web_dev
GIT_SHA=$(git rev-parse --short HEAD) \
BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ") \
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml build

docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d --force-recreate --remove-orphans

# 日常发布不再清理 Docker 镜像/构建缓存，避免误删相邻版本镜像或导致回滚时重新构建。
# 清理操作仅在磁盘告警或每周按需执行：./scripts/cleanup-docker.sh
```

## 3. 发布流程

### 3.1 拉取最新代码

```bash
cd /root/web_dev
git pull origin main
```

### 3.2 执行数据库迁移（如有）

```bash
cd /root/web_dev/backend
source .venv/bin/activate
export DATABASE_URL="postgresql+psycopg://bz:***@127.0.0.1:5432/bz_stock"
alembic upgrade head
```

### 3.3 重建受影响的服务

```bash
cd /root/web_dev
GIT_SHA=$(git rev-parse --short HEAD) \
BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ") \
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml build

# 仅重启受影响的服务（如 backend 改动）
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d --force-recreate --remove-orphans backend

# 如 Worker 改动，重启所有 worker
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d --force-recreate --remove-orphans \
  worker-bars-scheduler worker-strategy-scheduler worker-calendar \
  worker-monitor worker-strategy-batch worker-outbox worker-delivery \
  worker-after-close worker-capture

# 如前端改动
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d --force-recreate --remove-orphans frontend
```

### 3.4 镜像复用与 --no-build 部署

若当前 `GIT_SHA` 对应的镜像已存在且无需重新构建（仅配置或 env 变更），可直接复用镜像：

```bash
cd /root/web_dev
GIT_SHA=$(git rev-parse --short HEAD)

# 校验镜像是否存在
if docker image inspect "market-dev-backend:${GIT_SHA}" >/dev/null 2>&1; then
  echo "镜像已存在，跳过构建"
  docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d --no-build --force-recreate --remove-orphans
else
  echo "镜像不存在，执行构建"
  GIT_SHA=$(git rev-parse --short HEAD) \
  BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ") \
  docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml build
  docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d --force-recreate --remove-orphans
fi
```

> 日常发布不再默认执行 `docker image prune` / `docker builder prune`，避免误删相邻版本镜像或导致回滚时重新构建。清理操作仅在磁盘告警时按需执行（见第 7 节）。

### 3.5 验证发布

```bash
# 健康检查
curl http://localhost:8000/health/ready
curl http://localhost:8000/version | python3 -m json.tool

# 验证 git_sha 已更新
GIT_SHA=$(git rev-parse --short HEAD)
VERSION_SHA=$(curl -sS http://localhost:8000/version | python3 -c "import sys,json; print(json.load(sys.stdin)['git_sha'])")
[ "$GIT_SHA" = "$VERSION_SHA" ] && echo "OK: git_sha 一致" || echo "FAIL: git_sha 不一致"
```

## 4. 回滚流程

### 4.1 代码回滚

```bash
cd /root/web_dev
# 查看最近 5 次提交
git log --oneline -5

# 回滚到指定 commit
git reset --hard <commit-sha>
```

### 4.2 数据库迁移回滚

```bash
cd /root/web_dev/backend
source .venv/bin/activate
export DATABASE_URL="postgresql+psycopg://bz:***@127.0.0.1:5432/bz_stock"

# 回滚一个版本
alembic downgrade -1

# 回滚到指定版本
alembic downgrade <revision-id>

# 查看当前版本
alembic current
```

> **注意**：数据库回滚可能造成数据丢失，仅在迁移有 down 方法时可用。生产回滚前必须先备份。

### 4.3 服务回滚

```bash
cd /root/web_dev
GIT_SHA=$(git rev-parse --short HEAD) \
BUILD_TIME=$(date -u +"%Y-%m-%dT%H:%M:%SZ") \
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml build

docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d --force-recreate --remove-orphans
```

> 回滚时优先使用 `--no-build` 复用旧版本镜像；仅当镜像缺失时才重新构建，避免误删可用于回滚的相邻版本镜像。

### 4.4 数据库备份与恢复

```bash
# 备份
PGPASSWORD=*** pg_dump -U bz -h 127.0.0.1 -d bz_stock -F c -f /backup/bz_stock_$(date +%Y%m%d_%H%M%S).dump

# 恢复
PGPASSWORD=*** pg_restore -U bz -h 127.0.0.1 -d bz_stock -c /backup/bz_stock_YYYYMMDD_HHMMSS.dump
```

## 7. 磁盘清理

### 7.1 镜像堆积原因

每次 `docker compose build` 会创建新镜像，旧镜像变为 `<none>` 标签（悬空镜像）但不会自动删除。多次部署后镜像和 build cache 会大量堆积，导致磁盘占用持续上升。

### 7.2 清理命令

```bash
# 清理悬空镜像（<none> 标签的旧镜像）
docker image prune -f

# 清理 build cache（构建缓存）
docker builder prune -f

# 查看磁盘占用
docker system df
df -h /
```

### 7.3 部署后自动清理

建议在部署脚本中加入自动清理：
```bash
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml build
docker image prune -f  # 部署后立即清理悬空镜像
docker compose --env-file /etc/market-dev/market.env -f docker-compose.prod.yml up -d --force-recreate
```

### 7.4 定期清理

建议每周清理一次 build cache（crontab）：
```bash
# 每周日凌晨 3 点清理 build cache
0 3 * * 0 docker builder prune -f > /dev/null 2>&1
```

### 7.5 监控指标

- `/var/lib/docker` 目录大小：`du -sh /var/lib/docker`
- 可回收空间：`docker system df` 的 RECLAIMABLE 列
- 磁盘使用率：`df -h /`（目标 < 75%）
