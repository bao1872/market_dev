## 修改前行为

- feature_snapshot 步骤由 `compute_for_trade_date` 内部 upsert（flush）+ caller 显式控制 commit/rollback；
- progress 由前端用 `Date.now()` 估算历史速度与 ETA，存在客户端伪造实时风险；
- loader 存在逐股/逐周期查询 MDAS/pytdx 路径，生产全市场存在外部行情触达与性能不确定性；
- 缺旧/新路径等价性、compute 只读、write 原子性、批量查询边界的专项回归（仅存量 service/backfill/orchestrator 用例）。

## 修改后行为

- **两阶段 compute/write**：`compute_records_for_trade_date` 只读 SELECT + 纯函数计算 → records（不 upsert/flush/commit）；`bulk_upsert_records`（每批 100 flush）+ `finish_snapshot_run(published_at)` 同一 `db` session 仅一次 commit；任一步失败整体 rollback，旧 `published` run 下无新旧混合；
- **compute 阶段守卫**：monkeypatch `upsert_snapshot`/`bulk_upsert_records` 调即失败，验证 compute 不写库；
- **progress 全后端计算**：`speed_per_minute`/`eta_seconds` 由后端基于 `started_at`/`updated_at` 计算并随 progress 上报；前端 `AdminAfterClosePipelinePage` 只展示，移除 `Date.now()` 估算与 `running`/`effectiveStart` 等未使用变量；
- **loader DB-only 批量查询**：daily/15min/adj-factor 一次批量拉全市场，调用次数常数（不逐股 pytdx/MDAS）；生产默认单进程、无持久缓存/新表/索引/migration；RSS 门禁 `_FEATURE_SNAPSHOT_MAX_RSS_MB=1800`；
- **关键链路专项测试** `test_feature_snapshot_consistency.py`（9 用例，冻结业务时钟 `TRADE_DATE=2026-06-25`、禁用 pytdx/Redis cache）：① 旧/新 10 股 × 4 场景逐字段深比较等价；② compute 只读外部不可见；③ write 成功可见/失败 rollback 旧 published 仍完整；④ loader 批量边界（20 股=3 次、40 股=6 次）+ 50 股 compute-only smoke（RSS<1800MB 无失控增长）。
