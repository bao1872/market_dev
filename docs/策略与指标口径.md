## 3. Bollinger Band 监控

事实源：`backend/app/strategy/monitors/bollinger_monitor.py`

### 3.1 算法调用

调用 `features.bollinger_features_plotly.bollinger`（严格不修改）。

### 3.2 监控参数

| 常量 | 值 | 说明 |
|------|----|----|
| `BB_WIN_DEFAULT` | 20 | 布林带窗口 |
| `BB_K_DEFAULT` | 2.0 | 布林带标准差倍数 |
| `NOTIFY_COOLDOWN_SECONDS` | 600 | 冷却时间 10 分钟 |

### 3.3 事件类型

| event_type | 检测条件 | direction |
|-----------|---------|-----------|
| `bb_upper_touch` | `prev_close < ref_upper ≤ cur_close`（从下方穿越上轨） | negative |
| `bb_mid_touch` | `prev_close` 和 `cur_close` 分列中轨两侧 | neutral |
| `bb_lower_touch` | `prev_close > ref_lower ≥ cur_close`（从上方穿越下轨） | positive |

### 3.4 状态输出

- `bb_upper` / `bb_mid` / `bb_lower`：布林带参考线（取上一根已完成 bar）
- `current_price`：当前 1m 收盘价
- `prev_close`：上一根 1m 收盘价
- `bb_width`：带宽
- `bb_pos`：位置（0-1）

### 3.5 已完成 bar 判断

`_get_completed_bar_index`：
- 日线：最后一根 bar 是今天且未收盘（< 15:05）→ 取 `iloc[-2]`
- 否则取 `iloc[-1]`

## 6. StrategyEvent 契约

事实源：`backend/app/models/strategy_event.py` + `strategy_assets/schemas/strategy_event.schema.json`

### 6.1 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键（DB 生成） |
| `event_key` | Text (UNIQUE) | 事件唯一键（幂等去重） |
| `strategy_version_id` | UUID FK | 策略版本 |
| `instrument_id` | UUID | 股票 ID |
| `event_type` | Text | 事件类型（如 `bb_upper_touch`） |
| `event_time` | DateTime(tz) | 事件发生时间（按 bar 时间） |

| `schema_version` | Integer | envelope schema 版本 |
| `payload` | JSONB | 事件负载（自包含，不依赖外部状态） |
| `snapshot` | JSONB | 事件发生时上下文快照（bars/state/metrics 冻结） |
| `created_at` | DateTime(tz) | 创建时间 |

### 6.2 幂等保证

- `event_key` UNIQUE 约束：相同 event_key 的事件不重复写入（`ON CONFLICT DO NOTHING`）
- `dedupe_key` 构建：`build_dedupe_key(strategy_version_id, instrument_id, idx, event_type)`

## 7. MonitorEvaluation 契约

事实源：`backend/app/models/monitor_evaluation.py`

- **exactly-once 语义**：`(策略版本, 股票, bar时间)` 唯一，`INSERT ON CONFLICT DO NOTHING`
- **租约机制**：`lease_owner` + `lease_expires_at`，默认 300 秒
- **重试退避**：`30 * 2^retry_count` 秒，最大 5 次
- **恢复机制**：Worker 启动时调用 `recover_stale_evaluations` 恢复过期租约

## 9. 数据就绪检查

事实源：`backend/app/services/strategy_batch_service.py`

```python
DATA_COVERAGE_THRESHOLD = 0.9          # 数据就绪检查覆盖率阈值
AUTO_PUBLISH_COVERAGE_THRESHOLD = 0.8   # 自动发布质量门禁
_STRATEGY_BATCH_DAILY_LOOKBACK_DAYS = 5000  # 策略批量计算日线回看天数
```

**门禁条件**（自动发布）：
1. 状态必须为 `completed`（非 `partial_failed`/`failed`）
2. 数据覆盖率 `succeeded_count / total_instruments >= 80%`
3. 无致命错误（`failed_count == 0`）
