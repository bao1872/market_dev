# CHANGE-20260715-006

## 变更编号
CHANGE-20260715-006

## 任务名称
MiniKline 闭包根治 + SMC Pine 对齐（RMA NA 语义 + 首个 pivot off-by-one + EQH/EQL 三时间点）

## 需求出处
用户要求：1) 根治 MiniKline ResizeObserver/rAF 闭包问题（mount 时创建的回调捕获首次 render 的 stale bars/timeframe）；2) 修正 `pine_rma` 严格复现 Pine `ta.rma` 的 NA 和 SMA 种子语义；3) 核对 `i==size` 首个 leg/pivot（旧代码 `i > size` 延迟了一根）；4) EQH/EQL DTO 分离三时间点（anchor/confirmed/detection）；5) 再核对 ATR200、highest/lowest 窗口、crossover/crossunder NA、OB slice end-exclusive、trailing 顺序；6) Golden fixture 未提供时保留 `PINE_OUTPUT_GOLDEN_PENDING`；7) SMC 仍仅进入 `/stock`，默认关闭。

## 修改前行为
1. `MiniKlineCard` 的 `ResizeObserver` 在 mount 时创建一次，回调直接闭包捕获 `applyViewportRange`，而后者引用 `bars`/`timeframe`；当 `bars`/`timeframe` 变化时，回调仍使用首次 render 的 stale 值，导致 viewport range 错误
2. `pine_rma` 在 `bar_index < length-1` 时返回逐步 SMA（min_periods 行为），与 Pine v5 `ta.rma` 不一致（Pine v5 返回 `na`）；导致 ATR(200) 在前 199 根产生非 na 值
3. `start_of_new_leg`/`start_of_bearish_leg`/`start_of_bullish_leg` 使用 `i > size`，导致首个 leg/pivot 在 `i == size+1` 才检测（延迟一根）；Pine 在 `i == size` 即可首次检测 leg 变化
4. `get_current_structure` 使用 `if i <= size: return`，与 `i > size` 一致延迟一根
5. EQH/EQL DTO 只有 anchor（前一 pivot bar）和 confirmed（新 pivot bar, ref_i=i-size）两个时间点，缺少 detection bar（leg change 确认 bar, i）

## 修改后行为
1. `MiniKlineCard` 闭包根治：
   - 新增 `barsLengthRef`/`timeframeRef` 持有最新值（每次 render 同步，在 effects 之前）
   - `applyViewportRange` 改为 `useCallback([], )` 稳定函数，从 refs 读取最新值
   - 新增 `scheduleApplyRange` 稳定函数，取消 pending rAF 后调度新 rAF
   - `ResizeObserver` 回调调用 `scheduleApplyRange`（不再直接闭包捕获 bars/timeframe）
   - 卸载清理函数取消 pending rAF
2. `pine_rma` 严格复现 Pine v5 `ta.rma`：
   - `bar_index < length-1`：返回 `na`（数据不足以计算 SMA 种子）
   - `bar_index == length-1`：写入 `SMA(src, length)` 种子
   - `bar_index >= length`：Wilder 递推 `rma[i] = (rma[i-1] * (length-1) + src[i]) / length`
3. 首个 pivot off-by-one 修复：
   - `start_of_new_leg`: `i > size` → `i >= size`
   - `start_of_bearish_leg`: `i > size` → `i >= size`
   - `start_of_bullish_leg`: `i > size` → `i >= size`
   - `get_current_structure`: `if i <= size: return` → `if i < size: return`
4. EQH/EQL DTO 三时间点（EQL 和 EQH 两处）：
   - `anchor_index`/`anchor_time`：前一 pivot bar（piv.bar_index/piv.bar_time）
   - `confirmed_index`/`confirmed_time`：新 pivot bar（ref_i=i-size）
   - `detection_index`/`detection_time`：leg change 确认 bar（i）
