## 修改前行为

### 缺口 1: DEFAULT_PARAMS.show_trend 默认值错误
- Pine L74 `showTrendInput = input(false, ...)` 默认 `false`。
- Python `smc_pine_core.py` 的 `DEFAULT_PARAMS["show_trend"]` 误设为 `True`，与 Pine 默认值不一致，无法通过 parity 审计。

### 缺口 2: SMC 输入依赖 MDAS 默认范围，误称"全历史"
- SMC 计算直接复用 `indicator_service` 的 `daily_agg`/`macd_bars`（默认 `include_realtime=True`，含 partial bar）。
- 1d 默认仅约 5000 日历日、1h 约 180 交易日、15m 仅 `bars+1000`、1mo 未保证 ≥200 根，但文档/注释称"全历史"，与实际不符。
- 无明确的 SMC input contract，parity 模式与生产模式混用。

### 缺口 3: 无 deterministic 模式，partial bar 污染历史对齐
- SMC 计算包含当前未完成 partial bar，与 TV 历史导出（仅已完成 bar）不可比。
- 无 `smc_mode` 字段标识 deterministic/realtime，无法区分对齐口径。

### 缺口 4: test_smc_tv_parity.py 仅取 glob 第一份 fixture，断言不全
- TV_COLUMNS 仅 20 列，新增的 trailing/pivot/ATR/parsedHigh-Low/crossed/times 列未读取。
- 测试只取 glob 第一份 fixture（`_FIXTURES[0]`），未参数化全部 fixture。
- OB 列未进入事件比较，几何端点无断言。

### 缺口 5: 无 first-divergence 报告
- parity 失败时仅输出整体不匹配，无首个差异 bar 的上下文（前后 OHLC、状态、触发条件），难以定位根因。

### 缺口 6: 误称"Pine 完全对齐"
- 文档/注释声明 SMC 与 Pine"完全对齐"，但实际 FVG/MTF levels/Premium-Discount/Pine 原色被有意排除或改色，声明与实现不符。

## 修改后行为

### Fix 1: DEFAULT_PARAMS.show_trend 改为 False
- `backend/app/strategy_assets/algorithms/features/smc_pine_core.py` L87：`"show_trend": True` → `"show_trend": False`。
- 添加注释引用 Pine L74；说明 `show_trend` 仅参与 `internal_gate`（Pine L784），gate 已由 `show_internals=True` 满足，故改为 `false` 不影响计算行为，但必须匹配 Pine 默认值以通过 parity 审计。
- 新增 `test_smc_default_params_match_pine` 逐项断言 DEFAULT_PARAMS 匹配 Pine L72-131。

### Fix 2: SMC input contract 明确各周期实际范围
- `backend/app/services/indicator_service.py` 文件头部新增 SMC input contract 注释块：
  - 1d: MDAS 无 limit → DB 全量日线（约 1000-5000 根，非"全历史"）
  - 15m: MDAS `limit=bars + _SMC_WARMUP_BARS(1000)` → bars+1000 根（非全历史）
  - 1h: MDAS 无 limit → DB 全量 1h（约 180 交易日，非"全历史"）
  - 1mo: MDAS `limit=_SMC_MONTHLY_MIN_BARS(200)` → 至少 200 根（ATR200 可初始化）
  - 1w: 复用 `macd_bars`（MDAS 无 limit → DB 全量周线）
- 禁止再称"全历史"，明确各周期实际范围。

### Fix 3: deterministic 模式显式 completed_only=True
- 新增常量 `_SMC_MODE_DETERMINISTIC="deterministic"` / `_SMC_MODE_REALTIME="realtime"`。
- SMC bar 获取重写：1d/1h/1w 独立 deterministic 查询（`completed_only=True` → `include_realtime=False`），15m `limit=bars+1000`，1mo `limit=200`，全部强制 `completed_only=True`。
- 输出 `smc_mode="deterministic"`，确保仅使用已完成 bar，与 TV 历史导出可比。
- realtime 模式（盘中）独立标识，不得与 TV 历史导出混比。
