## 2. 修改前理解

- 当前产品行为：
  - ConsensusZone 半成品代码存在于后端（consensus_zone_service.py / consensus_zone.py schema / test_consensus_zone.py），但前端未启用，违反"Phase 5 前成交量分布 disabled"约束；
  - 前端图层状态散落在 indicatorVisibility / detail-chart-strategy-groups / setLayers 等多个状态源，没有单一真源；
  - 个股详情初始 K 线显示最旧 K 线而非最新行情，根因是 makeDefaultViewport() 调用 createDefaultViewport(0) 构造假 viewport；
  - 已发布的 full snapshot run 可被普通重跑覆盖，缺少保护机制。
- 当前前端流程：StockResearchWorkspace 使用 makeDefaultViewport() 产生 {fromIndex:0,toIndex:0}，clampViewport 扩展为 [0,30) 显示最旧 K 线；viewportByTimeframe 只按 timeframe 保存，切换股票时带入旧视区。
- 当前后端流程：create_snapshot_run 无 published run 保护；upsert_snapshot 无 WHERE 子句保护；backfill 脚本无 --allow-republish 标志。
- 已阅读的当前设计文档：AGENTS.md、docs/current/MANIFEST.md、docs/current/02-data-api-contracts.md、docs/current/03-jobs-integrations-operations.md、docs/current/04-frontend-ux.md、docs/current/05-testing-acceptance.md、docs/current/code-doc-alignment.md、docs/maps/test-coverage-map.md。
- 发现的代码—文档—生产冲突：ConsensusZone DTO 契约（02-data-api-contracts.md §15）描述了已删除的服务；test-coverage-map.md §3.9 引用了已删除的 test_consensus_zone.py。

## 3. 修改目标

- 当前问题：ConsensusZone 半成品代码违反 Phase 5 约束；图层状态散落；K 线初始定位错误；已发布快照可被覆盖。
- 修改后行为：
  1. **ConsensusZone 移除**：删除 consensus_zone_service.py / consensus_zone.py / test_consensus_zone.py；indicator_service 不再提供 consensus_zone 端点；前端工具栏成交量分布保持禁用并显示"成交量分布尚未开放"。
  2. **图层单一状态源**：所有图层开关由 ChartLayerVisibility 类型统一管理（7 键：trend/node/boll/volume/macd/sqzmom/breakout）；localStorage key 为 panji:chart-layer-visibility:v2；StockResearchWorkspace 持有唯一 layerVisibility state；删除 indicatorVisibility / detail-chart-strategy-groups / setLayers 旧状态源。
  3. **K 线 viewport 修复（P0-5）**：删除 makeDefaultViewport()；无已保存 viewport 时向 StrategyChart 传 undefined；StrategyChart 在 calc.length 已知后使用 createDefaultViewport(calc.length) 定位到末尾；viewport 复合 key `${symbol}:${timeframe}` 确保切换股票重置；auto-follow effect 在用户位于最右端时自动跟随新 bar。
  4. **Published snapshot 保护（P0-4）**：create_snapshot_run(scope='full', allow_republish=False) 检查已存在 published full run，存在则抛 PublishedSnapshotRunExistsError；upsert_snapshot(allow_republish=False) WHERE 子句保护已归属 published run 的 snapshot；after_close_orchestrator 捕获异常后优雅降级（跳过计算复用已有 run）；backfill 脚本新增 --allow-republish 标志。
- 验收标准：pytest 全绿、tsc/build 通过、契约测试通过、docs consistency 通过、真实浏览器 E2E 验证 K 线最右为 API 返回最新 bar。
- 明确不修改：Capture 路由、飞书投递、Migration 历史、DSA 计算逻辑、MACD 算法、SQZMOM 算法。
