# CHANGE-20260707-042: Indicator Overlay All Timeframes

## 变更编号
CHANGE-20260707-042

## 任务名称
Indicator overlay 全周期支持（DSA VWAP 1d/15m/1h/1w/1mo + BB 1w/1mo 修复）

## 需求出处
用户纠正 PR #31 的"DSA overlay 1d-only by design"规则：所有周期都必须能渲染 DSA VWAP，不仅 1d，1d/15m/1h/1w/1mo 都要支持。同时发现 1w/1mo BB 没画出来，也要一起修。

明确约束（用户原话）：
- 不要改 DSA/BB 数学公式
- 不做事件/相似场景/板块聚合
- 不要截图验收，不生成 coverage/大日志/截图
- 磁盘快满，不新增持久大文件
- 仍保留 source mismatch 保护，不能为显示而强行绕过校验
- Temporal V1 仍按既有规则，不把短周期 DSA 当主输入

根因（PR #31 的两个错误规则）：
1. **DSA 1d-only 误禁用**：PR #31 把 DSA overlay 限制为"1d-only by design"，并禁用 15m/1h toggle；但用户原本要求 DSA 在所有周期都可渲染（用于核查该周期结构，非主趋势锚）。
2. **1w/1mo BB 字段被直接 pop**：PR #31 在 `_adapt_watchlist_bb` 中对 1w/1mo 路径直接 `pop` BB 字段，并在 `chart_layers` 循环 `continue` 跳过 1w/1mo BB 图层，导致前端无 BB overlay 可渲染。

## 修改前行为
- `indicator_cache.ALGORITHM_VERSION = "v4"`（PR #31 bump），缓存键含 `:v4`；
- `dsaOverlayPolicy.ts`：`DSA_DISABLED_HINT` 常量 + `shouldCheckDsaMismatch(timeframe)` 仅 1d 返回 true，15m/1h 返回 false；
- `StrategyChart.tsx`：15m/1h DSA 按钮 disabled，title 显示 `DSA_DISABLED_HINT`；
- `_adapt_watchlist_bb`：1w/1mo 路径直接 `pop` BB 字段（bb_upper/bb_mid/bb_lower/bb_pos/bb_width），返回空 BB；
- `chart_layers` 循环：`if timeframe in ("1w", "1mo") and strategy_id == "watchlist_monitor" and layer.get("id") == "bb": continue` 跳过 1w/1mo BB 图层；
- `MarketDataContext(bars_daily=daily_bars)`：DSA 永远用日线 bars 计算，即使是非日线周期；
- `daily_time_list = [idx.isoformat() for idx in daily_bars.index]`：DSA time 数组永远是日线日期格式。

## 修改后行为
- `indicator_cache.ALGORITHM_VERSION = "v5"`，旧 v4 缓存 key 不匹配新 `build_cache_key`，强制重算（避免旧 v4 缓存返回 1d-only DSA + 1w/1mo 无 BB）；
- `dsaOverlayPolicy.ts` 重写：
  - `shouldAllowDsaOverlay(timeframe)`：1d/15m/1h/1w/1mo 全部返回 `true`；
  - `shouldCheckDsaMismatch(timeframe)`：全周期返回 `true`（全周期渲染，全部需校验 source 对齐，不绕过 mismatch 保护）；
  - `DSA_TITLE_HINT(timeframe)`：1d 返回 "DSA VWAP 日线结构锚。"，非 1d 返回 "DSA VWAP 当前周期验证图层：用于核查该周期结构，不作为主趋势锚。"；
  - 删除 `DSA_DISABLED_HINT` 常量；
- `StrategyChart.tsx`：
  - DSA toggle 在所有周期可点击（不 disabled）；
  - title 改用 `DSA_TITLE_HINT(timeframe)`；
  - 保留 source mismatch banner（matched/klineKeys.size < 0.5 时暂停渲染），不绕过校验；
- `_adapt_watchlist_bb`：1w/1mo 路径合并到 15m/1h 路径，统一用 `compute_bollinger(macd_bars, length=20, mult=2.0)` 计算 BB（不再 pop BB 字段）；
- `chart_layers` 循环：删除 1w/1mo BB `continue` 逻辑，1w/1mo BB 图层正常进入 renderer；
- `MarketDataContext(bars_daily=macd_bars)`：DSA 在所有周期用当前 timeframe bars 计算，不再仅由日线驱动；
- `daily_time_list = [idx.isoformat() for idx in macd_bars.index]`：DSA time 数组与策略输出长度一致，15m/1h 含时间部分（`YYYY-MM-DDTHH:MM:SS`），1d 仍为 `YYYY-MM-DD`。

## 风险
- `ALGORITHM_VERSION` bump v4→v5 会使所有 indicator 缓存失效，首次请求需重算（单只股票 < 1s，可接受）；
- 15m/1h/1w/1mo DSA 数值会变化（从 disabled 改为可渲染，DSA VWAP 由当前周期 bars 计算），用户视觉上会看到新增 DSA overlay（这是修复目标，非回归）；
- 1w/1mo BB 数值会从 null 变为 `compute_bollinger(macd_bars)` 计算结果（修复目标，非回归）；
- DSA source mismatch 保护仍保留：所有周期都校验 source 对齐，matched/klineKeys.size < 0.5 时仍暂停渲染（不绕过校验）；
- Temporal V1 不受影响：仍按既有规则，不把短周期 DSA 当主输入；
- 性能：无新增表/worker/缓存；只单只股票 API 计算路径调整；
- 磁盘：无新增持久大文件，不生成截图/coverage/大日志。

## 遗留问题
- 无。本次修复 DSA VWAP 全周期支持 + 1w/1mo BB 两个问题，不改 DSA/BB 数学公式，不做事件/相似场景/板块聚合。
