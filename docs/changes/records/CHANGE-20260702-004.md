## 2. 修改前理解

- 当前产品行为：DSA 选股计算（`dsa_selector` released 版本）在 `backend/tools/dsa_benchmark.py` 中已有基准测试脚本，可分层采样活跃 A 股股票并输出冷/热启动耗时分布。此前因环境数据限制被标记为跳过。
- 当前前端流程：无变化。DSA 性能基准测试为后台运维/发布验收数据，不直接改变前端交互。
- 当前后端流程：`dsa_benchmark.py` 加载 released `dsa_selector` runtime，从 `bars_daily` 读取日线行情，对样本股票分别执行一次冷启动和一次热启动 `runtime.execute(ctx)`，统计 p50/p90/p95/p99/max/avg。
- 当前数据/API/Worker/第三方关系：仅读取生产 Docker PostgreSQL 容器中的 `instruments` 和 `bars_daily` 数据，不写库、不发消息、不调用第三方服务。
