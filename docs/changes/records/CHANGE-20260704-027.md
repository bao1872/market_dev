# CHANGE-20260704-027

## 变更编号

CHANGE-20260704-027

## 任务名称

DSA Run 总超时与 Computable Universe 口径修复

## 需求出处

- 2026-07-04 DSA-only 运行后 1881 只 failed（run_id=6510a98c），全部 reason_code=timeout。
- after_close_orchestrator._DSA_POLL_TIMEOUT_SECONDS=7200 与 strategy_batch_service._RUN_TOTAL_TIMEOUT_SECONDS=600 冲突。
- 历史 bars 不足标的此前未前置分类，进入 DSA 计算产生无意义 failed。
- execute_run 覆盖 skipped_count 导致质量门禁 succeeded + skipped == total 失败。
- ALIGN-030 全市场约 187 只 active 标的日线 < 60 根。

## 修改前行为

- _RUN_TOTAL_TIMEOUT_SECONDS 硬编码 600 秒，与编排层 7200 秒轮询超时冲突。
- run 级总超时耗尽后剩余 pending 项统一标记 failed/timeout。
- create_batch_run 将全部 active A 股作为 computable universe。
- execute_run 初始化 skipped = 0，覆盖预置 skipped_count。
- docker-compose.prod.yml 未声明 STRATEGY_RUN_TOTAL_TIMEOUT_SECONDS。

## 修改后行为

- _RUN_TOTAL_TIMEOUT_SECONDS 默认 7200 秒，可通过 STRATEGY_RUN_TOTAL_TIMEOUT_SECONDS 环境变量配置。
- run 级总超时耗尽后剩余 pending 项标记 failed/run_timeout_budget_exhausted。
- 新增 _classify_computable_universe 方法，_DSA_MIN_HISTORY_BARS=60，历史不足标的标记 skipped/insufficient_history。
- _SKIPPED_REASON_ALLOWLIST 新增 insufficient_history。
- execute_run 初始化 skipped = run.skipped_count or 0。
- docker-compose.prod.yml 追加 STRATEGY_RUN_TOTAL_TIMEOUT_SECONDS: ${STRATEGY_RUN_TOTAL_TIMEOUT_SECONDS:-7200}。
- 质量门禁仍禁止 partial_failed 发布。
