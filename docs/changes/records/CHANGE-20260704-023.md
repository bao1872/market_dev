# CHANGE-20260704-023

## 变更编号

CHANGE-20260704-023

## 任务名称

生产验证修复：bars / indicators API page_size、bars 上限与 Node Cluster 15m/1h 契约对齐

## 需求出处

PR #11 部署后生产验证发现：
- 个股详情页切换 15m 时请求 `page_size=4000`，后端 `/api/v1/instruments/{id}/bars` 返回 422（`page_size <= 1000`）；
- 切换 1h 时请求 `page_size=1200`，同样触发 422；
- `/api/v1/instruments/{id}/indicators` 请求 `bars=4000` 时返回 422（`bars <= 500`）。

上述限制导致前端无法获取 Node Cluster 所需的 15m 4000 根 / 1h 1200 根输入，个股详情 K 线/指标在日内周期不可用。

## 修改前行为

- `backend/app/api/bars.py` 中 `_MAX_PAGE_SIZE = 1000`，`page_size` Query 校验 `le=1000`；
- `backend/app/api/indicators.py` 中 `bars` Query 校验 `le=500`；
- 前端 `StockDetailPage` 按 timeframe 请求 15m=4000、1h=1200，与后端限制冲突。

## 修改后行为

- `backend/app/api/bars.py` 中 `_MAX_PAGE_SIZE` 调整为 `4000`，与 Node Cluster 15m 4000 根契约对齐，1h 1200 根也得以通过；
- `backend/app/api/indicators.py` 中 `bars` 上限调整为 `4000`，与 bars API 及 Node Cluster 契约对齐；
- 修正 `backend/app/api/bars.py` 顶部 docstring 中 page_size 说明；
- 顺手修复同一文件中 Ruff 报错：删除未使用的 `datetime.time as dt_time` 导入，补充缺失的 `get_redis` 导入（`bars/health` 端点依赖）。

## 影响模块

- 后端：`backend/app/api/bars.py`、`backend/app/api/indicators.py`
- 前端：无需修改（已按契约请求）
- 文档：api-route-map.md 页大小说明、CHANGELOG
