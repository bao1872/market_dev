## 任务名称

趋势选股 result_id 未回填修复：(run_id, instrument_id) 关联 strategy_results + 历史债务分级审计

## 需求出处

- PR #15 (`fix/screener-full-universe-results`) 合并部署后，生产验证发现 succeeded 行 `payload` 全部为空、35 个 DSA 指标不显示。
- 根因：`strategy_run_items.result_id` 在 PR #14 batch service 写入 `strategy_results` 后未回填（始终为 None）。
- 影响：`query_run_items_with_results` 通过 `selectinload(StrategyRunItem.result)` 加载关联结果时返回 None，导致 succeeded 行降级为无指标行。
- 同时本 PR 顺带完成历史 Ruff/Mypy 债务分级审计（不清债务，只做审计）。

## 修改前行为

- `query_run_items_with_results` 使用 `selectinload(StrategyRunItem.result)` 加载关联，依赖 `strategy_run_items.result_id`。
- `strategy_run_items.result_id` 在 PR #14 batch service 写入 `strategy_results` 后从未回填（始终为 None）。
- `_apply_run_item_filters` 中 `metric_filter` 子查询通过 `StrategyRunItem.result_id IN (sub)` 关联，因 result_id 为 None 全部失效。
- sort LEFT JOIN 通过 `StrategyRunItem.result_id == sort_sub.c.result_id` 关联，同样失效。
- 生产现象：`/strategy-runs/{run_id}/results` succeeded 行 35 个 DSA 指标全部为空，仅显示股票代码和名称。

## 修改后行为

- `query_run_items_with_results` 移除 `selectinload(StrategyRunItem.result)`，改用额外查询：
  - `SELECT * FROM strategy_results WHERE run_id=? AND instrument_id IN (...)` 批量加载
  - 构建 `results_map: dict[uuid.UUID, StrategyResult]`，key 为 `instrument_id`
  - 构建 `RunItemResultRow` 时使用 `result=results_map.get(item.instrument_id)`（非 `item.result`）
- `_apply_run_item_filters` `metric_filter` 子查询改为：
  - `SELECT strategy_results.instrument_id FROM strategy_results JOIN strategy_result_metrics ON result_id=strategy_results.id WHERE run_id=? AND metric_key=? AND ...`
  - 主查询 `WHERE strategy_run_items.instrument_id IN (sub)`（非 `result_id IN (sub)`）
- sort LEFT JOIN 改为：
  - 子查询返回 `strategy_results.instrument_id` + `strategy_result_metrics.numeric_value`
  - 主查询 `OUTER JOIN sub ON strategy_run_items.instrument_id = sub.c.sort_instrument_id`
- 生产验证通过：run_id=f0c15e1c
  - `source_total=5293, filtered_total=5293`（= `run.total_instruments`）
  - succeeded 行正确显示 35 个 DSA 指标（payload_keys 包含 `bb.position`、`bb.width`、`node.position` 等）
  - skipped 行（reason_code=insufficient_history）显示股票代码和名称，payload=None
- ALIGN-032 关闭（生产验证通过），新增 ALIGN-033（result_id 未回填问题，P2）
- 新增 `docs/architecture-audits/AUDIT-20260704-ruff-mypy-debt-triage.md`（P0/P1/P2 分级，不直接清债务）
