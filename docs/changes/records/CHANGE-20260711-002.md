## 收口修复（同分支追加 commit）

### 根因
advice.md review 发现 PR #70 第一版存在以下缺陷：
1. **P0 timeframe 双重状态**：`StockResearchWorkspace` 内部 `useState('1d')` 与 `MarketWorkspacePage` 从 URL 传入 `useStockResearchData` 的 timeframe 脱节。URL `?timeframe=15m` 时数据为 15m 但图表以 1d 渲染。
2. **P0 请求未门控**：`useWatchlistMonitorStatus` 和 `useInstruments` 均无 `enabled` 参数，导致 watchlist scope 也请求 `/instruments`，market scope 也请求 `/watchlist/monitor-status`，搜索词不足 2 字符时仍请求前 50 只股票。
3. **P1 无用请求**：`useStockResearchData` 请求了 `MarketWorkspace` 未使用的 watchlist/batchInstruments/stockMemo。
4. **P1 URL 字段丢失**：URL 状态只有 scope/symbol/timeframe，每次 `setSearchParams` 会清除未来可能新增的 source/strategy/event_id；非法 timeframe 原样传后端。
5. **P1 策略硬编码**：`StockResearchWorkspace` 即使收到 `source='selection'` 仍固定 `WATCHLIST_MONITOR`。
6. **P1 周期文案错误**：所有 timeframe 下状态条都显示"完整日线"。
7. **P2 注释失真**：`StockResearchWorkspace` 注释声称被 `StockDetailPage` 复用，但实际未复用。
8. **P2 AccountMenu 未复用导航真源**：组件内 inline 构建菜单项，未调用 `appNavigation` helper。

### 修复内容
1. `marketWorkspaceUrlState.ts`：
   - 新增 `DisplayTimeframe` 类型 + `ALLOWED_TIMEFRAMES` 常量，非法 URL 值回退 `1d`。
   - 新增 `ResearchSource` 类型 + `source`/`strategy`/`eventId` 字段到 `MarketWorkspaceUrlState`。
   - 新增 `defaultStrategyForSource(source)`：watchlist→watchlist_monitor, selection→dsa_selector。
   - encode 时保留 source/strategy/event_id；strategy 等于 source 默认值时省略。
2. `useApi.ts`：
   - `useInstruments(params, options?: { enabled? })` 增加可选 enabled 参数。
   - `useWatchlistMonitorStatus(options?: { enabled? })` 增加可选 enabled 参数。
3. `MarketInstrumentPane.tsx`：
   - `useWatchlistMonitorStatus({ enabled: scope === 'watchlist' })`。
   - `useInstruments(params, { enabled: scope === 'market' && canSearch })`。
4. `useStockResearchData.ts`：
   - 移除 `useWatchlist`/`useBatchInstruments`/`useStockMemo` 查询。
   - 只保留 instrument/bars/indicators/quote/events 核心查询。
   - `timeframe` 参数类型从 `string` 收紧为 `DisplayTimeframe`。
5. `StockResearchWorkspace.tsx`：
   - 删除内部 `useState('1d')`，改为接收受控 `timeframe: DisplayTimeframe` + `onTimeframeChange`。
   - 接收严格类型 `source: ResearchSource` + `strategyKey: string`，禁止硬编码。
   - 新增 `barsStatusLabel(tf, isPartial)` 根据 timeframe 显示真实周期文案。
   - 新增 instrument/bars/indicators 错误状态显示（含重试按钮）。
   - 修正注释：当前仅 `MarketWorkspacePage` 使用。
6. `MarketWorkspacePage.tsx`：
   - 从 URL 解析 source/strategy/eventId。
   - `handleTimeframeChange` 写回 URL（保留 source/strategy/event_id）。
   - `handleSelectSymbol` 选择新股票时 `eventId: null`（清除旧 event_id）。
   - 传 timeframe/onTimeframeChange/source/strategyKey 给 `StockResearchWorkspace`。
7. `appNavigation.ts`：
   - 新增 `AccountMenuVariant` 类型 + `getAccountMenuItemsForVariant(isAdmin, variant)` 函数。
   - variant='admin' 时移除"管理后台"项，追加"返回行情"。
8. `AccountMenu.tsx`：
   - 移除 inline 菜单构建，改为调用 `getAccountMenuItemsForVariant(isAdmin, variant)`。
9. `global.scss`：
   - 新增 `.tv-chart-error` 和 `.tv-chart-retry` 样式。
10. 测试：
    - `marketWorkspaceUrlState.test.ts` 从 7 用例扩展到 16 用例（含非法 timeframe 回退、source 校验、strategy 默认推导、event_id 清除等）。
    - `appNavigation.test.ts` 从 6 用例扩展到 10 用例（含 4 项 getAccountMenuItemsForVariant 测试）。
