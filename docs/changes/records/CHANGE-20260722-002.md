## 背景

PROMPT.md §Phase 4.2-1.2 指出 mypy baseline 矛盾：

- `tools/quality_baselines/mypy.json` 显示 `total: 0, diagnostics: []`
- 实际运行 `mypy app`（mypy 1.9.0）产生 14 个 errors
- 禁止 baseline 保持 0 而实际有 14 条

## 变更内容

### 1. mypy 版本对齐

- pyproject.toml 要求 `mypy==2.1.0`，系统安装的是 1.9.0
- 使用 `pip install --break-system-packages --ignore-installed mypy==2.1.0` 升级
- mypy 2.1.0 产生 11 行 JSONL（1 error + 10 notes），而非 14 errors

### 2. main vs branch 对比（临时 worktree）

- 创建临时 worktree `/tmp/mypy-main-compare` 检出 `origin/main`（631b191）
- 在两个 worktree 中分别运行 `mypy app --output json --show-error-codes --no-error-summary --hide-error-context --no-pretty`
- **结果完全相同**：1 error + 10 notes
- 唯一的 error：`app/core/redis_client.py:70 [attr-defined] "Redis[Any]" has no attribute "aclose"; maybe "close"?`
- 10 个 notes 是 `annotation-unchecked`（mypy 2.1.0 新特性，severity=note，不是 error，不计入 baseline）

### 3. baseline 更新

- `tools/quality_baselines/mypy.json`：
  - `baseline_commit`: `a8481c7` → `8aae487`
  - `total`: 0 → 1
  - `unique`: 0 → 1
  - `diagnostics`: `[]` → 1 条诊断（redis_client.py attr-defined）

### 4. .gitignore 更新

- 添加 `mypy-report.jsonl`（CI 生成的临时文件，不应入库）

### 5. 临时 worktree 清理

- `git worktree remove /tmp/mypy-main-compare --force`
- 清理 `/tmp/mypy-*.txt` 和 `/tmp/mypy-*.jsonl`
