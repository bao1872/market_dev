CHANGE-20260723-004

## 任务名称
V3.3 部署前门禁 — Task 6 隔离视觉验收 SMC E2E fixture 增强

## 需求出处
- `ref/PROMPT.md` §6（隔离视觉验收）：SMC 1440×2560原图、360px手机预览、Detail与Capture同结构对照、Node Cluster和BB回归图片，至少10个BOS/CHoCH/EQH/EQL/OB记录
- CP-V3-C2 strict time-key 模式要求所有 SMC 事件提供 anchor_time/confirmed_time/second_pivot_time

## 修改前行为
1. **E2E mock fixture SMC 数据使用旧格式**：`buildSmcIndicators` 生成 index-based 事件（无 anchor_time/confirmed_time/second_pivot_time），在 CP-V3-C2 `strictTimeKey=true` 模式下所有事件触发 `missing_time` 诊断并被 skip，导致 SMC 视觉验收无法验证渲染
2. **fixture 缺少 `time` 数组和 `swing_bias`**：前端 StrategyChart 期望 `smc.time` 和 `smc.swing_bias` 字段
3. **fixture 使用旧字段名**：缺少 `equal_highs_lows` 单数组（前端期望单数组，非分开的 equal_highs/equal_lows）

## 修改后行为

### SMC fixture 升级为 CP-V3-C2 time-key 格式
- `buildSmcIndicators(barsCount, timeframe)` 新增 `timeframe` 参数
- 生成 `time` 数组（与 bar 时间一致）和 `swing_bias` 字段
- 所有事件提供 `anchor_time`/`confirmed_time`（EQH/EQL 额外提供 `second_pivot_time`）
- 时间格式遵循 `normalizeChartTime`：1d → "YYYY-MM-DD"，15m/1h → "YYYY-MM-DD HH:MM"
- 12 个 SMC 事件（3 BOS + 2 CHoCH + 3 OB + 2 EQH + 2 EQL），覆盖 display 60/90/120/250 窗口
- `order_blocks` 和 `equal_highs_lows` 使用正确字段名

### 视觉验收证据生成
- 5 张截图：SMC Capture 1440×2560、SMC Capture 360×640、SMC Detail 1440×2560、Node Cluster 回归、Bollinger 回归
- 12 个 SMC 锚点记录（10 可见）：anchor/confirmed/second_pivot time + price/zone + visibility
- 锚点表保存于 `/tmp/visual-acceptance/smc-anchor-records.md`
