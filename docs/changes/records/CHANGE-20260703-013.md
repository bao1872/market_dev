# CHANGE-20260703-013

## 变更编号

CHANGE-20260703-013

## 任务名称

修复 release candidate 上新增 backend/app 文件的 mypy 错误，解除 Mypy New Files CI 阻断

## 需求出处

PR #1（release/docs-aligned-candidate-v3）在 GitHub Actions Run #45 中 `Mypy New Files` 任务失败，mergeable_state 为 `unstable`，无法合并到 `main`。

## 当前问题

`Mypy New Files` 任务对 `backend/app` 下相对于 `origin/main` 新增的 19 个 Python 文件执行 mypy 增量检查，发现 13 个真实诊断：

- `app/models/access_audit_log.py:164`：`FromClause` 无 `indexes` 属性；
- `app/services/subscription_service.py:391/418-422/442`：`subscription` 被推断为 `Subscription | None`，在续期分支访问其属性并报错；
- `app/services/market_data_aggregation_service.py:412`：Redis `get` 返回 `bytes | str`，直接传入只接受 `str` 的反序列化函数；
- `app/scripts/fix_instruments_remove_indices.py:73`：`Result[Any]` 无 `rowcount` 属性；
- `app/api/capture.py:212` 与 `app/api/admin_subscription.py:1156`：自测代码对 `BaseRoute` 直接访问 `.path`；
- `app/api/admin_subscription.py:462`：返回 `dict[str, Any]` 与声明的 `SystemOverviewResponse` 不符。

## 当前设计理解

- Mypy 增量阻断是项目硬规则（AGENTS.md 第 14 条）：新增 backend/app 生产文件必须 mypy 零错误；
- 修复应为最小侵入式类型修正，不改变业务行为；
- 自测入口（`if __name__ == "__main__":`）同样属于生产文件，需保持通过 mypy；
- 修复后必须重新跑通 `Mypy New Files` 与 `Mypy Baseline Regression`，不得增加历史债务。

## 风险

- `subscription_service.py` 的 `assert` 在极端情况下会抛出 `AssertionError`；但逻辑上 `is_new_subscription=False` 已保证 `subscription` 非空，断言仅作为类型收窄与防御性检查；
- `admin_subscription.py` 使用 `SystemOverviewResponse(**data)` 替代直接返回 dict，FastAPI 原本也会通过 `response_model` 执行相同构造，运行时行为不变；
- 若 CI 环境与我本地环境存在 mypy 插件/依赖差异，可能仍有少量差异诊断，需以 CI 结果为准。

## 实际修改内容

1. `subscription_service.py`：在续期分支新增 `assert subscription is not None`，并在返回前再次断言，消除 `union-attr` 与 `return-value` 诊断；
2. `market_data_aggregation_service.py`：Redis 缓存读取时对 `bytes` 做 `utf-8` 解码后再反序列化；
3. `access_audit_log.py`：为 `__table__.indexes` 添加 `# type: ignore[attr-defined]`；
4. `fix_instruments_remove_indices.py`：为 `result.rowcount` 添加 `# type: ignore[attr-defined]`；
5. `capture.py` 与 `admin_subscription.py` 自测入口：使用 `getattr(r, "path", None)` 并过滤非字符串项，避免对 `BaseRoute` 直接访问 `.path`；
6. `admin_subscription.py`：`/admin/system-overview` 端点显式构造 `SystemOverviewResponse(**data)` 返回，匹配声明的返回类型；
7. 新增 `docs/changes/records/CHANGE-20260703-013.md`；
8. 更新 `docs/changes/CHANGELOG.md` 索引；
9. 更新 `docs/current/15-testing-acceptance.md` 记录本次修复。
