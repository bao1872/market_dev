# CHANGE-20260704-024

## 变更编号

CHANGE-20260704-024

## 任务名称

自选监控页 UI 调整、AGENTS 无备份部署规则、TCL 科技单标历史回补

## 需求出处

- 用户指令：自选股监控页去掉每行状态栏，恢复表头过滤，字体对齐趋势选股页；TCL 科技个股详情只显示约 6 个 bar，需查根因并单标回补；测试期部署默认不备份数据库。

## 修改前行为

- `WatchlistPage` 桌面表格含每行 `status` 列，移动端卡片头也显示 `MonitorStatusBadge`；
- 数据列 `filterable=false`，表头无法过滤；
- 表格未使用 `compact-table`，字体/布局与趋势选股页不一致；
- `AGENTS.md` 未明确测试期部署禁止备份数据库；
- `docs/current/code-doc-alignment.md` ALIGN-030 记录 `000100`（TCL科技）日线仅 5 根、15min 仅 66 根，导致个股详情 K 线只有约 6 个交易日；
- 无现成单标历史回补命令行工具。

## 修改后行为

- 桌面表格移除 `status` 列，数据列全部 `filterable=true`；
- 移动端卡片头移除状态徽章；
- `WatchlistPage` 页眉用 `MonitorStatusBadge` 全局展示市场状态；
- `WatchlistMonitorTable` 使用 `tableClassName="compact-table"`；
- 新增 `frontend/src/features/watchlist-monitor/__tests__/columns.test.ts` 契约测试，并加入 `npm run test:contract`；
- `AGENTS.md` 新增硬规则 11：测试期部署默认不备份数据库，禁止 `pg_dump` 和写入 `/root/backups`、`/root/web_dev/backups`；
- `docs/current/02-data-api-contracts.md`、`04-frontend-ux.md`、`maps/api-route-map.md`、`maps/frontend-route-map.md`、`maps/test-coverage-map.md` 同步更新；
- `docs/current/code-doc-alignment.md` ALIGN-030 更新证据：`000100` 已单标回补，全市场仍有约 81 只 active 标的日线 < 50 根；
- 新增 `backend/tools/backfill_single_instrument.py` 单标回补工具；
- 生产 DB 中 `000100` 日线从 5 根增至 846 根，15min 从 66 根增至 8000 根，60min 新增 2000 根；API `page_size=4000` 可返回 4000 根 15m bar。
