# CHANGE-20260713-010: 个股详情市值 + 列表 Excel 导出 + 右栏小 K 线 + 股票名称筛选 alias + 文档修正

## 变更编号
CHANGE-20260713-010

## 任务名称
个股详情补充总市值/流通市值；列表增加真实 Excel 导出；右侧事件状态栏增加日/周/月小 K 线；强化股票名称详情入口和筛选；文档修正四层数量语义

## 需求出处
用户指令（2026-07-13/14）：以已推送部署的 commit 1684921 为基线，在分支 fix/pr74-context-layers-remove-consensus 上实施九大指令块。市值不得永久返回 null，必须审计 pytdx/mootdx/qstock/MDAS 能力并放入每日主数据同步链；Excel 导出补 API 集成测试；小 K 线补真实交互验证；filterAlias 补测试；全仓修正文档冲突。

## 修改前行为
1. 个股详情报价条无总市值/流通市值字段
2. instruments 表无 total_share/float_share/share_as_of 列
3. 无每日股本同步任务
4. pytdx_adapter.get_finance_info 引用不存在的 helper 函数（_resolve_market/_ensure_connected/_handle_disconnect）
5. /market 列表无 Excel 导出功能
6. /market 右栏只有 EventStatePanel，无小 K 线
7. 股票名称列头筛选与顶部搜索是独立状态，不同步
8. 部分文档错误描述 source_total 受 universe 影响 / 与筛选使用相同条件

## 修改后行为

### 一、个股详情总市值和流通市值
1. pytdx `get_finance_info` 实测可用，返回 zongguben（总股本）/liutongguben（流通股本）/updated_date（YYYYMMDD int）
2. 新增 migration 063：instruments 表增加 total_share/float_share（Decimal）/share_as_of（date）三列
3. 新增 `instrument_share_sync_service.sync_share_capitals`：每日 18:00 通过 pytdx 同步 SH/SZ 股本到 DB（BJ 跳过），单进程单连接批次 500，pytdx 同步调用通过 asyncio.to_thread 包装
4. bars_scheduler worker 新增 `scheduled_share_capital_sync` 任务（CronTrigger 18:00 Asia/Shanghai，max_instances=1）
5. quote 端点新增 `_compute_market_cap_fields` 辅助函数：从 Instrument 股本字段 + 当前价格计算市值，两个 quote builder 共用
6. QuoteResponse schema 新增 5 个字段：total_market_cap/float_market_cap/market_cap_as_of/market_cap_source/market_cap_degraded_reason
7. 数据缺失时返回 market_cap_degraded_reason="market_cap_data_unavailable"，不伪造
8. 禁止用户请求时第三方联网：quote 端点只从 DB 读取股本，pytdx 调用仅在每日同步任务中
9. 修复 pytdx_adapter.get_finance_info：使用 market_from_code/connect/disconnect 模式（参照 _fetch_xdxr_from_pytdx）

### 二、列表 Excel 导出
1. 新增 `POST /strategy-runs/{run_id}/results/export` 端点（需 published run + active subscription + trend_selection feature）
2. 新增 `excel_export_service`：使用标准库 zipfile + XML 生成真实 .xlsx（OOXML），禁止 openpyxl/xlsxwriter
3. MAX_EXPORT_ROWS=10000 上限，超过返回 422
4. 公式注入防护：=、+、-、@ 开头文本前缀单引号
5. 响应头含 X-Source-Total/X-Universe-Total/X-Filtered-Total/X-Export-Rows 四层数量
6. Content-Disposition 使用 RFC 5987 percent-encoding（filename*=UTF-8''）
7. 文件名格式：盘迹_DSA_YYYYMMDD_筛选结果.xlsx
8. 导出列由前端 ExportContext 传递（visibleColumns + payload_key），stock 列 payload_key=null（不导出操作列）
9. handleExport 复用 convertFiltersToMetricFilters（与 buildStrategyResultQueryParams 同源），禁止第二套筛选口径
10. 后端测试：test_excel_export_service.py（9 单元）+ test_excel_export_api.py（19 集成），覆盖权限/published/universe/筛选/行数/列白名单/注入/上限/MIME/临时文件/无N+1 + 文件验证（zip完整性+XML解析+workbook关系+单元格类型）

### 三、右侧小 K 线
1. 新增 `MiniKlineCard`：使用 lightweight-charts v4 createChart + addCandlestickSeries，三按钮（日线/周线/月线），默认日线
2. 新增 `useMiniKlineData`：薄封装 useInstrumentBySymbol + useBars，BARS_COUNT={1d:80,1w:60,1mo:48}，refetchInterval:false 不轮询
3. 新增 `MarketRightPanel`：组合 MiniKlineCard（顶部）+ EventStatePanel（底部），symbol 为 null 时 MiniKlineCard 显示提示
4. 面板收起时不挂载 MarketRightPanel（0 bars/context 请求）
5. chart 实例仅创建一次（useEffect 空依赖数组 []），无重复 canvas
6. ResizeObserver 响应式宽度，卸载清理 resizeObserver.disconnect() + chart.remove() + ref 清空
7. timeframe state 独立于 symbol（切股票保留用户周期）
8. 只请求活动周期（不预取三周期），切换周期命中 React Query 缓存
9. 不显示指标/成交量/Node/事件标记/工具栏

### 四、股票名称 filterAlias
1. DataTableColumn 新增 filterAlias?: 'keyword' 字段
2. stock 列设置 filterable=true + filterAlias='keyword'（与顶部搜索共用唯一 keyword 真源）
3. KeywordFilterPopover 组件：onApply/onClear 同时调用 setGlobalQuery + onKeywordChange（双向同步）
4. 列头激活状态基于 effectiveKeyword（与顶部搜索共用）
5. filterAlias='keyword' 不进入 filters state（使用 isKeyword flag 区分）
6. effectiveKeyword 受控模式：externalKeyword 提供时覆盖内部 globalQuery
7. URL 同步使用 effectiveKeyword，setSearchParams replace:true 避免循环
8. URL hydration 使用 skipNextUrlSyncRef 避免反向覆盖
9. currentConfig.keyword 基于 effectiveKeyword（preset 持久化共用真源）
10. applyPresetConfig 同时 setGlobalQuery + onKeywordChange（preset 应用双向同步）
11. stock/action 列不入 metric_filters（buildStrategyResultQueryParams + convertFiltersToMetricFilters 过滤）

### 五、文档修正
1. source_total = published run 原始总量，不受 keyword/industry/concept/metric_filters 影响，也不受 universe 影响
2. universe_total = all/watchlist 范围总量，业务筛选前
3. filtered_total = keyword+industry+concept+metric_filters 后总量
4. items = filtered_total 当前页
5. 删除"source_total 与筛选使用相同条件"和"source_total 受 universe 影响"旧描述
6. AGENTS.md clause 25 preset 增加 industry/concept 字段
