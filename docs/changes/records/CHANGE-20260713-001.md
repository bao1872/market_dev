# CHANGE-20260713-001: Regime Discovery V1 正确性修复 + 资源优化 + 全量 Phase E

## 变更编号
CHANGE-20260713-001

## 任务名称
Regime Discovery V1 横截面 rank 正确性修复、内存优化、全量 Phase E 验证

## 需求出处
用户 expanded requirements（2026-07-13）：修复全量 assignment 的横截面 rank 正确性、资源优化至 RSS ≤ 1.5GB、补全测试、完成全量 Phase E

## 修改前行为
1. Phase E 使用 `iter_chunks` 按 `(instrument_id, trade_date)` 排序后分块读取，导致同一 `trade_date` 的不同 instrument 被分割到不同 chunk，cross-sectional rank 在 instrument 子集上计算 — **正确性缺陷**；
2. `run_representation` 返回 `scaler_params` 键，但 `transform_feature_matrix` 期望 `scaler` 键 — Phase E 全量 assignment 未应用 RobustScaler；
3. `_compute_temporal_derivatives` 使用 per-instrument for 循环 + `df.loc[idx, ...]` 赋值（5177 个 instrument），产生大量中间 copy，RSS 峰值过高；
4. `pd.read_sql` 全量读取 621k 行时缓冲为 float64 Python 对象，峰值 RSS 达 1618 MB（> 1.5GB 目标）；
5. manifest `pca_summary` 包含 PCA model 对象，JSON 序列化失败；
6. reporting `generate_report_md` 参数契约与调用方不一致（`prevalence` / `dwell` 缺失）；
7. mypy 类型问题（`stability.py` 的 `Sequence[Any]` 不接受 `np.ndarray`）；
8. 无横截面 rank 正确性测试、无 sample vs full-assignment 模式区分测试。

## 修改后行为
1. **Phase E 重写**：使用 `get_all_matrix_rows` 一次性读取全量 621k 行（`chunksize=50000` 流式 + 逐 chunk float32），确保横截面 rank 基于完整市场横截面计算；`SET LOCAL statement_timeout = '600s'` 应对全量读取；
2. **Fit/Transform 分离**：新增 `fit_winsorize_bounds` / `transform_winsorize` / `transform_feature_matrix` / `transform_pca`，Phase E 复用 sample 拟合的预处理参数，不重新拟合；
3. **scaler 键修复**：`run_representation` 同时返回 `scaler` 与 `scaler_params`（同值），`transform_feature_matrix` 正确应用 RobustScaler；
4. **向量化时序派生**：`_compute_temporal_derivatives` 改用 `groupby().shift()` 向量化计算，消除 5177 次循环；
5. **内存优化**：`del` + `gc.collect()` 释放 sample 数据与 results X/labels；per-column `astype(np.float32)`；`pd.read_sql` chunksize 流式读取；
6. **manifest 修复**：`pca_summary` 只提取 `n_components` 与 `explained_variance_ratio`（JSON 可序列化）；
7. **reporting 参数契约修复**：`generate_report_md` 正确接收 `prevalence` / `dwell` 参数；
8. **mypy 修复**：`stability.py` 函数签名改为 `Sequence[Any] | np.ndarray`；
9. **新增 7 个测试**：`TestCrossSectionalRankFullSection`（4 个：完整横截面 rank 正确性、chunk 分割产生不同 rank、按 trade_date 分割结果一致、transform_feature_matrix 完整横截面）、`TestSampleVsFullAssignmentMode`（3 个：dry-run 提及两阶段、sample_rows 控制样本量、full assignment 使用 get_all_matrix_rows）；
10. **manifest 准确性**：`full_assignment` 字段记录 `rows` / `representation` / `method=get_all_matrix_rows` / `cross_section_scope=full_market_per_trade_date`；
11. **150k 样本 + 全量 Phase E 验证通过**：peak RSS 1242.55 MB < 1.5GB，570,011 有效行，k=3 stable（cross_sectional rep），输出 0.04 MB < 50MB，无 traceback。
