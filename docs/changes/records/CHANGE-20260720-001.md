## 2. 修改前理解

### 2.1 详情页 Node 输入错误（§一）

`indicator_service` 把 `WATCHLIST_MONITOR` 声明为只需要日线，但该 Monitor 内部包含 Volume Node，
因此不会稳定加载 Node 需要的 4000 根 15 分钟线。同时它把当前显示周期的 `macd_bars` 传入
`MarketDataContext.bars_daily`。切换到 15m/1h/1w/1mo 后，Node 收到的主数据可能不再是日线。
这正是"筹码共识价暂不可用"或不同周期 Node 不一致的主要代码风险。

### 2.2 SMC 只存在图表指标，没有进入盘中 Monitor（§二）

`WatchlistMonitor` 只组合 BB 和 VN，SMC 仅作为图表按需指标存在，未进入盘中监控体系。

### 2.3 飞书 Capture 仍然是混合指标视图（§三）

详情页 POST 请求没有请求体，用户无法选择 Node、BB 或 SMC；服务也会把 BB 和 Node 字段
混合生成文本和截图。

### 2.4 详情页发送飞书无指标选择（§四）

POST `/instruments/{id}/send-feishu` 无 body，无法指定指标视图。

### 2.5 全指标 Canonical 四链迁移仍未完全结束（§五）

CHANGE-20260719-001 §二 已建立 12 算法族 `production_wired` adapter 层，但四链
（详情/盘后/盘中/Capture）仍直接 import kernel 模块。AST 硬门禁
`test_four_chain_no_direct_kernel_import` 以 `xfail(strict=True)` 标记此目标，
`tests/allowlist.json` 记录 issue #83。

## 3. 修改目标

### 3.1 §一 详情页 Node 输入修复

1. 拆分 `MarketDataContext` 输入：`bars_display` / `display_timeframe` / `bars_daily` / `bars_15min` / `bars_minute`
2. 独立计算 Node Cluster，输出 `data.node_cluster`，固定输入 completed qfq 1d×250 + 15m×4000
3. 返回明确降级原因：`availability` + `degraded_reason`
4. 五周期一致性测试：1d/15m/1h/1w/1mo 切换时 `profile_hash` 完全一致

### 3.2 §二 日线 SMC 盘中监控

1. 新增 `backend/app/strategy/monitors/smc_monitor.py`
2. `WatchlistMonitor` 调整为 `self._bb` + `self._vn` + `self._smc` 三合一
3. SMC 计算合同：主输入已完成前复权日线；当天盘中结构固定；1 分钟线仅用于触发检测；
   调用现有 SMC Kernel；不加入 FVG
4. 第一版事件：`smc_bos_retest` / `smc_choch_retest` / `smc_order_block_first_touch`
5. 状态命名空间：`MonitorState` 升级为 `bb` / `node_cluster` / `smc` / `market`

### 3.3 §三 三种监控独立飞书图片

## 4. 实际修改

### 4.1 §一 拆分 MarketDataContext + 独立 node_cluster（commits fffcea1, 69e700e）

#### 4.1.1 `MarketDataContext` 字段拆分

```python
@dataclass
class MarketDataContext:
    bars_display: pd.DataFrame | None
    display_timeframe: str | None
    bars_daily: pd.DataFrame | None
    bars_15min: pd.DataFrame | None
    bars_minute: pd.DataFrame | None
```

- `bars_display`：当前页面周期，供 DSA、MACD、SQZMOM 及图表周期指标使用
- `bars_daily`：真正日线，只供日线结构算法
- `bars_15min`：Node 成交量分配辅助数据
- `bars_minute`：盘中事件触发数据

#### 4.1.2 `_REQUIRED_INPUTS` 修复

`WATCHLIST_MONITOR` 从 `frozenset({"daily"})` 改为 `frozenset({"daily", "15min"})`，
因为内部含 `VolumeNodeMonitor` 需要 15min。

#### 4.1.3 独立 `_compute_independent_node_cluster` 函数

新增函数固定输入 completed qfq 1d×250 + 15m×4000，输出 `data.node_cluster`：
- `profile_rows`：完整 100 行 VP 价格档位快照
- `profile_meta`：VP 元信息 + algorithm/schema/fingerprint/daily_hash/15m_hash/profile_hash
- `peak_rows`：Peak 节点快照（含 VA 外）
- `state`：当前价格状态
- `availability`：`available` | `degraded` | `unavailable`
- `degraded_reason`：`INSUFFICIENT_DAILY_BARS` | `MISSING_15M_BARS` | `PROFILE_EMPTY` | None

#### 4.1.4 五周期一致性测试

新增 `test_indicator_service.py::TestNodeClusterFivePeriodConsistency`：
同一股票切换 1d/15m/1h/1w/1mo，`profile_hash` 完全一致，`daily_source_hash` 完全一致，
`15m_source_hash` 完全一致，图表 bars frame hash 允许不同。

### 4.2 §二 日线 SMC 盘中监控（commit 2def073）

#### 4.2.1 新增 `backend/app/strategy/monitors/smc_monitor.py`

`SmcMonitor` 实现：
- 主输入：已完成、前复权日线（`bars_daily`）
- 当天盘中结构固定，不因 1 分钟价格变化重新计算
- 1 分钟线仅用于触发检测
- 调用现有 SMC Canonical Adapter（`compute_smc_adapter`），禁止复制公式
- 不加入 FVG

#### 4.2.2 第一版事件（三个）

| 事件类型 | 触发条件 |
|---|---|
| `smc_bos_retest` | 价格对已确认日线 BOS 结构位的回踩、触碰或穿越 |
| `smc_choch_retest` | 价格对已确认日线 CHoCH 结构位的回踩、触碰或穿越 |
| `smc_order_block_first_touch` | 价格第一次进入当前有效、未 mitigated 的日线 OB |

每个事件含稳定字段：`smc_entity_id` / `direction` / `boundary` / `zone` / `confirmed_at` /
`trigger_price` / `source_1m_bar_time`

去重键：`instrument + event_type + smc_entity_id + touch_episode`

#### 4.2.3 `WatchlistMonitor` 三合一

```python
self._bb = BollingerMonitor()
self._vn = VolumeNodeMonitor()
self._smc = SmcMonitor()
```

#### 4.2.4 `MonitorState` 命名空间升级

```json
{
  "bb": {},
  "node_cluster": {},
  "smc": {},
  "market": {
    "current_price": 0,
    "previous_close": 0,
    "change_pct": 0
  }
}
```

兼容读取旧平铺状态一次。

#### 4.2.5 `EVENT_LABELS` 扩展

新增 3 个 SMC 事件文案（以"价格"开头）：
- `smc_bos_retest` → "价格回踩 BOS 结构位"
- `smc_choch_retest` → "价格回踩 CHoCH 结构位"
- `smc_order_block_first_touch` → "价格首次进入订单块"

总数从 5 扩展到 8（4 监控 + 3 SMC + 1 快照分享）。
