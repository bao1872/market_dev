# CHANGE-20260707-041: Indicator Overlay Final Alignment

## 变更编号
CHANGE-20260707-041

## 任务名称
Indicator overlay 最终对齐修复（DSA VWAP 15m/1h 误禁用 + 多周期 BB 图层错位）

## 需求出处
PR #30 修复 DSA overlay source alignment 后，前端 15m/1h DSA VWAP 仍被禁用/不渲染，多周期 BB 目测不对（15m/1h BB 是日线阶梯线而非当前周期 BB）。用户要求从代码和 API 查根因，不要再说"已修复"。

根因（三处）：
1. **Redis cache schema 未 bump version**：PR #30 修改了 `source_bar_times` 格式（按 timeframe）和 `indicator_service` 15m/1h 改用 `macd_bars` 计算 source 字段，但 `ALGORITHM_VERSION` 仍为 `v3`，旧 v3 缓存被命中返回旧格式 `source_bar_times`（日线日期格式）和日线阶梯线 BB，导致 15m 仍触发 DSA source mismatch。
2. **15m/1h BB 是日线阶梯线**：`indicator_service._adapt_watchlist_bb` 在 15m/1h 路径用 `_map_daily_to_intraday` 把日线 BB 映射到日内时间序列（阶梯线），并非当前周期 BB。
3. **15m/1h 误报 DSA mismatch**：DSA overlay 按钮在 15m/1h 被 disabled 后，mismatch 校验仍执行，source_bar_times 为空或日线格式时误报 "DSA 数据源不一致"。

## 修改前行为
- `indicator_cache.ALGORITHM_VERSION = "v3"`，PR #30 修改 source 格式后旧 v3 缓存仍被命中；
- `_adapt_watchlist_bb` 15m/1h 路径用 `_map_daily_to_intraday` 映射日线 BB 到日内时间序列（阶梯线），非当前周期 BB；
- DSA overlay 在 15m/1h 下：按钮被 disabled，但 mismatch 校验仍执行，15m source_bar_times 为空或日线格式时误报 "DSA 数据源不一致，已暂停渲染"；
- 无 overlay 对齐诊断工具，难以定位 15m/1h overlay 错位根因。

## 修改后行为
- `indicator_cache.ALGORITHM_VERSION = "v4"`，旧 v3 缓存 key 不匹配新 `build_cache_key`，强制重算，避免旧格式 `source_bar_times` 和日线阶梯线 BB 污染渲染；
- `_adapt_watchlist_bb` 15m/1h 路径改用 `compute_bollinger(macd_bars, length=20, mult=2.0)` 重新计算 BB upper/mid/lower/pos/width，字段映射 `bb_pos_01`→`bb_pos`、`bb_width_norm`→`bb_width`，NaN 转 `None`；`len(macd_bars) < 20` 时 BB 字段填 `None`，`time` 数组仍与 `macd_bars` 对齐；
- DSA overlay 周期策略：DSA 是日线级别结构锚，**仅 1d 周期渲染**；15m/1h DSA 按钮 disabled 并提示 "DSA VWAP 当前仅支持日线结构锚；15m/1h 请使用 Swing、BB、SQZMOM。"；`shouldCheckDsaMismatch(timeframe)` 仅 1d 返回 `true`，15m/1h 返回 `false`，避免误报 mismatch；右侧 `StockStructuralStatePanel` 仍可显示 daily DSA 背景和 m15 response；
- 新增 `?debugIndicatorAlignment=1` URL 参数诊断工具：`console.table` 输出 bars（timeframe/count/first/last/canonical_first/canonical_last）、dsa_mismatch（check_enabled/mismatched/source_bar_hash/source_bar_times_count）、indicators.layers（layer_id/renderer/fields/time_count），默认不打印。

## 风险
- `ALGORITHM_VERSION` bump v3→v4 会使所有 indicator 缓存失效，首次请求需重算（单只股票 < 1s，可接受）；
- 15m/1h BB 数值会变化（从日线阶梯线变为当前周期 BB），用户视觉上会看到 BB 形状变化（这是修复目标，非回归）；
- DSA overlay 在 15m/1h 下不再渲染（disabled），但右侧结构状态因子面板仍显示 daily DSA 背景和 m15 response，不影响结构状态展示；
- `?debugIndicatorAlignment=1` 仅诊断用，不影响生产渲染逻辑；
- 性能：无新增表/worker/缓存；只单只股票 API 计算路径调整。

## 遗留问题
- 无。本次修复 DSA VWAP 误禁用和 BB 图层错位两个问题，bar 因子/Temporal 收尾检查为后续任务。
