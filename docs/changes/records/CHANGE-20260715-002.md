## 修改前行为
1. SMC 算法基于 ref/smc.py（Python 重写版），ATR 使用 SMA（rolling mean），CMR 除数用 i+1（bar 0 有值），warmup 不足（1d 仅 250 根，ATR200 预热不足）
2. smc_indicator.py 为 957 行单体模块，无独立 Pine 语义核心
3. SMC 输出 events 使用 kind 字段（'internal'/'swing'），前端 SmcEvent 接口也用 kind
4. StrategyChart SMC renderer：CHoCH=虚线、BOS=实线（按事件类型区分线型），标签左对齐，trailing 文案用 SH/SL
5. MiniKline viewport 使用 fitContent，五周期 clamp（15m/60m 50-64, 日线 48-58, 周线 40-52, 月线 30-40），右侧 3 bar 留白
6. indicator_service SMC 计算使用 macd_bars（1d=250 根），SMC 输出被 _truncate_lists 截断
7. indicator_cache ALGORITHM_VERSION=v6

## 修改后行为

### 一、smc_pine_core.py（唯一 Pine 语义核心，852 行）
1. 新增 `backend/app/strategy_assets/algorithms/features/smc_pine_core.py`
2. Pine 语义原语实现：
   - `pine_rma(src, length)`: Wilder's RMA，SMA 播种 + 递推 `rma[i]=(rma[i-1]*(length-1)+src[i])/length`
   - `pine_true_range(highs, lows, closes)`: Pine ta.tr
   - `pine_atr(highs, lows, closes, length)`: `ta.atr(n) = ta.rma(ta.tr, n)`
   - `pine_cumulative_mean_range(highs, lows, closes)`: `ta.cum(ta.tr)/bar_index`，bar 0 = NaN（除零）
   - `pine_highest/pine_lowest`: 滚动极值（不含 ref_i bar）
   - `pine_crossover/pine_crossunder`: 穿越检测
3. 默认参数逐项匹配原始 Pine：
   - Historical, Colored
   - internal structure=true, size=5, All, confluence=false, tiny
   - swing structure=true, length=50, All, small
   - Strong/Weak=true
   - internal OB=true, 5, ATR, High/Low
   - swing OB=false
   - EQH/EQL=true, bars=3, threshold=0.1, tiny
   - swing points/MTF/premium-discount=false
4. `_SMCPineState` 状态机：逐 bar 运行，Pine 执行顺序（swing→internal→equal→BOS/CHoCH→trailing→mitigation）
5. FVG 完全排除：不计算、不返回、不缓存、不渲染，不提供 FVG 开关
6. anchor/confirmed 因果契约：每个事件含 anchor_index/time 与 confirmed_index/time
7. events 使用 `internal: bool`（true=internal, false/缺失=swing），替代旧 `kind` 字段
8. `compute_smc_pine()` 公开 API

### 二、smc_indicator.py（薄包装层，117 行）
1. 重构为薄包装，委托到 `smc_pine_core.compute_smc_pine`
2. `_SMCState = _SMCPineState` 向后兼容别名
3. `compute_smc_indicators()` 签名不变，直接委托

### 三、indicator_service warmup 修复
1. 1d timeframe 使用 `full_daily_bars`（DB 全量日线，≥500 warmup）
2. 其他周期复用 macd_bars（15m≈12000、1h≈3000、1w≈714、1mo≈166，均为可获得最大历史）
3. 不调用 `_truncate_lists` 截断 SMC 输出：time 数组需保持完整长度以对齐 anchor/confirmed 索引
4. 前端通过 smcToDisplay 映射自动过滤展示区外事件

### 四、indicator_cache 版本升级
1. ALGORITHM_VERSION v6→v7（旧 v6 SMC 缓存基于 SMA 实现，必须强制失效）
2. `:smc` 后缀隔离不变（SMC 与非 SMC 独立缓存）

### 五、StrategyChart SMC renderer 对齐
1. SmcEvent 接口：`kind?` → `internal?: boolean`
2. SmcOrderBlock 接口：新增 `internal?: boolean`
3. BOS/CHoCH 线型：internal=虚线 dash=[4,3]，swing=实线（不再按 BOS/CHoCH 区分）
4. 标签位置：从左端 `x1+2` 改为中点 `(x1+x2)/2`，对齐方式 `'center'`
5. 标签字号：internal=8px (tiny)，swing=11px (small)
6. trailing 文案：SH/SL → 强高/弱高/强低/弱低
7. 分类规则：强高 if swingBias===-1 else 弱高；强低 if swingBias===1 else 弱低
8. OB box：半透明矩形（alpha 0.12），mitigated OB 进一步降低（alpha 0.05）
9. 颜色：多头红 #FF4D4F，空头绿 #22C55E（A 股红涨绿跌）
10. Historical 模式：绘制全部事件（不因标签碰撞删除）

### 六、MiniKline viewport 彻底重写
1. 目标根数按周期固定：15m=48、60m=44、日=40、周=36、月=30
2. barSpacing clamp 5.5–8px（窄宽度时减少根数）
3. 左侧 1-2 根留白：`from=max(-2, n-visible-1)`
4. 右侧 3 根留白：`to=n-1+3`
5. 不调用 fitContent、resetTimeScale 或 scrollToRealTime

9. 切周期不复用旧 logical range
