## profile_meta 诊断字段（Node Cluster）

> `profile_meta` 由 `prepare_node_cluster_bars` 返回，用于诊断 Node Cluster 行情准备阶段的输入与参数版本。
> `input_*_bars` 为运行时实际根数（受数据量限制可能少于下方期望值），期望值取自 indicator_contract 常量。

| 字段 | 期望值 / 来源 | 说明 |
|------|--------------|------|
| `input_daily_bars` | 250（NODE_CLUSTER_PRIMARY_BARS） | 准备后的日线根数（运行时实际值） |
| `input_15m_bars` | 3600（NODE_CLUSTER_LOW_BARS） | 准备后的 15m 根数（运行时实际值） |
| `input_minute_bars` | 2（NODE_CLUSTER_MINUTE_BARS） | 准备后的 1m 根数（运行时实际值） |
| `primary_period` | 1d（NODE_CLUSTER_PRIMARY_PERIOD） | 主周期 |
| `low_period` | 15m（NODE_CLUSTER_LOW_PERIOD） | 低周期 |
| `parameter_version` | v1.1.0 | 参数版本标识（Node Cluster 参数变更时同步更新） |

## 图表行情输入参数

> `load_chart_bars` 服务（`backend/app/services/chart_bars_service.py`）统一为 `/bars` API 和 `indicator_service` 提供 250 根日线行情输入。
> 数据源：DB 优先 + Pytdx 兜底，前复权 + DatetimeIndex 升序 + 去重 + 未完成 Bar 过滤。

| 参数名 | 值 | 说明 |
|--------|-----|------|
| CHART_BARS_COUNT | 250 | 图表行情输入根数（与 DSA_LOOKBACK、INDICATOR_BARS['1d'] 一致） |

**诊断字段**（`indicator_service` 响应）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `source_bar_times` | list[str] | 250 个 ISO 日期字符串（行情输入时间序列） |
| `source_bar_hash` | str | OHLCV 拼接的 SHA256 哈希前 16 位（数据源指纹） |

## 刷新时点

- 日线 / 15m / 60m：每个交易日 16:00 定时刷新（CronTrigger hour=16 minute=0, Asia/Shanghai）
- 周线 / 月线：不存储 DB，从日线动态合成（convert_kline_frequency），不参与定时刷新
- 1m：不在定时调度范围（调度仅覆盖 d/15m/60m），由监控策略实时拉取
