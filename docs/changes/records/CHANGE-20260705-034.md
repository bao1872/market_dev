# CHANGE-20260705-034

## 变更编号

CHANGE-20260705-034

## 任务名称

Admin 监控资格修复 + 个股详情实时行情可信化

## 需求出处

- 用户指令：在 `fix/admin-monitor-and-realtime-quote` 分支按 TDD 修复 admin 自选股被监控过滤、个股详情实时行情伪实时两个问题。
- 关键约束：
  - 不部署生产，只做代码、测试、文档；
  - 不恢复 stash 中的结构因子/时序特征等无关改动；
  - admin monitor 资格链路必须三处口径一致（monitor_batch/event_recipient/outbox_relay）；
  - `/quote` 必须以可信状态返回，daily_fallback 不得在 UI 显示成实时；
  - `mergeRealtimeQuoteIntoBars` 只合并可信实时 quote；
  - 午休口径统一复用 `market_status_service.compute_market_session`；
  - 删除 `1m` 配置；
  - 先采集单机性能，不盲目提高轮询频率；
  - pytdx 单例/线程锁/Redis 短缓存必须有失败保护；
  - 完成后汇报，不部署。

## 修改前行为

- 旧 `filter_eligible_recipients` 要求 `member + 非 admin + 有效 subscription`，导致 active admin 自选股被 `monitor_batch_service`/`event_recipient_service`/`outbox_relay` 过滤，无法进入监控 universe、无法接收监控事件/通知。
- `/api/v1/instruments/{instrument_id}/quote` 未返回 `source`/`is_realtime`/`freshness_seconds`/`degraded`/`degraded_reason`，前端固定显示“实时行情”，无法区分 pytdx 实时、日线 fallback、数据延迟、行情降级。
- `frontend/src/utils/chart.ts::mergeRealtimeQuoteIntoBars` 未校验 quote 可信度，会把 daily_fallback 合入 K 线。
- `market_data_aggregation_service` 和 `/quote` 各自写 9:30-15:00 连续判断，午休口径与 `/market/status` 不一致。
- `StockDetailPage` 工具栏残留 `1m: 2` 配置，但后端 bars API 不支持 1m。

## 修改后行为

### Admin 监控资格

- 新增 `app.services.eligible_user_service.filter_monitor_eligible_recipients` 与 `is_user_eligible_for_monitor`：
  - active admin 直接放行；
  - active member + 有效 subscription 放行；
  - disabled admin、无订阅普通用户排除。
- `monitor_batch_service._resolve_monitor_users` 改用 `filter_monitor_eligible_recipients`。
- `event_recipient_service.expand_event_recipients` 对监控来源事件改用监控资格过滤。
- `outbox_relay._expand_notification_message_created` 对 `monitor_event` 来源使用监控资格过滤，其他来源保持原有资格逻辑。

### 实时行情可信化

- `backend/app/schemas/bar.py::QuoteResponse` 新增字段：`source`（`"pytdx" | "daily_fallback"`）、`is_realtime`、`freshness_seconds`、`degraded`、`degraded_reason`。
- `backend/app/api/bars.py::get_instrument_quote` 重构：
  - 仅当 `market_status_service.compute_market_session` 返回 `MORNING_SESSION`/`AFTERNOON_SESSION` 时尝试 pytdx；
  - pytdx 成功 → `source="pytdx"`、`is_realtime=true`、`degraded=false`；
  - 非交易时段 fallback → `source="daily_fallback"`、`is_realtime=false`、`degraded=false`；
  - 交易时段 pytdx 失败 → `source="daily_fallback"`、`is_realtime=false`、`degraded=true` 并记录原因；
  - 使用 Redis 短缓存（10s TTL）削峰；
  - pytdx 同步调用通过线程池提交到 async event loop，模块级单例适配器 + 线程锁防止多线程同时操作 socket，连接异常时支持断线重连与超时控制。
- `backend/app/services/market_data_aggregation_service.py` 改用 `compute_market_session` 统一判断实时交易时段，消除独立 9:30-15:00 判断。

### 前端可信展示

- `frontend/src/api/endpoints.ts` 扩展 `QuoteResponse` 与 `BarListResponse` 类型，匹配后端可信字段。
- `frontend/src/pages/StockDetailPage.tsx`：
  - 移除 `1m: 2` 配置；
  - 顶部报价条旁新增行情状态徽章（实时行情 / 日线回退 / 数据延迟 / 行情降级）并显示 `update_time`；
  - 新增 K 线状态条展示 `data_source`/`as_of`/`is_partial`/`degraded`/`degraded_reason`。
- `frontend/src/utils/chart.ts::mergeRealtimeQuoteIntoBars` 增加可信合并条件：
  - `quote.is_realtime === true`
  - `quote.source === "pytdx"`
  - `quote.freshness_seconds <= 60`
  - 不满足时只用于顶部报价 fallback/状态提示，不混入 K 线；
  - 1d 保留日期语义，intraday 使用 `quote.update_time`。
- `frontend/src/hooks/useApi.ts`：`useRealtimeQuote`/`useBars`/`useIndicators` 均设置 `refetchIntervalInBackground: false`，页面 hidden 时停止后台轮询。
