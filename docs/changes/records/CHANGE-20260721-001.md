## 2. 修改前理解

### 2.1 Node 永久"指标加载中"（§一，Phase 2 已完成主体）

`ChartRenderFrame` 错误地把"图表展示帧"与"算法计算输入帧"视为同一个对象。指标接口可能使用更长 warmup、不同 completed-only 口径或独立日线/15m 输入，而前端却严格比较完整 source hash 与首末时间，导致请求早已完成但 frameMismatch 永久为 true。

### 2.2 "筹码共识价暂不可用"（§二，Phase 2 剩余）

Feature Snapshot 不写入 Canonical Node 结果（`availability`/`degraded_reason`/`profile_hash`/`daily_source_hash`/`bars_15m_source_hash`），StockContext 无法区分 NODE_PROFILE_EMPTY/NODE_15M_MISSING/NODE_COMPUTE_FAILED/NODE_INSUFFICIENT_DAILY_BARS，全部显示成"暂不可用"。Atomic Facts 中的"筹码共识价"与详情页 Node 没有共享同一个 Canonical 结果。

### 2.3 indicator_view 不生效（§三，Phase 4 已完成主体）

Capture URL 含 `indicator_view` 参数但 `CaptureStockPage` 未读取、Snapshot API 未接收、`StrategyChart` 未传 `layerVisibility`，导致选项形同虚设，图片仍是布林带+筹码峰混合 PC 截图。

### 2.4 移动飞书舞台（§四，Phase 4 已完成主体）

当前 1920×1200 PC 截图，手机阅读字号小、信息密度失衡。需移植为 1440×2560 9:16 移动舞台。

### 2.5 复权闭环 FR-11 缺失（§五，Phase 3 发现）

`rebuild_factor_series` 只失效 MDAS 缓存（`mdas:{instrument_id}:*`），不失效 `bars` 和 `indicator` 缓存。因子变化后，indicator 缓存（TTL 300s）会继续提供基于旧因子的指标结果，导致短暂不一致。

## 3. 修改目标

### 3.1 Phase 2 剩余：Feature Snapshot 写入 Canonical Node 字段

1. Feature Snapshot 必须写入 Canonical Node 结果：`poc_price`/`profile_hash`/`availability`/`degraded_reason`/`daily_source_hash`/`bars_15m_source_hash`
2. Bump Snapshot `_SCHEMA_VERSION` 3→4（旧快照不可见，符合"旧新结果不可混用"）
3. StockContext 区分 5 态：NO_PUBLISHED_RUN / SNAPSHOT_MISSING / NODE_PROFILE_EMPTY / NODE_15M_MISSING / NODE_COMPUTE_FAILED / NODE_INSUFFICIENT_DAILY_BARS
4. Atomic Facts 中的"筹码共识价"和详情页 Node 必须消费同一个 Canonical 结果

### 3.2 Phase 3：复权闭环 FR-11 修复

1. `rebuild_factor_series` 精确失效该股票下游缓存：MDAS + bars + indicator
2. 单层失效失败不阻塞其他层（缓存 TTL 会自然过期）
3. 监控 Profile（in-process）和 Capture（filesystem per-event）依赖 TTL 自然过期

### 3.3 Phase 4：indicator_view 真正控制 Capture（已在前序会话完成）

1. `CaptureStockPage` 读取 `indicator_view` URL 参数
2. Snapshot API 接收 `indicator_view`，驱动 `include_smc`
3. `StrategyChart` 使用 `INDICATOR_VIEW_LAYER_PRESETS` 替代默认全图层
4. `MobileIndicatorStage` 1440×2560 9:16 移动舞台组件

### 3.4 Phase 5：测试验证

1. 后端单元/集成测试全绿
2. ruff/mypy 无新增错误
3. 前端测试待 Phase 6 Docker 构建时运行

### 5.7 兼容性和回滚

- `_SCHEMA_VERSION` 3→4 后，旧快照不可见，需要重新运行 after_close 流水线生成新快照
