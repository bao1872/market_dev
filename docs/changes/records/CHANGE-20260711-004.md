# CHANGE-20260711-004: 统一行情工作区原型最终对齐（阶段五）

## 变更编号
CHANGE-20260711-004

## 任务名称
统一行情工作区原型最终对齐：入口统一、研究上下文面板、管理员调试、视觉响应式、清理旧页面

## 需求出处
advice.md 阶段五：按三张原型 PNG 完成统一行情工作区剩余全部原型对齐任务。

## 修改前行为
- `ScreenerPage` 查看详情进入 `/stock/:symbol`（通过 `navigate(state.returnTo)`）
- `MessagesPage` 单只股票消息进入 `/stock/:symbol`
- `/market` URL 状态只有 scope/symbol/timeframe/source/strategy/event_id（无 debug/returnTo）
- `/market` 右栏为 `StockStructuralStatePanel`（仅结构状态因子，无事件解释）
- `event_id` URL 参数仅解析、保留并传递，未被工作区消费
- 无管理员调试面板（普通用户和管理员看到相同内容）
- `StockStructuralStatePanel` 无 debug props
- 旧 `WatchlistPage.tsx` 和 `IndexPage.tsx` 仍存在（死代码，不在路由中）
- `MarketWorkspace.module.scss` 右栏固定宽，无 debugMode 样式

## 修改后行为
- `ScreenerPage` 查看详情改进入 `/market?scope=market&symbol=...&source=selection&strategy=dsa_selector&returnTo=...`（`buildMarketEntryFromScreener`）
- `MessagesPage` 有股票时进入 `/market?symbol=...&event_id=...`（`buildMarketEntryFromMessage`）；无股票保持消息页
- `/stock/:symbol` 兼容旧链接并保留原功能；返回按钮优先 URL `returnTo`，其次 `location.state.returnTo`，最后按 source fallback
- `/market` URL 状态扩展为 scope/symbol/timeframe/source/strategy/event_id/debug/returnTo；左栏选新股清 event_id 和 returnTo（保留 debug）
- `/market` 右栏改为 `ResearchContextPanel`（`features/research-context/`）：
  - `event_id` 存在时调用 `useStrategyEventDetail` 显示事件时间、类型（通俗解释）、关联价格、关键证据
  - 无 `event_id` 时展示 `useStructuralFactors`/`useTemporalFeatures` 的人类可读总结
  - 普通用户不显示内部字段名、算法参数、JSON 或商业机密
  - 面板关闭时不挂载 hooks、不发请求
- 管理员 `accessProfile.is_admin && URL debug=1` 时额外渲染 `AdminFactorDebugPanel`：
  - 展示原始 factor/feature/timeframe/bar_time/版本/数据源/warmup/degraded 信息和可折叠 JSON
  - 普通用户即使手工加 `debug=1` 也不可见
- `StockStructuralStatePanel` 新增 `debug?: boolean` props，debug 时显示原始 structural-factors JSON
- `MarketWorkspace.module.scss` 按三张原型 PNG 重做 CSS：左栏 280px（1024px 时 240px）、中栏 flex:1 min-width:480px、右栏 340px（1920px 时 380px、debug 模式 380px）；所有交互元素添加 `:focus-visible` 轮廓
- 旧 `WatchlistPage.tsx` 和 `IndexPage.tsx` 已删除（死代码确认不在 App.tsx 路由中）
- `frontend/scripts/contract-tests/index-page-columns.test.ts` 已删除（整个文件针对 IndexPage）
- `contract.test.ts` 和 `columns.test.ts` 删除引用已删除文件的测试，保留 ScreenerPage 和共享模块测试
