## 需求出处

## 修改前行为
- 普通用户与管理员共用同一 `AppShell`（侧栏 + 顶栏），侧栏含"主页/趋势选股/我的自选/消息中心/通知与设置"五项。
- 主入口为 `/overview`（IndexPage）；`/watchlist` 为独立自选页。
- `AdminRoute` 非 admin 重定向到 `/overview`；兜底路由 `*` 重定向到 `/overview`。
- 登录注册成功 / 续期成功均跳转 `/overview`。
- 消息/设置在侧栏；无账户菜单。

## 修改后行为
- 普通用户主入口改为 `/market`（行情，本阶段直接复用 `WatchlistPage`）；`/overview` 重定向到 `/market`，`/watchlist` 重定向到 `/market?scope=watchlist`。
- 普通用户使用 `UserAppShell`（顶栏品牌 + 一级导航"行情/趋势选股" + 右上角 `AccountMenu`；无左侧栏）。
- 管理员使用独立 `AdminAppShell`（侧栏管理导航 + 账户菜单），仅承载 `/admin/*`。
- `ProtectedLayout` 只负责认证与 access profile，不再固定渲染壳层，返回 `<Outlet/>`。
- `AccountMenu` 下拉含消息、设置、管理后台（仅 `is_admin`）、退出；支持点击外部 / Escape 关闭、基本 ARIA。
- `AdminRoute` 非 admin 重定向到 `/market`；兜底路由 `*` 重定向到 `/market`。
- 登录注册成功 / 续期成功跳转改为 `/market`。
- `/capture/stock/:symbol` 继续位于两套壳层之外。
- 导航/路由常量集中于 `frontend/src/navigation/appNavigation.ts`。
- `IndexPage`/`WatchlistPage`/`ScreenerPage`/`StockDetailPage` 页面文件与 API 不删除、不重构。
