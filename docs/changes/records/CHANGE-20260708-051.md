## 变更编号
CHANGE-20260708-051

## 任务名称
修复 capture worker 偶发 502（page.goto networkidle 超时）导致 monitor 图片缺失

## 需求出处
生产环境观察：2026-07-08 最近 24h 内 `capture_jobs` 34 条 `failed`、5 条 `succeeded`；失败记录 `error_code=CAPTURE_REQUEST_FAILED`，`error_message` 为 `502 Bad Gateway`。

## 修改前行为
`app.services.stock_capture_service.capture_stock_chart` 调用 `page.goto(url, wait_until="networkidle", timeout=30000)`。前端 `/capture/stock/:symbol` 页面若存在长连接、持续轮询或慢请求，`networkidle` 事件可能永远不会触发，导致 30s 超时，capture worker 返回 HTTP 502，`capture_jobs.status=failed`，对应 monitor 事件只有文字通知、没有图片。

## 修改后行为
`page.goto` 改为 `wait_until="load"`，并在页面 load 后通过 `page.wait_for_selector('[data-render-ready="true"]')` 等待业务数据（bars + indicators）渲染完成。该选择器已不依赖事件查询，只等 Snapshot API 数据加载，因此能可靠触发截图。
