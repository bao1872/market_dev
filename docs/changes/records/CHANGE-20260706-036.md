# CHANGE-20260706-036: Monitor 投递与 live bar 后续修复

## 变更编号
CHANGE-20260706-036

## 任务名称
Monitor 投递与 live bar 后续修复

## 需求出处
PR #25 已合并，但远端 main 仍存在：
1. `delivery_worker.py` 对 `monitor_event` 仍可能走普通 `is_user_eligible`，admin 自动监控投递会被排除；
2. `monitor_batch_service.py` 的 1m 仍是 `include_realtime=False`；
3. `MarketDataAggregationService` 的 1d 仍没有交易时段 partial daily bar；
4. `/quote` 时区需确保前端显示上海时间。

根因：`8c991e3d` 统一 MDAS 后，旧 `/bars` 1d 实时合并语义未完整迁移；后续 PR #25 修复 quote 可信化，但未恢复 1d partial bar，导致个股详情 K线实时只能依赖前端 `mergeRealtimeQuoteIntoBars()` 视觉补丁。

## 修改前行为
- `delivery_worker.py` 对 monitor_event 使用普通会员资格，active admin 被误标 `USER_INELIGIBLE`；
- `monitor_batch_service.py` 1m 输入使用 DB 历史分钟线，无盘中 live 数据；
- 1d bars 在交易时段只返回昨日收盘前已完成日线，不反映盘中价格；
- `/quote` 可能返回 naive datetime 或 `+00:00` 字符串，前端按本地时区解析可能显示为纽约时间。

## 修改后行为
- `delivery_worker.py` 复用 `MONITOR_SOURCE_TYPES`（`app/constants/monitor_source_types.py` 单点真源，与 `outbox_relay.py` 共享），投递前调用 `is_user_eligible_for_monitor` 复核；active admin 与 active member + 有效 subscription 放行；disabled admin / 无订阅普通用户标记 `dead/USER_INELIGIBLE`；`stock_detail_share` 仍跳过资格；`beta_application_admin` 仍跳过 subscription。
- `monitor_batch_service.py` 1m 输入改为 `include_realtime=True` 并剔除最后一根未完成 bar；日线/15m 输入保持 `include_realtime=False`；`MonitorCycleResult` 新增 `last_minute_is_partial`；cycle done 与单标的日志输出 `instrument/symbol/source_bar_time/minute_data_source/minute_is_partial/events_detected/events_written`。
- `market_data_aggregation_service.py` 在 `timeframe=1d && include_realtime=true && 交易时段（MORNING_SESSION/AFTERNOON_SESSION）` 时，用当日已完成 1m bar 合成 partial daily bar 追加到响应末尾，返回 `data_source=hybrid`、`is_partial=true`、`last_live_bar_time`；非交易时段、收盘后、`include_realtime=false` 时不合成；不写库。
- `backend/app/api/bars.py` 与 `backend/app/core/pytdx_adapter.py` 对 naive datetime 和 `+00:00` 字符串统一按 Asia/Shanghai 解释，确保 `/quote.update_time` 带 `+08:00`。
- `AGENTS.md` 新增 `### 13. 个股详情 K线实时契约`，把 `/bars?timeframe=1d&include_realtime=true` 固化为个股详情 K线实时的唯一后端契约，明确 `mergeRealtimeQuoteIntoBars()` 只能做兜底视觉增强。
