# 产品与业务规则

## 1. 产品定位

V1.1 交易平台是面向 A 股个人投资者的多用户选股与盘中监控平台，核心能力：

- **每日选股**：基于趋势稳定性算法的盘后全市场选股
- **盘中监控**：基于布林带 / Volume Node Cluster 的 1m bar 实时事件检测
- **消息投递**：通过飞书 Webhook / 平台应用将事件实时推送给用户
- **会员体系**：邀请码注册 + 30 天会员有效期，到期后允许登录但仅可续期

## 2. 用户角色（RBAC）

事实源：`backend/app/models/user.py` + `backend/app/core/deps.py`

| 角色 | 名称 | 权限范围 |
|------|------|----------|
| `admin` | 超级管理员 | 全部 API + `/admin/*` 管理端点 |
| `user` | 普通用户 | 用户端 API（选股查询、自选股、监控状态、消息、设置） |

**安全约束**：
- 用户 ID 由 JWT 上下文注入，不接受客户端传入
- 私有资源（自选股、消息、渠道）按 `user_id` 严格隔离
- `status` 非 `active` 的用户拒绝登录与访问

## 3. 用户流程

### 3.1 注册与登录

事实源：`backend/app/api/auth.py`

```
注册：邀请码 → 校验 → 创建账户（active）+ 分配 user 角色 → 开通 30 天会员 → 生成 token
登录：email + password → bcrypt 校验 → 检查 status=active → 查询会员状态 → 生成 access + refresh token
续期：登录态 + 邀请码 → 未到期顺延 30 天 / 已到期从当天计算 30 天
```

**会员到期处理**：
- 允许登录，返回 `membership_expired=true`
- 前端跳转 `/membership-expired` 续期页
- 仅可访问续期相关端点

### 3.2 选股查询

事实源：`backend/app/api/strategy_runs.py` + `backend/app/services/selector_query_service.py` + `frontend/src/pages/ScreenerPage.tsx`

```
1. 用户访问 /screener 页面
2. 前端调用 GET /strategies/{key}/published-runs?limit=1 获取最新一个已发布批次
3. 前端调用 GET /strategies/{key}/results 查询该批次结果（支持 metric_filters / sort_by）
4. 普通用户仅能查询 status=published 的 run 结果
5. 表头筛选只作用于已发布全量结果，不影响计算和发布
```

**选股页口径**（advice.md 最新口径）：
- 只展示最新一个 published run，删除日期/批次下拉和回补入口
- 已删除 `run_type` 参数、回补 API/service/schema/router 及相关测试引用
- 筛选操作符：`gt` / `gte` / `lt` / `lte` / `eq` / `between`
- 单位换算：`ratio`（小数→百分比）、`percent`、`percentile` 字段在筛选时自动归一化

### 3.3 自选股与监控

事实源：`backend/app/api/watchlist.py` + `backend/app/services/monitor_batch_service.py`

```
1. 用户在 /watchlist 加入自选股（active=true）
2. 监控 Worker 在交易时段每 30 秒执行一轮监控
3. 监控范围 = 所有用户自选股去重 + watchlist_monitor 策略执行
4. 检测到事件 → 写入 strategy_events + outbox → 创建通知消息
5. 用户在 /messages 查看消息，或通过飞书渠道接收推送
```

### 3.4 设置页

事实源：`frontend/src/pages/SettingsPage.tsx`

用户可配置：
- **我的通知渠道**：飞书 Webhook / 飞书平台应用
- 渠道验证：POST `/notification-channels/{id}/verify`
- 渠道测试：POST `/notification-channels/{id}/test`

> 占位说明：用户级通知规则将在后续版本支持（详见 spec `fix-homepage-timezone-and-fake-features`）。

## 5. 业务边界

### 5.1 选股（Selector）

- **触发时机**：交易日 16:00 日线刷新完成后自动触发 DSA，18:30 兜底调度
- **执行位置**：后端 Worker（`strategy_batch`），不在 HTTP 请求内
- **状态机**：`queued → running → completed/partial_failed → published/failed`
- **自动发布门禁**：状态=completed + 覆盖率≥80% + failed_count=0
- **幂等键**：`strategy_key:strategy_version_id:trade_date:run_type:attempt_no`

### 5.2 监控（Monitor）

- **触发时机**：交易日 9:30-11:30 / 13:00-15:00，每 30 秒一轮
- **执行位置**：后端 Worker（`monitor_scheduler`）
- **监控范围**：所有用户自选股去重
- **事件冷却**：同一 `instrument_id + event_type + boundary` 在 600 秒内不重复写入
- **评估幂等**：`(策略版本, 股票, bar时间)` 唯一，`INSERT ON CONFLICT DO NOTHING`
- **session 聚合**：每个上午/下午只创建一条 SchedulerJobRun

### 5.3 消息投递（Delivery）

事实源：`backend/app/services/outbox_relay.py` + `backend/app/services/delivery_worker.py`

- **Outbox 模式**：业务写入与 Outbox 记录同事务，保证一致性
- **两阶段投递**：Outbox Relay 扩张为 MessageDelivery → Delivery Worker 实际投递
- **状态机**：`pending → sending → success/failed → retrying/dead`
- **重试退避**：`2^n * 30s`，最大 3 次
- **静默时段**：22:00-08:00 不投递，仅站内可见
- **消息组**：同一事件的 text+image 两条投递通过 `message_group_id` 关联
- **截图缓存**：按 `event_id + instrument_id + chart_version` 缓存 5-15 分钟

### 5.4 飞书渠道

事实源：`backend/app/services/feishu_webhook_adapter.py` + `feishu_platform_app_adapter.py` + `backend/alembic/versions/041_notification_channel_active_unique.py`

| 渠道类型 | 配置字段 | 适用场景 |
|---------|---------|---------|
| `feishu_webhook` | `webhook_url` + `sign_secret` | 群机器人（HMAC-SHA256 签名） |
| `feishu_platform_app` | `app_id` + `app_secret` + `receive_id` + `receive_id_type` | 个人用户（tenant_access_token） |

**唯一活跃渠道约束**（advice.md 最新口径）：
- 每个用户最多只能有 **1 个 active 飞书渠道**（`feishu_webhook` 或 `feishu_platform_app`）
- 由 `notification_channels` 表上的部分唯一索引 `uq_notification_channels_active_feishu` 强制（`user_id` 维度，`WHERE status='active' AND adapter_type IN ('feishu_webhook','feishu_platform_app')`）
- 详情页发送飞书时后端自动选择当前用户唯一 active 渠道，前端不再提供渠道下拉
- Outbox 投递使用 `target_channel_id` 精确指向单条渠道，禁止广播到全部渠道

**脱敏规则**：`app_secret` / `sign_secret` 在 API 读取时仅显示末 4 位。

**普通用户发送权限**：普通用户可发送自己的飞书（手动触发始终附带已有备忘录）；自动触发才受 `notify_feishu` 控制。

### 5.5 截图服务

事实源：`backend/app/services/stock_capture_service.py` + `backend/app/capture_main.py`

- **触发**：Delivery Worker 投递图片消息时调用
- **流程**：访问 `/stock/{symbol}?capture=feishu&token=<capture_token>` → 等待 `data-render-ready="true"` → 截取 `data-testid="stock-detail-capture"` 区域 → 返回 PNG
- **token**：短期 JWT（`type=capture`，仅截图场景，不可用于常规 API）
- **静态服务**：截图保存到 `CAPTURE_STATIC_DIR`，通过 `CAPTURE_STATIC_URL_PREFIX` 访问

## 6. 前端展示规则

### 6.1 A 股行情颜色规则

事实源：`frontend/src/styles/market-colors.scss` + `frontend/src/features/watchlist-monitor/adapters.ts::changePctColorClass`

A 股行情方向颜色统一口径（红涨绿跌），通过 `.market-up` / `.market-down` / `.market-flat` 类名表达：

| 类名 | 颜色 | 含义 |
|------|------|------|
| `.market-up` | `#d8413f` | 上涨 |
| `.market-down` | `#00a861` | 下跌 |
| `.market-flat` | `#9e9e9e` | 平盘 |
| `.market-up.strong` | `#ff1744` | 上涨强调（现价、涨跌幅等醒目位置） |
| `.market-down.strong` | `#00e676` | 下跌强调 |

**约束**：
- 股票价格方向一律使用 `.market-*` 类名，禁止用 `.pos` / `.neg` 表示股票涨跌方向。
- `.pos` / `.neg`（西方口径，绿涨红跌）仅用于系统状态（成功/失败、健康/异常等），不得用于股票方向。
- 颜色类由 `changePctColorClass(changePct)` 按 `change_pct` 正负统一生成。

### 6.2 选股表格布局规则

事实源：`frontend/src/pages/ScreenerPage.tsx`（DSA 选股表格列定义）

- **列宽**：使用 `<colgroup>` 显式声明各列宽度，确保列 `width` 生效（不被 flex 自动分配覆盖）。
- **股票列**：合并展示「名称 + 涨跌幅 + 代码·市场」于同一单元格，涨跌幅按 6.1 颜色规则着色。
- **当前趋势列**：合并方向与持续时间为单列「当前趋势」（上涨 N 天 / 下跌 N 天 / 方向未形成）。
- **用户友好文案**：表头将 `DSA` / `VWAP` / `offset_*` 等技术字段替换为用户友好列名（详见 `策略与指标口径.md` 2.6.2 节），如「趋势参考价」「距趋势参考价」「日均趋势变化」等。

### 6.3 自选股涨跌幅规则

事实源：`backend/app/strategy_assets/manifests/watchlist_monitor.yaml`（outputs: `previous_close` / `change_pct`）+ `backend/app/strategy/monitors/watchlist_monitor.py`

自选股监控（`watchlist_monitor` v1.1.0）新增涨跌幅输出：

| 字段 | 类型 | 说明 |
|------|------|------|
| `previous_close` | number | 前一交易日收盘价（前复权） |
| `change_pct` | number (percent) | 当日涨跌幅（%） |

**计算口径**：
- `change_pct = (current_price - previous_close) / previous_close * 100`
- `previous_close` 取前一交易日已完成的日线收盘价，**不使用今日未完成的日线**。
- 涨跌幅着色按 6.1 规则（`change_pct > 0` 红、`< 0` 绿、`== 0` 灰）。
