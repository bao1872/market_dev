## 2. 修改前理解

- **当前产品行为**：复权因子由 `bar_repository.rebuild_adj_factors` 按单只股票 + xdxr fingerprint 变化触发重建；`bars_daily.adj_factor` 为权威因子序列
- **发现的缺陷**：
  - xdxr fingerprint 仅能发现"公司行为集合变化"，无法发现"公司行为未变但历史因子序列已错误"的存量错误（如 603538 美诺华 bug 模式：fingerprint 未变但因子已 mismatch）
  - 缺少全市场只读审计能力，无法主动发现存量错误
  - 缺少 dry-run → 串行重建的安全修复路径，无法在写库前预览修复范围
  - `instruments` 表无字段记录"上次对账使用的算法版本"，无法判断哪些股票需要重审
- **已阅读的当前设计文档**：`docs/current/01-system-architecture.md`、`docs/current/02-data-api-contracts.md`、`AGENTS.md`、`backend/app/repositories/bar_repository.py`、`backend/app/services/market_data_aggregation_service.py`

## 3. 修改目标

- **当前问题**：复权因子存量错误无法主动发现 + 缺少安全修复路径 + 缺少对账版本追踪
- **修改后行为**：
  1. 新增 `factor_algorithm_version`/`factor_reconciliation_version` 常量，版本变化时触发全市场重审
  2. 新增只读审计服务 `FactorConsistencyAuditor`：加载 stored 因子 → 重算 expected → 逐日比对 → 分类 mismatch（含 603538 bug 模式识别）
  3. 新增 dry-run → 串行重建修复任务 `FactorReconciliationTask`：全程串行、每只股票独立事务、失败不写 1.0 伪装
  4. `instruments` 表新增 3 列跟踪对账状态（`factor_algorithm_version`/`factor_reconciliation_version`/`factor_reconciled_at`）
  5. `bar_repository.compute_expected_adj_factors` 只读重算（与写库的 `rebuild_adj_factors` 区分）
- **验收标准**：
  - `_compare_factors` 6 类场景单元测试通过（一致 / 603538 bug / 值 mismatch / 容差 / NULL / 行数）
  - `FactorConsistencyAuditor` 零副作用守护测试通过（不写库、不失效缓存、不导入 rebuild）
  - `FactorReconciliationTask` dry_run 零副作用 + rebuild_batch 串行事务隔离测试通过
  - migration 065 upgrade/downgrade/upgrade 验证通过
  - 失败不写 1.0 伪装成功（`error_code` 非空、`after_hash` 为空）
- **明确不修改**：
  - `bars_daily.adj_factor` 仍为权威因子序列（不引入新真源）
  - MDAS 仍为唯一行情读取出口（业务层不直接调用 `_query_*`/`apply_adj_factor*`）
  - 不做无边界全市场回补（只重建审计发现的不一致股票）
  - 603538 真实数据缺失时仅对该股票做小范围补齐/重建，不运行全市场回补

## 5. 实际修改

### 代码文件

**新增**：
- `backend/app/constants/factor_contract.py`（60 行）— 因子算法版本常量
- `backend/app/services/factor_consistency_audit.py`（586 行）— 只读审计服务
  - `FactorConsistencyAuditor.audit_single_stock`：加载 stored 因子 → 重算 expected → 逐日比对 → 分类 mismatch
  - `audit_active_stocks`：分批 yield 全市场审计结果
  - `_compare_factors`：纯函数（可单元测试），分类 603538 bug 模式 / 因子值 mismatch / NULL / 行数不匹配 / 日期序列不匹配
  - `_hash_factor_series`：因子序列内容 hash（before/after 比对 + 缓存键）
  - `FactorAuditSummary` / `summarize_audit_results`：全市场汇总

  - `rebuild_batch`：全程串行，每只股票独立事务，失败回滚不影响其他
  - `_rebuild_single`：调用 `rebuild_factor_series` + 重新审计验证 `after_hash`
  - `ReconciliationPlan` / `ReconciliationItem` / `ReconciliationReport` / `ReconciliationItemResult`：不可变数据类
  - 失败不写 1.0 伪装成功（`error_code` 非空，`after_hash` 为空）

**修改**：
