## 任务名称

docs 信息架构重构为 v2 system map（current + maps + onboarding + restore checklist）

## 需求出处

用户于 2026-07-03 提供 `panji_docs_restructured_candidate_v2.zip` 并明确要求：
"应用 v2 文档重构包，重构 docs/ 为 current + maps + onboarding + restore checklist 结构，
并同步更新 tools/check_docs_consistency.py 及测试。只做 docs/tools，不改生产业务代码。"

## 当前问题

1. 旧 `docs/current/00-18` 共 19 个文件，每个文件头部重复基线字段，维护成本高且易不一致；
2. 缺少系统实现地图（maps），代码入口散落在各文档中，AI 还原系统时缺少结构化索引；
3. 缺少 AI onboarding 与 restore checklist，新会话或灾难恢复时缺少快速入口；
4. `check_docs_consistency.py` 要求每个 current 文档重复基线头，与 v2 单一基线（MANIFEST）模式不兼容；
5. `update_docs.py --check` 依赖旧 00-18 文件名（如 `12-strategy-indicator-contracts.md`），v2 不再使用这些文件名。

## 当前设计理解

- v2 将 current 压缩为 MANIFEST + 6 个核心文档 + open-decisions + code-doc-alignment；
- 基线字段集中在 MANIFEST，其他 current 文档不再重复；
- 新增 `docs/maps/` 8 个实现地图文件，作为代码入口的结构化索引；
- 新增 `docs/AI-ONBOARDING.md`、`docs/RESTORE-CHECKLIST.md`、`docs/MAINTENANCE.md`、`docs/MIGRATION-MAP.md`、`docs/SOURCE-SNAPSHOT.md`、`docs/TRAE-APPLY-INSTRUCTION.md`；
- 旧 current 19 个文件归档到 `docs/archive/current-legacy-20260703/`，不删除；
- `docs/changes/records/` 历史 CHANGE 完整保留。

## 影响模块

- `docs/` 整体结构重构（current + maps + archive + 新增根级文档）
- `tools/check_docs_consistency.py`（重写为 MANIFEST 集中基线规则）

## 实际修改内容

### 文档结构重构

2. 应用 v2 current：MANIFEST + 6 个核心文档（00-product-business、01-system-architecture、02-data-api-contracts、03-jobs-integrations-operations、04-frontend-ux、05-testing-acceptance）+ open-decisions + code-doc-alignment；

6. `docs/changes/records/` 17 个历史 CHANGE 完整保留，未删除。

### check_docs_consistency.py 重写

新规则：
1. 只检查 `docs/current/MANIFEST.md` 的 `实现核对基线` 字段（40 位 SHA）；
2. MANIFEST 的 SHA 必须是真实 git commit；
3. SHA 必须是当前 HEAD 祖先；
4. current 其他文档不要求重复基线头；
5. archive 旧文档不参与 baseline 一致性检查；
6. 保留本地 Markdown 链接检查（docs/ 递归 + AGENTS.md）；
7. 保留 `待填写` 占位符检查（跳过反引号内的规则描述）；
8. 保留 feishu_webhook 当前方案回归阻断（删除语境豁免）；
9. 保留 open-decisions 把 Webhook vs Platform App 写回 OPEN 的阻断（"已决定"豁免）；
10. 输出显示：MANIFEST baseline、current 文档数量、maps 文档数量、链接检查数量。

### test_check_docs_consistency.py 重写

覆盖 11 个场景：
1. MANIFEST 合法 baseline 通过；
2. MANIFEST 缺 baseline 失败；
3. baseline 非 40 位 SHA 失败；
4. baseline 非真实 commit 失败；
5. baseline 非 HEAD 祖先失败；
6. current 文档无重复 baseline 也通过；
7. 坏本地链接失败；
8. 待填写失败（跳过反引号内）；
9. feishu_webhook 当前方案失败；
10. open-decisions 写回 Webhook OPEN 失败；
11. archive 旧 baseline 不触发失败。

测试使用 `tmp_path + monkeypatch`，不依赖真实 docs 被破坏。
