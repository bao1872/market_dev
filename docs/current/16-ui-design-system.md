# 16 UI 设计系统

## 1. 品牌和视觉

品牌为盘迹（PanJi）。界面为深色、专业、研究型；避免收益承诺、夸张钩子、重复高亮边框和无意义空白。上涨红、下跌绿，同时使用文字或形状避免只依赖颜色。

## 2. 布局和组件

页面、侧栏、顶部栏、卡片、表格和图表使用统一间距与层级。Button、Input、Table、Card、Modal、Badge、Toast、Chart Control 和 Job Status 必须定义 loading、disabled、empty、error、partial 和 permission 状态。

## 3. 数据时间和新鲜度

任何行情、策略结果、任务和消息页面都应显示真实数据时间。个股详情至少显示 as_of、数据源、partial/degraded；趋势选股显示业务日期和覆盖率；任务显示心跳与 Git SHA。

## 4. 部分成功

飞书文字成功、图片失败必须使用独立状态和 `partial_failed` 视觉，展示失败步骤和仅重试图片动作，不能使用普通成功 Toast。

## 5. 页面主线

- 发现：全市场趋势特征；
- 验证：产业逻辑链、个股行情和指标；
- 追踪：自选监控、消息和飞书。

产业逻辑区展示上下游、催化、验证点和风险链条，不用另一张 K 线代替。

## 6. 图表

K 线、成交量和指标共享时间轴；DSA 与 Node 图层可开关；缺失数据不连虚假线；partial Bar 有视觉区别；截图模式保留图例、数据时间和重要数值。

## 7. 可访问性

足够对比度、键盘可操作、表单 label、Modal 焦点管理、reduced motion、状态不只用颜色、图表提供文本摘要。
